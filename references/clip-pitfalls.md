> **目标**：下次再撞这个坑，能 1 步查到根因 + 3 修法 + 选 A/B/C 的判定逻辑
> **关联铁律**：#40 V13 范式可控范围 / #41 主体一致性不二次声明

## 1. 场景描述

**绘本**：Good afternoon 8 张图（Pic 绘本）
- **图 3** = 大树下立耳兔举手向树（立耳兔，耳朵竖立朝上）
- **图 4** = 花丛中垂耳兔蹲坐闻花（垂耳兔，耳朵自然耷拉）

**原 prompt 设计**（v15 默认）：镜头一/二/三用 @Image3（立耳），镜头四切到 @Image4（垂耳原图）。**但**用户检查后说"图 4 太丑，我们仍然使用图三的兔子"——要**@Image3 立耳兔 + @Image4 花丛场景**。

**需求**：v15 段 1 不能强制 @Image4 = 垂耳（用户明确否决），但要切到 @Image4 的花丛场景。

---

## 2. V13 范式 = 主体外观 vs 场景分离

**用户原话沉淀**（2026-06-06）："对，就是按 V13 的范式，这也是一种可用的方法，即梦是支持的。"

**写法**：
```
主体小兔子@Image3（= 主角外观，耳朵竖立），
场景参考@Image4（= 花丛+闻花构图）
```

即梦/seedance 2.0 支持"主体外观@图1 + 场景参考@图2"分离，但**视觉权重 vs 文本约束会竞争**。

---

## 3. v2 实测失败

**prompt**（v15 框架 + 主体外观@Image3 + 场景@Image4 分离）：

```
主体定义：奶白色小兔子@Image3（圆润Q版，耳朵竖立朝上，耳朵内侧粉红...），
奶白色小兔子@Image4（...）—— 段 1 对每张图各说一次（铁律 #41）

分镜绑定：@Image3 作为镜头一/二/三/四的主角参考帧（立耳小兔子），
@Image4 作为镜头四的场景参考帧（花丛+闻花构图）；

镜头四（6-11s · 末帧=朗读"flower"· 退场感）：
小兔子@Image3 出现在@Image4 的五彩花丛场景中，耳朵竖立朝上（保持立耳特征），
后腿蹲坐在花丛中...
```

**抽帧对账**（11.1s 视频）：

| 帧 | 期望 | 实际 | 对吗 |
|---|---|---|---|
| 7s | 立耳兔在花丛闻花 | **耳朵竖立** ✅ + 花丛 ✅ + **但动作错了**（举手不是闻花）| ⚠️ 半对 |
| 8s | 立耳兔闻花 | **垂耳兔** ❌ + 花丛 ✅ + 闭眼闻花 | ❌ 耳型飘了 |
| 10s | 立耳闻花退场感 | **垂耳兔** ❌ + 闻花 | ❌ |
| 11s（末帧）| 立耳闻花 | **垂耳兔** ❌ + 闻花 | ❌ |

**结论**：**7-11s 镜头四全部跑出 @Image4 的垂耳特征**——prompt 文本约束的"立耳"**完全被视觉权重压下去了**。

---

## 4. 失败根因

seedance 2.0 `--ref-images 3.jpg 4.jpg` **同时传**时：

1. **每个镜头按时间分段**对应参考图视觉特征
2. **6-11s 镜头四离 @Image4 更近**（时间上更靠后），**@Image4 的视觉特征（垂耳+闻花构图）权重最高**
3. **prompt 文本约束"立耳"被视觉权重压下**——模型在视觉冲突时**优先采用参考图视觉特征**

**用户纠错沉淀**：**模型在多参考图场景下，跨图一致性由视觉权重决定，不是文本约束**——这就是为什么铁律 #41 说"不写'同一个'"——写了也没用。

---

## 5. 3 个修法候选

### 方案 A · `--image` 锁首帧 ⭐ 首选

```bash
# 不传 --ref-images，--image 锁首帧
python3 seedance.py create \
  --image /path/to/3.jpg \
  --prompt "0-6s: 大树下立耳兔举手向树; 6-11s: 切到@Image4花丛，立耳兔蹲坐闻花，末帧退场感 5s" \
  --duration 11 \
  --ratio 16:9 \
  --watermark false --generate-audio true
```

**优点**：角色外观锁死（立耳兔），prompt 自由描述场景切换
**缺点**：失去多图参考，prompt 需要更详细描述 6-11s 的花丛构图
**风险**：未实测（Pic 绘本时间紧没跑 A 方案）

### 方案 B · prompt 加权

保留 `--ref-images 3.jpg 4.jpg`，prompt 加权：
- 段 1 写 "**耳朵必须竖立朝上，不得耷拉**"（加粗/重复）
- 镜头四显式写 "**保持立耳特征**"
- 段 4 风格加 "**耳朵竖立**" 重复约束

**优点**：改动小
**缺点**：v15 标版测试**不一定生效**（视觉权重 > 文本约束）
**风险**：实测 v2 就是 prompt 加了"保持立耳特征"，**7-11s 全部失败**

### 方案 C · 撤回 V13 范式

镜头四直接用 @Image4，**接受垂耳**（Pic 绘本 8 张原图耳型本来就不一致，垂耳是事实不是 bug）。

**优点**：简单，零返工
**缺点**：违反用户"用图三的兔子"明确指令
**判定**：**用户没明确说"必须分离"就回 C**。

## 6. 修复判定逻辑（决策树）

```
镜头需要"主体外观@A + 场景@B"？
  ├─ 是 → 用户是否明确说"必须用 A 的外观"？
  │    ├─ 是 → 走 A 方案（--image 锁首帧）
  │    └─ 否 → 走 C 方案（接受 @B 原图外观，撤回范式）
  └─ 否 → 走 v15 默认（每个镜头主语 = 该镜头对应图）
```

## 9. 关键 takeaway

> **seedance 2.0 的视觉参考权重 > prompt 文本约束**——任何"主体外观 vs 场景分离"的镜头，**首选 `--image` 锁首帧**，不要指望 prompt 文本约束。
