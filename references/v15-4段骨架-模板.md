# v15 4 段骨架 · 模板（v1.0.3+pic12 新增 · 2026-06-07）

> **用户根本性纠错**（2026-06-07）："底层核心逻辑是提示词的写法结构，所有任何场景都是在这个结构上做微调。"
>
> **含义**：v15 4 段骨架 = **固定的 4 段写法** = 模板。所有场景（绘本/漫剧/故事/广告/动画）= **在模板上做变量微调**，**不**是"每次从 0 拼"。

## 模板正文（v1.0.3+pic12 · 主 agent 必用）

```yaml
# v15 4 段骨架模板 · 填变量即可（不要改结构）
# 所有场景（绘本/漫剧/故事/广告/动画）共用这个结构

段 1 · 主体定义
主体定义：{主角1}@Image{N}（{视觉特征1} + {视觉特征2} + {视觉特征3} + {姿态/动作} + {表情}），{配角2 if 多主体}（{视觉特征} + {动作}）；{背景}@Image{N}（{主色} + {次色} + {纹理} + {氛围}）；

段 2 · 分镜绑定 + 文字保留 v2
分镜绑定：@Image{N} 作为唯一参考帧；
{镜头1 描述}{拟声1}一响，{画面微动元素1}
{镜头2 描述}{拟声2}一响，{画面微动元素2}
{镜头3 描述}{拟声3}一响，{画面微动元素3}
{镜头4 描述}{拟声4}一响，{画面微动元素4}
{镜头5（可选 收势）}{拟声5}一响，{画面微动元素5}
{末帧策略}画面继续微动（{末帧微动具体动作1} + {动作2} + {动作3}），末帧 {X}s 静默消化时间，末帧 1s 内必须包含至少 1 个动作元素（{必含动作}），不得成为定格海报；

参考图原有的所有文字（顶部 1/6 画面的彩色拼贴英文 "{EN_WORD}" 和中文"{ZH_WORD}"字）必须完整保留作为画面元素，模型不得删除或替换这些文字，文字位置锁定在顶部 1/6 画面不要重新生成该区域内容，让文字自然融入场景；

段 3 · 风格锁定（v14/v15 继承）
风格：2D paper collage style, {具体风格词1}, {具体风格词2}, {具体风格词3}。

段 4 · BGM 段（v15 新增 v7-Say 密集音效型）
无任何背景音乐、无旁白人声、无哼唱。
```

## 11 个变量（必填）

| # | 变量 | 来源 | 必填依据 |
|---|---|---|---|
| 1 | `{N}` | 该 Clip 对应的图序号 | 主 agent 知道（来自 C 子 agent JSON 的 `image_index`） |
| 2 | `{主角1}` + `{视觉特征1-3}` | C 子 agent `characters[].visual_features` | vision_analyze 实战 |
| 3 | `{姿态/动作}` + `{表情}` | C 子 agent `characters[].action` + vision | vision_analyze 实战 |
| 4 | `{背景}@Image{N}（{主色} + {次色} + {纹理} + {氛围}）` | A 子 agent `style_recognition.page_by_page_observations[].key_visual` | A 跑过 |
| 5 | `{镜头1-5 描述}` + `{拟声1-5}` | C 子 agent `shots[].action` + `shots[].sfx` | C 必填 |
| 6 | `{末帧微动具体动作1-3}` | C 子 agent `end_frame_microaction` | C 必填 |
| 7 | `{X}s 静默消化` | B 子 agent `silence_recommendation_seconds` | B 算过 |
| 8 | `{必含动作}` | C 子 agent 实战校准 1 (`_实战校准_末帧微动_具体动作`) | C 必填 |
| 9 | `{EN_WORD}` + `{ZH_WORD}` | 旁白 `narration_text.en` + `narration_text.zh` | 已知 |
| 10 | `{具体风格词1-3}` | A 子 agent `style_anchors[]` | A 跑过 |
| 11 | `{EN_WORD 位置}` 锁 | "顶部 1/6 画面"（绘本默认）| 锁位置不重绘 |

## 主 agent 填模板的工作流（v1.0.3+pic12 实战）

```python
# 主 agent 拿到 C 子 agent 的 clips/clip{N}.json + A 的 style-recognition.json + B 的 narration-quantization.json
# 填 11 个变量 → 写出 v15 终稿 prompt → 存到 clips/clip{N}-prompt.txt → 跑 seedance.py

def fill_v15_template(clip_meta, style, narration, image_index):
    variables = {
        'N': image_index,
        '主角1': clip_meta['characters'][0]['name'],

        '视觉特征3': clip_meta['characters'][0]['visual_features']['feature_3'],
        '姿态/动作': clip_meta['characters'][0]['action'],
        '表情': clip_meta['characters'][0]['expression'],
        # ... 其他 10 个变量
    }
    
    return V15_TEMPLATE.format(**variables)
```

## 4 段骨架的"为什么 4 段不是 3 段/5 段"（用户原话背书）

- **段 1 主体定义** = "画面有什么"（角色 + 背景 + 视觉特征）——**必填**（不写 = 模型自由发挥）
- **段 2 分镜绑定 + 文字保留** = "画面怎么动"（@ImageN 锁角色 + 镜头序列 + 拟声 + 文字保留）——**必填**（不写 = 模型自由切分镜头 + 文字消失）
- **段 3 风格锁定** = "画面长啥样"（2D paper collage + 具体风格词）——**必填**（不写 = 风格漂移到 3D/写实）
- **段 4 BGM 段** = "画面怎么响"（无 BGM + 拟声 = 绘本场景；或音频参考 = 漫剧场景）——**必填**（不写 = Seedance 自带 BGM/人声，绘本场景翻车）

**4 段都不能省**。**省任何 1 段 = 翻车征兆**（pic2 实战验证）。

## 不同场景的"变量微调"（v15 模板不变，只调变量）

| 场景 | 段 1 主体定义 | 段 2 镜头描述 | 段 3 风格 | 段 4 BGM |
|---|---|---|---|---|
| **绘本**（默认）| 角色 + 背景 + 视觉特征 | 1-5 镜头（语义块）| 2D paper collage | 无 BGM + 拟声密集 |
| **漫剧** | 角色 + 场景 + 情绪 | 1-3 镜头（动作导向）| 写实 / 3D | BGM（情绪配乐）+ 台词 |
| **故事动画** | 角色 + 场景 | 1-4 镜头（节奏块）| 水彩 / 油画 | BGM 调性匹配 |
| **广告** | 品牌 + 产品 + 场景 | 1-2 镜头（精炼）| 高级感 | 品牌 BGM |

**段数可以变，4 段结构不变**。**所有"加"都是变量微调，不动模板**。

## 与 v13/v10 范式的区别

| 范式 | 段数 | 适用场景 | 缺点 |
|---|---|---|---|
| v13 | 2-3 段 | 短动画 | 没文字保留 = 翻车 |
| v10 | 5+ 段 | 领读绘本 | 段太多 = prompt 长 + 慢 |
| **v15**（默认）| **4 段** | **绘本/漫剧/故事/广告** | **平衡**（段 1-4 都不能省，加段 = 改 v15.1）|

## 红线

- ❌ **不能改模板结构**（4 段不能合并/拆分/重排）
- ❌ **不能省任何 1 段**（4 段必填，少 1 段 = 翻车）
- ❌ **不能"另起炉灶"写新模板**（v13 / v10 / v8 已废弃，全部 v15 化）
- ❌ **C 子 agent 不能写 prompt_draft 字段**（避免 prompt 写法不一致）
- ❌ **段 2 不写"朗读 X 词"指令**（v1.0.5+pic14 实战新增 · 2026-06-09 Pic7 Horse R7 翻车）——seedance 把"朗读 '家族词组'"当必生成朗读指令 → 5 词家族词组 = 5 次独立朗读 → 抢节奏 + 重复发 + 错乱（用户反馈"完全错乱的语音"）。**v15 段 2 = 镜头描述 + 拟声**（音效嵌入视觉句，用 `,` 串接），**不**是朗读台词表。
- ❌ **段 4 不写具体音效清单**（v1.0.5+pic14 实战新增）——段 4 BGM 段 = **声音策略的兜底句**（A 档"无任何背景音乐、无旁白人声、无哼唱" / B 档"画面元素动作音效保留...，不发完整朗读，TTS 后期对齐"），**不**是拟声清单。**具体音效描述**属段 2 镜头描述末尾的拟声字段，**不**属段 4。
- ❌ **段 2 文字保留约束不重复 2 遍**（v1.0.5+pic14 实战新增）——v15 段 2 文字保留 = **1 段独立约束**，**不要**在 prompt 里复制 2 遍（重复 = 模型困惑 + token 浪费）。
- ✅ **主 agent 必填 11 变量**（缺 1 = self_check 不通过）

## 反模式（v1.0.5+pic14 实战扩到 8 条）

- ❌ "我根据这个 Clip 特殊需求重新设计 prompt 结构" → **不**允许，**永远 v15 4 段**
- ❌ "段 1 主体定义太长了，省略" → **不**允许，**段 1 必填视觉特征**
- ❌ "段 3 风格锁定我用 3D" → **不**允许乱改，**2D paper collage 是绘本默认**
- ❌ "段 4 我加背景音乐" → **不**允许，**绘本场景必禁 BGM**（避免拟声被覆盖）
- ❌ "我把 4 段合并成 1 段连续写" → **不**允许，**4 段独立 = 模型理解稳定**
- ❌ **"段 2 我写'朗读 家族词组/第2词/第3词'指令"** → **不**允许（v1.0.5+pic14 实战新增 · 2026-06-09 Pic7 Horse R7 翻车）——seedance 把每个"朗读 X 词"当**独立朗读指令** → 家族词组 5 词 = 5 次独立朗读 → **抢节奏 + 重复发 + 错乱**（用户反馈"完全错乱的语音"）。**v15 段 2 = 镜头描述 + 拟声**（音效嵌入视觉句，用 `,` 串接），**不**是朗读台词表。**段 2 绝不写** `朗读 "..."` 这种动词 + 引号 + 旁白文本的格式。**TTS 朗读 = 后期对齐**（不在 prompt 里指挥）。
- ❌ **"段 4 我加'卡片高亮配叮/图标浮现配叮咚/...'"** → **不**允许（v1.0.5+pic14 实战新增）——段 4 BGM 段 = **声音策略的兜底句**（A 档："无任何背景音乐、无旁白人声、无哼唱" / B 档："画面元素动作音效保留...，不发完整朗读，TTS 后期对齐"），**不**是拟声清单。**具体音效描述**（"卡片高亮配叮/图标浮现配叮咚..."）属于**段 2 镜头描述末尾的拟声字段**（紧贴动作，用 `,` 串接），不属于段 4。
- ❌ **"段 2 文字保留我复制 2 遍以强调"** → **不**允许（v1.0.5+pic14 实战新增）——v15 段 2 文字保留 = **1 段独立约束**（"参考图原有的所有文字（...）必须完整保留作为画面元素..."），**不要**在 prompt 里复制 2 遍（重复 = 模型困惑 + token 浪费 + 段 2 句号断句切碎视觉）。

**实战教训**（2026-06-09 Pic7 Horse R7 v1.0.4 重跑）：clip4 prompt 同时违反上述 3 条 = 视频跑出"完全错乱的语音"（用户原话反馈"里面的语音都是错的，完全都错乱了"）。**修复**：删"朗读 X 词" + 段 4 改 B 档兜底句 + 段 2 文字保留只留 1 遍 → 重跑（task `cgt-20260609201914-stqmd` · seed 76737 · 5.6MB · 14s）→ 声音错乱问题闭环。详细实战数据见 [references/2026-06-09-pic7-horse-validation.md](references/2026-06-09-pic7-horse-validation.md) §clip4 v15strict 重跑。
