# video-prompt-narrative · 叙事设计技能

> 本技能解决：Prompt 写出来好看但视频「突然断了」的根因问题。
> 通过叙事结构分析 + 写法选择 + 收势设计，让每个 Clip 有完整的起承转合和收势。

---

## ⚠️ 根因先行

**「突然中断」不是剪辑问题，是 Prompt 叙事设计问题。**

模型收到 Prompt 后生成视频，如果 Prompt 只描述「画面里有什么」，模型会：
- 每个时间格生成独立小镜头
- 动作进行中时间到就戛然而止
- 没有设计「收势」的动作落点

**解决路径**：
```
叙事内容分析 → 选择写法（连续运镜/时间轴分镜）
→ 设计收势动作 → 写进入 Prompt → 生成完整视频
```

---

## Step 1 · 叙事内容分析

### 判断维度

| 维度 | 连续运镜 | 时间轴分镜 |
|------|---------|----------|
| 情节结构 | 单线动作/单一情绪 | 多段节奏/有明显转折 |
| 镜头需求 | 推/拉为主，景别连续变化 | 不同景别切换 |
| 收势特点 | 镜头停在最终状态 | 各段分别收尾 |
| 典型场景 | 情感特写/传承/静物 | 舞蹈高潮/多人物互动/多场景转折 |

### 决策树

```
情节是单线动作还是多段节奏？
├── 单线动作/单一情绪 → 连续运镜写法
│   └── 示例：小女孩举火把仰望、祖孙传承对视
└── 多段节奏/有转折 → 时间轴分镜写法
    └── 示例：篝火舞蹈（起→承→高潮→收）、多场景故事
```

## Step 2 · 连续运镜写法

### 适用场景
- 单线动作（举火把、行走、凝视）
- 单一情绪（宁静、专注、传承）
- 镜头推近特写然后停住

### 写法结构

```
[主体+环境] + [动作描述] + [镜头运动] + [收势定格]
```

**⚠️ 时长分配原则（通用逻辑）**：
- 前段（起/承）：尽量简洁，不做过多铺陈，让路给收势
- 收势阶段：给足空间，确保有「落住」的感觉
- 核心原则：前段不能拖到影响收势的完整性
- 收势的本质：动作/情绪「落住」的瞬间，不是「动作进行中被截断」
- **注意**：时长分配是内容层面的原则，不是硬编码数字。应根据实际内容灵活调整，而非套用固定比例

### 示例

**Clip 3（小女孩举火把·连续运镜）**：
```
中景深蓝夜空下，小女孩双手握持燃烧的火把，身体微微前倾，目光凝视火焰，面带微笑，神情专注而宁静。火把火焰轻轻摇曳，橙红火光映照她的脸庞。镜头缓慢稳定推近至面部特写。最终镜头缓缓拉远至全景，小女孩保持举火把仰望的静态姿势，火焰稳定燃烧，停留3秒后自然收尾。
温暖插画风格，色彩饱和，叙事完整有悠长余韵。
```

**Clip 9（祖孙传承·连续运镜）**：
```
室内传统居所，阳光从红色木门斜射。老人白发深棕服饰，双手稳稳握持燃烧的火把，火把顶端松枝火焰稳定燃烧。孩童坐在身旁，扎双辫戴黑色尖顶帽，专注凝视老人手中火把，神情柔和好奇，老人温和微笑眼神柔和看向孩童。镜头缓慢推进至祖孙对视面部特写，火光映照两人脸庞。最终画面定格：镜头完全静止不动，结尾画面不渐隐不淡出，停留至最后一帧。
温暖插画风格，色彩饱和，叙事完整有传承沉淀感。
```

### 收势写法要点

必须包含：
1. **收势阶段明确标注**：「最终镜头缓缓拉远至全景」「最终定格」
2. **收尾动作质感**：「缓慢拉远」「镜头稳定推近」「停留X秒」
3. **时长标注**：「保持X秒不动」「停留3秒后自然收尾」

### 执行检查清单（连续运镜）

- ✅ 无时间切分/shot标记，整段连续运镜
- ✅ 收势阶段明确写出「最终定格」「缓慢拉远」等词
- ✅ 收势停留时长≥3秒
- ✅ 旁白不写在 prompt 里

## Step 3 · 时间轴分镜写法

### 适用场景
- 多段节奏（舞蹈有起承转高潮）
- 需要不同景别切换
- 有明显转折点

### 写法结构

```
[00-XXs] Shot N: [景别] [画面描述]
[XX-XXs] Shot N+1: [景别] [画面描述]
...
[收势段]: [收势动作描述] + [收势镜头处理]
```

### 示例

**Clip 7（篝火舞蹈·时间轴分镜）**：
```
[00-02s] 中景篝火熊熊燃烧，火焰橙黄翻腾。小女孩站篝火前中央，双手高高举过头顶，脚尖点地，欢快舞蹈。
[02-04s] 周围村民围篝火半圆形，左侧女性演奏三弦月琴，右侧男性演奏吹管乐器，众人微笑注视小女孩。
[04-06.5s] 小女孩舞蹈达高潮，旋转身体裙摆飞扬，周围人物跟随节奏拍手，火把在夜风中摇曳。
[06.5-09s] 小女孩定格在双手高举姿态，篝火火焰稳定燃烧，镜头缓慢拉远至篝火全景，保持静态姿势3秒后自然收尾。
温暖插画风格，色彩饱和，叙事完整。
```

### 收势写法要点

每个收势 shot 需包含：
1. **收势 shot 的具体动作**：「定格」「稳定燃烧」「静止姿势」
2. **镜头运动**：「镜头缓慢拉远至XX」
3. **时长标注**：「镜头完全静止不动，结尾画面不渐隐不淡出，停留至最后一帧」

### 执行检查清单（时间轴分镜）

- ✅ 有明确的时间切分标记 `[00-XXs]`
- ✅ 不同 shot 有不同景别或动作内容
- ✅ 最后一 shot 是收势 shot，明确写出「定格」「保持静态」
- ✅ 旁白不写在 prompt 里

## Step 4 · 收势设计

### 收势的本质

收势 = 动作/情绪「落住」的瞬间。

不是「动作做完」，而是「镜头在完成状态上停住」。

### 时长分配原则：基于旁白内容推导

**核心逻辑**：时长分配由旁白的叙事功能决定，不是硬编码比例。

根据旁白内容判断：

| 旁白类型 | 旁白在做什么 | 时长分配逻辑 |
|---------|------------|------------|
| **交代型** | 说明背景/引入场景 | 视觉快速交代，2-3秒足够，重点在收势落住 |
| **动作型** | 描述一个完整动作事件 | 起→承→转→合，顺着动作节奏走，收势是动作完成态 |
| **情感型** | 传递情绪/感受/氛围 | 前段简洁，快速进入情感核心，收势给足余韵 |
| **高潮型** | 情绪到达峰值 | 前段快节奏推进，收势给到峰值后的稳定态 |

**判断优先级**：
1. 这个旁白在叙事功能上是什么？（交代/动作/情感/高潮）
2. 旁白的核心信息落在哪个词/哪个moment？
3. 为了把这个核心信息传递给观众，视觉需要多长来呈现？

### 三种收势模式

| 模式 | 收尾动作 | 适用类型 |
|------|---------|---------|
| **完成式** | 动作已完成，镜头停住 | 单线动作 |
| **定格式** | 人物在最终姿态静止，镜头拉远 | 舞蹈高潮/情感特写 |
| **交接式** | 镜头在完成状态上稳定停住 | 传承/交接场景 |

### 收势写法检查清单

- ✅ 显式写出「收势」或「最终定格」
- ✅ 写出镜头运动（拉远/推近/停住）
- ✅ 写出停留时长「保持X秒不动」
- ✅ 情感镜头收势≥3秒

## Step 5 · 旁白处理

### ⚠️ 旁白写在 Prompt 里——待后续完善

> **重要纠正（2026-05-26）**：Seedance 模型具备旁白生成能力，当前问题是**还未找到正确的写法**，而非「旁白不能写进 Prompt」。
> 当前策略：旁白在剪辑阶段合成，Prompt 里不写旁白。
> 待完善方向：探索旁白写入的合适写法（如标注位置、语气、时机等）。

> 💡 已知经验：Seedance 默认 `--generate-audio true`，Prompt 中写旁白文本会触发模型生成旁白音频，但旁白时机与画面不一定是最佳匹配，有优化空间。

## 经验教训（实测总结）

1. **起承转合是叙事弧，不是分镜数量**：同一镜头内通过运镜可以完成整个叙事弧，不能教条化
2. **两种写法都可用**：连续运镜 vs 时间轴分镜，根据叙事内容自动选择
3. **收势是叙事完成感的关键**：给到 3-4 秒情感镜头收势
4. **旁白不写进 Prompt**：剪辑阶段合成，不在 AI 生成阶段
5. **Prompt 禁止写「旁白：...」**：会触发 AI 生成旁白音频
6. **收势写法核心词**：「最终画面定格：镜头完全静止不动，结尾画面不渐隐不淡出，停留至最后一帧」
7. **情感镜头时长分配**：前段让路给收势是通用逻辑，但比例应根据旁白叙事功能灵活调整
