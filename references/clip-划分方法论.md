## 核心原则

**Clip 时长划分的第一依据 = 旁白实际朗读时长**。
不是图的数量、不是叙事弧、不是均分总时长。
**画面跟随旁白走,不是旁白迁就画面**。

---

## 一、问题背景

### 错误做法(当前 skill 实际执行结果)

```
收到 8 图 + 8 段旁白 + 用户 TTS 31s
↓
按"叙事弧 + 图分组"划分 → 3 图 = 1 clip
↓
把总时长 31s 均分到 4 个 clip
↓
clip1 = 6s 装了旁白段 1+2+3
↓
❌ 6s 根本读不完 3 句旁白 → TTS 说到段 3 时画面已经切走
```

### 正确做法

```
收到 8 图 + 8 段旁白 + 用户 TTS 31s
↓
先逐段计算旁白朗读时长
↓
按朗读时长 + clip 上限 [4, 15] 合并相邻段
↓
每 clip 时长 ≥ 其包含旁白段朗读时长总和
↓
✅ 每个 clip 都装得下自己的旁白
```

## 二、标准流程(5 步)

### Step 1 · 逐段计算旁白朗读时长

**领读型绘本**(中文列含嵌入英文目标词 · TTS 只读中文列):

```
每段 TTS = 中文字数 / 中文速率 + 英文词数 / 英文速率 + 词间停顿
```

**速率档位**(从 skill 铁律 #25 的 5 档中选最匹配用户 TTS 的那档):

| 档位 | 中文速率 | 英文速率 | 适用场景 |
|---|---|---|---|
| 慢估(兜底) | 3.0 字/秒 | 1.0 词/秒 | 长句 + 复杂词 |
| 领读短句档 | 4.0 字/秒 | 1.4 词/秒 | 绘本短句 + 双语领读 |
| 偏快档 | 4.0 字/秒 | 1.5 词/秒 | 极短单词开篇 |

### Step 2 · 累加相邻段,合并到 [4, 15] 区间

```
从段 1 开始累加
累加 + 下一段 ≤ 15s ?
├─ 是 → 继续合并(进入下一组)
└─ 否 → 当前组 = 1 clip(开始新组)
```

### Step 3 · 向上取整到 [4, 15] 整数

每组时长向上取整到 4-15s 范围的整数。

### Step 4 · 总和对齐用户 TTS

每组拟时长总和 vs 用户给 TTS → 差 ≤ 5s 内可接受,超出必调整拆分策略。

### Step 5 · 图跟随旁白段分配

**画面跟随旁白走** — 图按其对应的旁白段分配到 clip。

## 三、反模式(必避)

| 反模式 | 描述 | 后果 |
|---|---|---|
| 先按图分组再凑时长 | 3 图 = 1 clip → 均分总时长 | clip 时长 < 旁白实际时长 → TTS 画面错位 |
| 以叙事弧决定 clip 数 | 3 弧 = 3 clip → 不算旁白时长 | 弧内旁白段太多 = 超时 |
| 总时长均分到 N 个 clip | 31s / 4 = 7.75s → 每 clip 8s | 不考虑各段旁白长短差异 |
| 不算词间停顿 | 8 段 × 0.5s = 4s 被漏算 | 总时长偏短 4s |
| 以图数量决定 clip 数量 | 8 图 = 4 clip(2 图/clip) | 旁白段长短不一 = 有的 clip 装不下,有的 clip 空废 |

## 四、Skill 优化建议

### 强化铁律 #33

**现有**:TTS 旁白时长 = Clip 合并/拆分唯一核心约束
**建议改为**:

```
先逐段计算 TTS → 再合并 → 每 clip 时长 ≥ 旁白段 TTS 总和
```

### 删除 #M1 整本节奏作为 Step 4 第一依据

`#M1` 整本节奏原则 = 整本视角分配,不是"按叙事弧决定 clip 数"。
权威文档 = "先按旁白时长分 clip → 再考虑整本节奏 → 再考虑视觉"。
**优先级反转** = 旁白 > 整本节奏 > 视觉。

### 修正 Step 4 流程

**旧**:通读旁白 → 标叙事弧 → 5 项一致性合并到 N 个 Clip(导演分镜)
**新**(权威文档 5 步):

1. 逐段计算旁白朗读时长(选最匹配档位)
2. 累加相邻段,合并到 [4, 15] 区间
3. 向上取整到整数
4. 总和对齐用户 TTS(差 ≤ 5s)
5. 图跟随旁白段分配
