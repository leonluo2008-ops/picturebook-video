# 参考图驱动 prompt 设计法（**2026-06-16 Run 跑 第 1 次跨本验证**）

> **核心命题**：视频画面 = 参考图的**动态呈现**，**不是**凭印象重新设计。
>
> **用户原话（必读 5 遍）**：「**视频的镜头画面设计，要充分建立在参考图和旁白的基础上，本来我发给你的参考图和旁白是具有强关联对应关系的，你在设计视频画面时，必须以参考图为重要依据，让视频流畅合理，尽量避免和参考图冲突。比如参考图中的人物所处的位置，朝向，画面布局等，视频应该是参考图的动态呈现，而不是重新设计视频**」

## 决策逻辑（#M2 规则推导）

| 维度 | 反模式（凭印象设计）| 决策逻辑（参考图驱动）|
|---|---|---|
| **主体位置** | 写"小男孩在中央"但参考图实际在左侧 | **写 prompt 前先查参考图主体在哪个位置 = 写进 prompt = 不挪位置** |
| **主体朝向** | 写"张臂迎面跑来"但参考图是侧身朝右跑 | **写 prompt 前先查主体朝向 = 写进 prompt = 不改朝向** |
| **景别** | 写"缓推到中近景"但参考图本就是全身景别 | **写 prompt 前先查景别 = 固定原景别 = 不推不拉不切新景别** |
| **主体姿态** | 写"张臂双脚交替"但参考图是张臂侧身单脚在前 | **写 prompt 前查姿态 = 保留姿态做"循环动作版"**（跑步 = 前后腿交替 + 双臂前后摆动）|
| **招牌/文字位置** | 写"招牌在画面顶端"但参考图在右侧中央 | **写 prompt 前查招牌位置 = 写进 prompt = 不挪位置** |
| **凭空加元素** | 写"花瓣轻飘"但参考图没有花瓣 | **只写 image-inventory "有"列里的元素**（铁律 #3 #4 强化版）|

## 写 prompt 前必查 5 项（**5 项自检清单**）

```
写每个 Clip prompt 前必走 5 问：

1. 主体在画面哪个位置？（左/中/右/上/下）
   → 写到 prompt："主体位于画面 XXX @ImageN"

2. 主体朝向哪里？（面对镜头/侧身/背面/朝左/朝右）
   → 写到 prompt："侧身朝右 / 正面朝镜头"

3. 景别是什么？（全身/中景/半身/特写）
   → 写到 prompt："保持 XXX 景别" + "固定 XXX 景别不做推拉"

4. 主体姿态具体是什么样？（手/脚/头/躯干位置）
   → 写到 prompt："左脚在前右脚在后 + 双臂前后摆动 + 头微低微笑"

5. 招牌/文字位置在哪里？（顶部/中央/右侧/角落）
   → 写到 prompt："招牌位于画面 XXX @ImageN" + "红色'跑'字位于右下角 @ImageN"
```

**5/5 必查 · 漏 1 项 = 必翻车**（v1 Clip 3 漏 3 项 = 5 处冲突全部浮现）。

## image-inventory 必含 3 列（**Step 2.0 强化版**）

| # | 风格 | 主体 | 背景 | 文字 | 动作 | 场景跳变 | **主体位置** | **朝向** | **景别** | 有 | 没有 |
|---|---|---|---|---|---|---|---|---|---|---|---|

**说明**：
- 主体位置 = 左/中/右/上/下（**必写** · 决定 prompt 第 1 问答案）
- 朝向 = 面对镜头/侧身/背面/朝左/朝右（**必写** · 决定 prompt 第 2 问答案）
- 景别 = 全身/中景/半身/特写（**必写** · 决定 prompt 第 3 问答案）
- 招牌位置 = 在"文字"列里具体写"右侧中央" / "顶部" / "右下角"（**必写** · 决定 prompt 第 5 问答案）

## 正确范式（prompt 模板 · 参考图驱动版）

```yaml
主体定义：[主体描述]位于画面[位置] @Image[N]（[视觉特征]），[招牌描述]位于画面[位置] @Image[N]；[背景描述] @Image[N]（[背景特征]）；

分镜绑定：@Image[N] 作为唯一参考帧，必须保持参考图的整体构图（[主体位置]+[招牌位置]的版式）不重新设计画面布局；
镜头一（0-Xs · [动作主题]）：保持参考图原有[景别]景别不做推拉，固定[景别]，[主体描述]保持[姿态]+循环[动作分解]，[招牌描述]保持原位不动，末帧微动：[1s 内动作元素清单]；

参考图原有的所有文字（[招牌描述]，[文字描述]）必须完整保留作为画面元素，模型不得删除或替换这些文字，文字位置锁定在参考图原有位置（[位置]+[位置]）不要重新生成该区域内容，让文字自然融入场景；

风格：[风格词清单]。

段 4 · BGM 段：[声音策略]。

[4 行末尾约束]
```

**关键约束**：
- ✅ "保持参考图的整体构图...不重新设计画面布局" = 显式说给模型听（避免模型自由发挥）
- ✅ "固定 XXX 景别不做推拉" = 显式说不动景别
- ✅ "[招牌描述]保持原位不动" = 显式说不动招牌
- ✅ "[主体描述]保持[姿态]" = 显式说姿态不变

## 反模式（必避 · 5 类）

| 反模式 | 实战错例 | 修复 |
|---|---|---|
| **凭印象写朝向** | "张臂迎面跑来"（参考图是侧身朝右跑）| 写"侧身朝右跑步姿态 + 前后腿交替 + 双臂前后摆动" |
| **凭印象写景别** | "缓推到中近景"（参考图本是全身）| 写"保持全身景别 + 固定景别不做推拉" |
| **凭印象写位置** | "招牌在画面顶端"（参考图在右侧中央）| 写"招牌位于画面右侧中央 @ImageN + 红色'跑'字位于右下角" |
| **凭印象加元素** | "花瓣轻飘"（参考图没有花瓣）| 只用 image-inventory "有"列里的元素 |
| **凭印象写动态** | "笑容越来越大越来越近"（参考图是跑向右侧，不是跑向镜头）| 写"保持跑步姿态 + 跑向右侧招牌方向" |
