# video-prompt - 分镜生视频技能

## 技能描述
分镜生视频技能。将校验完成的Clip表中的每个Clip，翻译成AI能直接执行的视频生成提示词，然后调用视频工具根据Clip表**并行生成所有视频片段**。

它是**视频创作流程 Phase 8「分镜生视频」**的核心技能，完成参考素材生成和用户确认后，调用本技能批量生成所有视频片段。

---

## 适用场景
- 故事成片创作：完成所有前期准备（分镜、素材、校验）后，批量生成所有Clip视频
- 单个Clip修改：用户对某个Clip不满意，重新生成该Clip
- 营销广告视频：对应 `video-prompt` 通用规则同样适用，营销场景也可直接使用

---

## 核心原则
1.  **完全忠实于Clip表**：Prompt必须完全基于分镜描述，不增加不删减内容
2.  **一致性优先**：明确要求保持参考主体的外观、服装、场景与参考图一致
3.  **清晰动作描述**：用动作动词明确描述镜头内的动态变化，让AI理解运动方向
4.  **并行生成**：所有Clip一次性并行提交生成，不串行分批

---

## Prompt编写规范
每个Clip的Prompt必须包含以下要素，顺序如下：

1.  **参考主体引用**：如果用到我们生成的参考主体，必须写 `<subject>主体名称</subject>`；如果用到用户提供的普通参考图，写 `@图片N`
2.  **场景描述**：描述当前Clip所在场景，明确要求保持场景与参考图一致
3.  **分镜动作描述**：按顺序描述每个镜头的画面内容和人物动作，明确动态变化
4.  **台词/音效说明**：标注本片段对应的台词、旁白或音效（不需要生成音频，只是提示画面配合）
5.  **风格一致性说明**：明确要求整体风格与参考素材保持一致，画面连贯自然

---

## Prompt示例
```
<subject>男主_遗忘事务所</subject>坐在办公桌后，<subject>女主_遗忘事务所</subject>坐在对面，场景保持<subject>办公室_遗忘事务所</subject>一致。
男主抬头看向女主，女主慢慢抬头说出台词，女主推过表格给男主。
台词：女主说"我想忘记他"，男主说"填写详细信息"。
整体风格保持冷灰写实色调，人物动作自然流畅。
```

---

## 工具选择规则
根据Clip的输入情况选择正确的视频生成命令：
1.  **纯文生视频**：没有参考图/参考主体（极少情况）→ 使用 `text2video`
2.  **已有首帧参考图**：以参考主体图作为首帧 → 使用 `image2video`，将首帧资源ID放入 `resource`
3.  **多参考素材（角色+场景+动作参考）**：使用 `multi_modal2video`，将所有参考图放入 `ref_images`，参考视频放入 `ref_videos`

### 参数默认值
- **时长**：使用Clip表中计算出的总时长（必须严格与表中一致，不能超出15秒）
- **宽高比**：默认 `16:9`（符合常规视频比例），用户有特殊要求按要求调整
- **分辨率**：默认 `720P`
- **模型版本**：默认 `seedance2.0_vip`

## 输出格式规范
### 文件命名
`{故事名称}_视频生成任务清单.md`

### 内容结构
```markdown
# {故事名称}视频生成任务清单

| Clip编号 | 输出ID | 生成命令 | 参考素材 | 时长(秒) | Prompt |
|----------|--------|----------|----------|----------|--------|
| 1 | [output_id] | [命令类型] | [参考主体/参考图ID列表] | X.X | [Prompt完整内容] |
| 2 | ... | ... | ... | ... | ... |

## 总信息
**总Clip数：** N个
**预计总时长：** XX.X 秒
```

---

## 工具调用规范
1.  所有Clip必须**并行调用 `dreamina_cli` 一次性提交**，不允许串行分批生成
2.  每个Clip对应一次独立的工具调用，每个调用有独立的 `output_id`
3.  `output_id` 命名规范：`{故事名}_clip_{序号}`，全部小写，下划线分隔
4.  检查每个Clip的时长是否都 ≤ 15秒，严格遵守硬约束

---

## 参考主体资源分配规则（非常重要）
如果Clip中用到 `<subject>主体名称</subject>`，必须：
1.  找到该主体对应的 `output_id`（资源ID）
2.  如果是 `image2video` 并且主体就是首帧 → 将主体资源ID放入 `resource` 参数
3.  如果是 `multi_modal2video` → 将所有用到的参考主体资源ID放入 `ref_images` 参数
4.  绝对不能只在Prompt中写 `<subject>主体名称</subject>` 但不填入对应资源ID

---

## 执行流程
1.  读取校验完成的Clip表和用户选定的参考主体
2.  为每个Clip编写Prompt，确定生成命令和参数
3.  整理生成任务清单写入Markdown文件
4.  所有Clip并行调用 `dreamina_cli` 一次性提交生成任务
5.  提交完成后告知用户正在生成，等待所有片段生成完成后进入视频剪辑阶段

---

## 执行检查清单
✅ 确认每个Clip的Prompt完全忠实于分镜描述，没有新增内容
✅ 确认所有参考主体都已正确引用，并且资源ID已填入对应参数
✅ 确认每个Clip时长都 ≤ 15秒，严格遵守硬约束
✅ 确认所有输出ID命名规范，每个ID唯一
✅ 确认所有生成任务并行提交，没有串行分批
✅ 确认工具命令选择正确（image2video/multi_modal2video/text2video）
✅ 确认宽高比、分辨率等参数设置合理

## 最终阶段说明
完成本步骤后，所有视频片段生成完毕，接下来进入 `video_editor` 阶段：将所有生成的视频片段拼接合并，并添加智能背景音乐，最终输出完整成片。
