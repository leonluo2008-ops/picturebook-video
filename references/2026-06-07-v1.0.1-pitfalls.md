## Pitfall #1 · 3.35s 朗读套 11s 中句档（收势页过度退场）

**触发场景**：C 子 agent 拿到 B 子 agent 量化数据，看到"3.35s 朗读 + 收势调性" → 凭印象选"中句档 7-10s" + 5 镜头节奏 `2-1-3-5` = 11s 视频

**反模式**：
- ❌ 把"收势页"理解为"必须长"= 加 4-5s 拉远退场
- ❌ 套"标版必跑"机械 5 镜头 = 凭空加 4s 空镜
- ❌ 把"末帧策略 4 档表"里"收势/退场 = 4-5s"当真

**用户原话（2026-06-07）**：
> "不管是收势，还是最后一个 clip，都应该遵循我们昨天晚上要求的设计原则，**先以旁白的阅读朗读时长，加上合适的呼吸时长，供观众消化就可以了**。clip 的视频明显时间太长了，只要按照之前的规范就可以，不用特别的增加。"

**修复**：
- 朗读 3.35s + 收势调性 = **短偏长档 6-7s**（不是中句档 7-10s）
- 节奏公式 `1-1-朗读-消化` = 6s 3 镜头
  - 镜头一（0-1s 场景建立）
  - 镜头二（1-2s 角色跃入）
  - 镜头三（2-5s 朗读）
  - 末帧消化 1s（不是 4-5s 拉远）
- **总时长 = 朗读 + 末帧消化 = 3.35s + 1.5s ≈ 6s**

**Pic2 实战对照**（v1.0.0 旧 vs v1.0.1 新）：

| 版本 | 节奏 | 镜头数 | 时长 | 用户反馈 |
|---|---|---|---|---|
| v1.0.0 | `2-1-3-5` 套中句档 | 5 | 11s | "明显时间太长了" |
| v1.0.1 | `1-1-朗读-消化` 走短偏长档 | 3 | 6s | （待用户 review）|

**根因**：
1. 节奏档位表里"3.5-5s 朗读 → 中句档 7-10s" + 末帧策略表"收势/退场 = 4-5s" 给 C 子 agent 暗示"收势页应该长"——**两表都过度乐观**
2. C 子 agent 没算"B 输出 3.35s + 末帧消化 1.5s = 6s" = 实际成本
3. 用户原话"先以旁白 + 合适的呼吸" = 总时长 = 朗读 + 1-2s 静默，不为"收势"加镜头

## Pitfall #2 · 末帧消化时间 = 朗读 × 0.3-0.6 倍（不是按调性档位硬套）

**触发场景**：C 子 agent 拿到 B 子 agent 量化数据，看到"标准词组 + 收势调性" → 选末帧 4-5s 拉远退场

**反模式**：
- ❌ 末帧消化 = 调性档位（"收势 = 4-5s"）
- ❌ 末帧消化 = 朗读 × 1-2 倍（**v1.0.0 旧表的错误建议**）
- ❌ 末帧消化 = 凭空加空镜（"让观众感受呼吸和温馨"）

**正确公式**：
- 末帧消化 = **朗读 × 0.3-0.6 倍**（标准词组）
- 标准词组末帧消化 **1-2s**（朗读完留 1-2s 让观众消化文字）
- 收势/退场末帧消化 **2s**（不是 4-5s 拉远退场）
- **末帧消化不强行加镜头一/二时长**（保持 1s+1s 不变）

**反模式原因**：
1. 旧档位表写"标准词组 × 1-2 倍" → 收势 = 6.7-13.4s = C 子 agent 套 11s 中句档
2. 末帧策略表写"收势/退场 4-5s 退场感" → C 子 agent 套 4-5s 拉远

**修复方向**：
- ✅ 末帧消化 = 朗读 × 0.3-0.6 倍
- ✅ 总时长 = 朗读 + 末帧消化（不为了"塞戏"加镜头一/二时长）

## Pitfall #3 · C 子 agent self_check 没拦住"套错档位"（v1.0.0 盲区 v2）

**触发场景**：C 子 agent 跑出 `rhythm_formula: "2-1-3-5"`, `total_duration_seconds: 11` → self_check `节奏_按_档位_选_不是_凭印象: true` 通过 → D 子 agent 跑出 11s 视频 → **用户判断太长**

**盲区**：
- C 子 agent self_check 9 项只查"文本合规"（@Image / 角色指代 / 末帧文字 / 节奏档位文字等）
- **不查**"本 Clip 节奏档位是否真的匹配旁白朗读时长"
- **不查**"末帧消化时间是否合理（1-2s vs 4-5s）"

**Pic2 实战**：C self_check 9/9 ✓ → D 跑出 11s → 用户看视频判断"太长" → 重发 C 改 6s

**修复方向**：
- C 子 agent self_check 加 `_实战校准_节奏档位_匹配朗读` 字段：必填 `narration_seconds` + `silence_seconds` + `total_duration_seconds` 三个数算 `total = narration + silence × 0.3-0.6` = 验算公式
- C 子 agent self_check 加 `_实战校准_末帧消化_不超朗读60%` 字段：必填 `末帧_duration <= narration × 0.6`
- D 子 agent 跑完 → 抽帧 → 验算 `总时长 vs B 子 agent 量化时长` 偏差 < 30%（避免凭印象拉长）
- 主 agent 收到 D 报告 → 看到 `总时长 > B 量化 × 1.5` → 报告回 C 改（不重跑 D）

**根因 v2**：
- v1.0.0 C/D 边界是"设计 vs 执行"，但 C 缺"对 B 输出（量化时长）的验算"
- v1.0.0 self_check 9 项没"匹配量化时长"项——C 凭印象套档位 self_check 也通过
- **反馈环 = 用户实战判断 → 手动重发 C**（已 work 但慢）
