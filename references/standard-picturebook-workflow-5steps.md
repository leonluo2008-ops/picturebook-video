# 绘本视频 · 标准工作流程 5 步速查

> **本文是 picturebook-video 当前**唯一权威**的"标准绘本视频工作流"**。下次任意绘本进来，按本文 5 步走即可。
>
> **用户元偏好**（铁律 #116）：绘本做完不增加 skill 版本号。skill 文字不出现绘本名（本文 = skill 通用方法论，**不是**某本绘本专属）。

---

## 0. 必先通读旁白 + 标叙事弧（铁律 #112）

**8 段旁白 → 不是 8 段 8 Clip · 也不是 6 段相邻拼 · 是先标 4 段叙事弧**：

| 段 | 叙事弧标签 | 关键判断 |
|---|---|---|
| **1** | **唤醒 + 观察** | 认知起点 · 主标 + 颜色 |
| **2** | **多感官感知** | 触觉 + 嗅觉 |
| **3** | **行动 + 反馈** | 行为闭环 |
| **4** | **情感收势** | 态度升华 |

**判断口诀**：**通读旁白 → 抓"谁对什么做了什么/感受到什么" → 标语义段 → 不按图分段**。

**⚠️ vision 必全 = 全部 N 张图必看光**（不是抽样 1/N/中/末）：
- 8 张图 = vision 8 次 · 1 张都别跳
- 每图必看 7 维：风格 / 主体 / 背景 / 文字 / **动作** / **场景跳变** / **主体对位**
- 合并前必建 image-inventory.md（7 列表 + 场景跳变列必填）
- **只识别部分图 = 凭印象拼 prompt = 翻车征兆**

## 1. 单 Clip 时长计算（铁律 #113）

| 段类型 | 朗读时长 | 静默 | 单 Clip 时长 |
|---|---|---|---|
| 极短档（≤2.5s 朗读）| < 1.5s | 0.5-1s | 4-5s |
| 短句档（2.5-3.5s 朗读）| 2-3s | 1-1.5s | 5-6s |
| 收势档 | 2.86s | 1.5-2.5s | 5-7s |

**总时长 = TTS 总时长 + 5-7s 余量**（不是 TTS + 10s+ 拖慢余量）

**v5 公式"末帧静默 ≥ 2s"是参考值不是红线**（铁律 #74 闭环 + 铁律 #113 强化）。紧凑节奏绘本按用户期望砍 50% 静默。

**⚠️ 硬上限 · 每 Clip 时长 ≤ 15s**（seedance duration 物理上限 · API 拒收 > 15s）：
- 写"X 时长 Clip"前**必查** seedance schema（`maximum=15`）
- 算出来 > 15s = **必拆**成 2 个 ≤ 15s Clip（按 image-inventory 场景跳变拆）
- 凭印象编数字 = 严重违规

**⚠️ 场景跳变 = 拆 Clip**（image-inventory 必建）：
- 同色系连续（水/陆/字卡）= 可合并
- 跨场景（水↔陆↔字卡跳变）= **必拆**
- 例：水+陆+水 = 拆 2 段（水陆合并 / 独立水），不强求 1 Clip

## 2. 范式选择（铁律 #110）

| 触发 | 范式 | seedance 参数 |
|---|---|---|
| **默认 · 任何绘本** | **v15 4 段导演分镜 + v6 文字持续可见段** | `--ref-images img1.jpg [img2.jpg ...]` |
| 领读型 + 顶部 1/6 文字 | **v6 5 段**（v15 + 段 5 文字全程可见） | 同上 |
| 用户**明确指定**首尾帧 | v7 2图=1Clip 合并 | `--image` + `--last-frame`（**默认不用**） |

**`@Image` 官方语法**（铁律 #114 · 必查）：

```
单图：   @Image1
多图：   @Image1 + @Image2 + @Image3    ← 中间必须有空格
错误：   @Image1+2+3                     ← seedance 错误语法
双重：   @Image@Image1                   ← fill 脚本 bug · 已修
```

**fill 脚本输出后必 grep 验证**：
```bash
grep -o "@Image[0-9 +]*" clips/clip*-prompt.txt | head
# 期望看到：@Image1 + @Image2 + @Image3（带空格）/ 单图 @Image1
# 看到：@Image1+2+3 或 @Image@Image → 错
```

## 3. 段 4 BGM 段 = B 档默认（铁律 #111 · 必走）

**绘本场景默认 = B 档**（无 BGM + 无朗读 + **显式允许并保留画面元素动作音效**）：

```
无任何背景音乐（no background music）·
无旁白人声/朗读（no voiceover/narration/singing）·
**允许并保留画面元素动作音效**（叮/咚/沙/沙沙等拟声 · 来自段 2 镜头描述 · 不是 BGM）·
不发完整朗读，TTS 后期对齐。
```

**3 档判定表**：

| 档 | 兜底句 | 适用 |
|---|---|---|
| **A 档**（全静音）| "无 BGM/无人声/无哼唱" | 纯 B-roll · **不是默认** |
| **B 档**（绘本音效）| "无 BGM + 显式允许音效" | **领读型绘本默认** |
| **C 档**（全音效）| "允许 BGM + 音效 + 朗读" | 社媒/漫剧/广告 |

**seedance 提交参数**（必带）：
- `--generate-audio true`（v1 用 false = 根本没音轨 = 翻车）
- `--watermark false`（绘本不加水印）

**反模式**：
- ❌ v1 A 档兜底句被 seedance 理解成"全静音"= 缺音效 = 翻车
- ❌ 加"无哼唱"= 禁拟声
- ❌ 加"画面元素动作音效保留 ... 等"= 双 B 档过激

## 4. fill_v15_template.py 输出后必查 3 件事

**填模板**：
```bash
python3 scripts/fill_v15_template.py --project-dir <project> --version v6 --tone "..."
```

**输出后必查**：

```bash
# 查 1：@Image 语法对位（带空格官方）
grep -o "@Image[0-9 +]*" clips/clip*-prompt.txt | head

# 查 2：段 4 B 档语言在（v6/v15）
grep "允许并保留画面元素动作音效" clips/clip*-prompt.txt

# 查 3：char_floats 动态生成（按 en_word 字母数 · 不是硬编码）
grep "字符顺序浮现" clips/clip1-prompt.txt
# 期望：M(0.3s) → a(0.6s) → n(0.9s) → g(1.2s) → o(1.5s) → 芒果(1.8s)
```

**任一不对** → 修 fill 脚本 + 重跑（**不**手动 patch prompt 字符串 = 污染）。

## 5. 端到端验证 gate（铁律 #79）

**单 Clip 跑通 → 必发飞书等用户目检 → OK 再批量**：

```
Clip 1（叙事弧第 1 段）
  ↓
seedance 提交（v15 范式 + B 档音效 + generate-audio true）
  ↓
等 succeeded + download + md5 校验
  ↓
send_message 飞书（完整证据链：md5 + task_id + 时长 + 视频本体）
  ↓
用户目检 → OK → 跑 Clip 2-N
       → 不 OK → 改 prompt 重跑该 Clip（不批量跳跑）
```

**反模式**：
- ❌ 1 个 Clip 跑通就批量跑剩下 8 段
- ❌ 抽帧自检（铁律 #87 · 视频交付不抽帧）
- ❌ 报告"vision 自检 N 帧" = 噪音

## 维护陷阱（绘本完成后必看）

> **核心元教训**：绘本做完 ≠ skill 变更。用户最强烈、最反复表达的偏好 —— **"不要做一个绘本，skill 里面就增加一个版本，绝对不要有这种东西"**。

| 陷阱 | 反模式 | 修复 |
|---|---|---|
| **1. skill 装绘本"版本号"** | 在 SKILL.md 加 `**#110**（v1.0.5+pic32 实战新增 · 绘本踩坑 ...）`，在铁律名里嵌入"绘本踩坑 · 用户根本性纠错"等 | 铁律以**通用方法论**命名（不绑绘本名）。绘本信息只放在 work/<日期-绘本名>/ 业务笔记里。**反问自检**："这个铁律是因为某本绘本踩坑，还是因为通用方法论？" → 仅后者入 skill |
| **2. 开新 git 分支做"实验"** | 每本绘本 = 新分支（`xxx-4clip-director-v1.0.5+picYY` 等）= 10+ 个分支污染仓库 | 永远**沿用现有 fix/feature 分支累加 commit**。绘本产物放 work/，不进 skill 仓 |
| **3. 1 个 commit = 1 个完整变更单元** | 在 fix 分支上 N 个半成品 commit = 不清晰 | 本轮工作（铁律 + fill 脚本 bug 修复 + 章节沉淀）= **1 个 commit**（详细 message 列出铁律 + bug + 章节） |
| **4. 子 agent 累积未提交修改污染** | 多个子 agent 在 fix 分支上改了一堆文件但**没 commit** = `git status` 满屏 `M` / `??` = 下次开工混淆 | 本轮工作开始前**必先 `git status`** 看当前未提交内容。要么 commit 掉，要么 stash 走（不要让孤儿改动一直挂在那里） |

### 用户耐心临界信号

- **情绪升级词**："我有点看不懂" / "现在版本太多了" / "做 1 个完整提交" / "绝对不要这种东西" = **立刻停下来不解释 + 不开问卷 + 直接修**
- **3 次纠错后必升级为元教训入 skill**（触达 2 次 = 铁律 #110-#116 + 维护陷阱 4 条）
- **修复 SOP**：① 首错即拍板（不"先算错→再问"两步走，元问题一次性拿主意） ② 不重复犯同根因错（第 1 次错→第 2 次决策前自检"是不是又在算同类元问题"） ③ 情绪词已出 = 不解释+修+报完成+不报过程
