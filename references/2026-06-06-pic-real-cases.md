## 1. 起点：v1 prompt（错版）

我最初按 v15 标版写 prompt，**`@图1` / `@图2`（中文"图"）+ 多余隔离句"其他元素不出现"**：

```
分镜绑定：@图1 作为唯一参考帧（封面：...），同时延续到 @图2（...）；
镜头一（0-2s · 场景建立）：@图1 的小兔子@图1 站在金色阳光草地上
...
"图片中的小鸭子、池塘、狐狸、其他动物等元素不出现"
```

**用户提出问题**："clip 出现了完全重复的两轮视频，请你抽帧识别一下，并检查提示词中，到底是哪里有问题？"

## 2. 抽帧 10 帧发现真相

帧序列显示**画面重复 2 轮**：

| 帧 | 实际画面 | 我 prompt 期望 | 偏差 |
|---|---|---|---|
| 1-2s | 封面兔子**张开双臂**（=图 1 原样）| 镜头一建立 | ✅ |
| 3-4s | **窗前兔子背对**（=图 2 原样）| 镜头二跃入 | ✅ 切到了 |
| 5-7s | 回到封面 + 兔子**挥手** | 镜头三挥手 | ✅（晚了 1s）|
| 8-10s | 切回窗前 + 兔子**举手向太阳** | 镜头四 sun | ✅ |

**结论**：模型把封面+窗景当**4 个独立短镜头**硬拼到一起，**没有过渡**，且顺序跟 prompt 不完全对齐。

## 3. 我第一轮根因诊断（**部分错**）

我以为是"v15 标版不适用本场景"——提出"一图一 Clip / 降级为单图思维 / 改动作描述用局部动效"等方案。

**用户纠错**（铁律级别）：
> "我跟你说一个东西，必须使用图一图二的时候，必须使用"@"。具体如何使用请你查看官方的文档，这里面是肯定有的，你不能乱改"

## 4. 查官方文档：确认 `@图N` 是错的

查 seedance2.0-tool `references/director-mode.md` §@-Tags 精确语法：

> **上传文件后，系统自动命名 `@Image1, @Image2, @Video1` 等**。
> 在 prompt 中用自然语言为每个文件分配角色。

官方文档**只有 `@Image1` / `@Image2` / `@Video1` / `@Audio1`**——**`@图N` 是中文别名，模型不识别**。

**这就是 v1 翻车的根因**——分镜绑定整段被模型忽略，**画面自由发挥**。

## 5. 我第二轮修复（**也部分错**）

我写了 v13 范式 prompt：

```
将图片1中的奶白色小兔子定义为主体小兔子。
将图片2中的奶白色小兔子定义为同一个主体小兔子。
将图片1中金黄色阳光草地...定义为场景A。
将图片2中室内暖橙色房间...定义为场景B。
```

**用户纠错**：
> "你使用V13的方式是不正确的，不能完全使用V13的方式。你只能取V13中我们可以借鉴的部分，实际上V15系列除了刚才你犯的错，其他的都非常好。你要取长补短来改prompt"

**取长补短原则**：
- ✅ v15 留：4 段骨架 / 4 镜头 2-1-3-3 / 拟声+动作一对一 / 末帧 = 朗读+静默 / 双重禁 BGM / 角色指代不用名字
- ✅ v13 借：`@Image1` / `@Image2` 显式绑定
- ❌ 不抄 v13：单独"主体定义段" / `[0-3s]` 时间轴分段 / "参考音频1的音色" / "参考视频1的运镜"（绘本本批无视频/音频参考）

## 6. 我第三轮修正（**还部分错**）

我又自作主张在 prompt 里加了一行"**图片中的小鸭子、池塘、狐狸、其他动物等元素不出现**"——这其实没必要（@Image1/2 都没那些东西），还成了"伪铁律"。

**用户纠错**：
> "你刚刚说的不增加小鸭子、池塘、狐狸等其他原元素，不是铁律，根据故事剧情的需要你可以增加"

→ **元素增/减由剧情决定**，**不写隔离句**。

## 7. 我第四轮修正（**还部分错**——自作主张对齐子代）

我看到 v2 prompt 后给镜头绑"对齐子代"——但**v15 SKILL.md 自己段 2 约束就写"@图N 必含"**（**中文"图"**），这本身就是 SKILL.md 的错。

**用户纠错**（要求"不要自作主张去调整参考图的纸袋"）：
> "你不要自作主张去调整参考图的纸袋，比如clip一的镜头一、镜头二，里面所使用的参考图image一、image二已经处理好角色一致性了，不要随意去错开子代，请修正"

→ **每个镜头里 `主角@ImageN` 的 N 必须等于该镜头对应的图**（同图连续 → 全部 @Image1；切图 → @Image2），**主语不乱用**。
