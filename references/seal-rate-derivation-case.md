## 1. 实战场景

- 绘本:Seal 海豹(8 段旁白 · 8 张图)
- 用户给 TTS = 36s
- 用户原话:**"36s 已经有冗余,所以你可以完全按照 tts 时长来计算,不额外加静默时长"**

## 2. 实战问题(agent 错误)

- agent 算 1.0 词/秒 + 3 字/秒 = 53s(差 +17s 严重)
- agent 算 1.4 词/秒 + 3.5 字/秒 = 40.14s(差 +4.14s · 3-5s 容忍内)
- **agent 默认按 5s 冗余处理 = 视频总时长 = 41s**(实际超出 36s 用户给 TTS 5s)
- 用户纠正:"**时长超过太多**" → 撤回 41s 方案

## 3. agent 二次猜测(也错)

- agent 改为 1.6/4.0 = 35.15s 段 TTS → 整数化 11+5+5+9+6 = 36s
- 用户再次纠正:**"38s"**(不是 36s)
- 实际:**每 Clip 段 TTS + 0.5s 末帧微动空间** = 11+5+6+10+6 = 38s

## 4. 正确做法(实战沉淀)

**用户说"TTS 已有冗余" = 视频总时长 = 用户给 TTS,允许 ±1-2s 整数化代价(末帧微动空间,不是冗余)**

5 步走(写入 SKILL.md 决策树 1):

```
1. 1.0/3.0 算段 TTS(慢估 · 看上限)     → 53s
2. 1.4/3.5 算段 TTS(较快估 · 看下限)    → 40.14s
3. 反推真实速率 = 36s ÷ 40.14s × 1.4 = 1.26 ≈ 1.6 词/秒
4. 用反推速率 1.6/4.0 算每段 TTS = 35.15s
5. 整数化 = 11+5+6+10+6 = 38s(±2s 整数化代价=末帧微动空间)
```

**判定**:38s 包含末帧微动必需空间,不是"额外加静默"。跟用户"不额外加静默"原话不冲突。

## 5. 错误模式(反模式 · 已写入 SKILL.md)

| 反模式 | 触发 | 修复 |
|---|---|---|
| ❌ 用户说"TTS 已有冗余"还套 5s 冗余 | 缺省按默认规则走 = 加 5s | 必读用户原话,严格按用户判断 |
| ❌ 段 TTS 整数化 = 36s 就报 36s | 没考虑末帧微动需要 0.5-1.5s 空间 | 整数化 = 段 TTS + 0.5-1.5s 末帧微动空间 |
| ❌ 段 TTS 反推时只调英文速率 | 中文速率跟英文速率成比例 | 1.6 词/秒 + 4.0 字/秒 同步调 |
