## 4. 本轮 4 个关键反馈 → 修复的对应关系

### 反馈 1 · 信息密度太高
- **症状**：12s 装 10.25s 朗读 + 0.5s 收势，几乎没留白
- **修复**：v15.1 节奏公式"2-1-3-3-3"——每词 1s 朗读 + 2-3s 静默停留（25% 信息密度）
- **对应铁律**：#36 末帧铁律 / #37 语义块拆分

### 反馈 2 · 末帧没停留
- **症状**："朗读完画面就结束，没有呼吸"
- **修复**：铁律 #36——末帧 = 朗读 1s + 静默 2-3s，**不是 0.5s 收势**，**"画面定格"是反模式标签**应删除
- **关键代码片段**：
  ```
  镜头五（9-12s · 朗读+静默）：...
  （朗读 1s + 静默 2s 停留，镜头停在画面上让听众消化单词，
   画面最终定格在末态，停留至最后一帧）
  ```

### 反馈 3 · 漏图连环错（漏图 2 + 漏图 8 + 缺 4b）
- **症状**：B 方案表只列到 4b 没列 clip5 = 漏收势图；v15.1 建议拆 4a+4b 但实跑只跑了 4a
- **修复**：铁律 #38 切分表必查 4 项（30 秒自检）
- **根因**：切分表没列全（少 3 张图）+ 跑前没对齐

### 反馈 4 · 老鼠一闪而过
- **症状**：clip3 10s 装 7.65s 朗读 + 鼠戏，rat 镜头被截短
- **修复**：clip3 拉长 12s 修（10s 装不下）
- **关键决策**：旁白时长 + 镜头动作时长 = 总时长上限，**不能挤掉收势**
