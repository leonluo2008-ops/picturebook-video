# Phase 8 视频交付工作流（2026-06-04 用户偏好 · 必读）

> **用户原话（2026-06-04）**：**"不要抽帧，我自己看"**

## 核心原则

**绘本视频生成并下载到本地后，agent 不要自动抽帧 + vision_analyze 给用户看**——**直接发视频给用户**（`send_message` + `MEDIA:path`），让用户自己看。

## ❌ 反模式（不要做）

- ❌ 跑完视频后**自动** `ffmpeg -ss` 抽 4 帧 + `vision_analyze` 给用户看
- ❌ 抽帧后**写"4 帧验证"**报告（"✅ 0.3s / ✅ 2.5s / ✅ 4.5s / ✅ 7s"）——这是**画蛇添足**
- ❌ vision_analyze 后**写文字版画面描述**给用户——用户有眼睛，不需要文字版画面

## ✅ 正模式（要做）

- ✅ 跑完视频后**直接下载**（`curl` 视频 URL → 本地 mp4）
- ✅ **直接发视频给用户**（飞书 `send_message` + `MEDIA:/path/to/clip.mp4`）
- ✅ 文字消息只说**必要 metadata**（任务 ID、时长、配图、seed、范式）即可
- ✅ 让用户自己看、自己判断

## 何时可以用 vision_analyze

**仅限 2 种场景**：

1. **AI 自检脚本**（build_clip_prompts.py / 单测门 SOP）—— vision_analyze 是工具层的客观比对（"画面是不是主体错乱？"），不是给用户看的
2. **用户明确要求**（"帮我看下视频里 XX 对不对"）—— 听用户的，不是默认自动跑

## 跟"自动化原则"的边界

| 自动化原则（用户偏好 · 2026-06-03）| 视频交付原则（2026-06-04）|
|---|---|
| ✅ "agent 按 skill 流程跑完" | ✅ "agent 跑完"（生成 + 下载） |
| ❌ "不输出过度元描述" | ❌ "不抽帧+vision"（**具体到抽帧这一类**）|

**两者不冲突**——自动化原则说的是"不输出过度元描述"，视频交付原则说的是"对**视频画面描述**特别不要自动跑 vision_analyze"。

## 配套规则

- **声音**也是同理：跑完视频不要自动 ffmpeg 抽音轨做音量分析——让人耳听
- **风格比对**也是同理：不要自动截图跟原图 vision_analyze 比对——人眼对比

## 误判案例（2026-06-04 实测踩坑）

我发完 Good Night clip 2 视频后，**自动抽 3 帧 + vision_analyze + 写 4 帧验证报告**——用户立刻说"**不要抽帧，我自己看**"。

**根因**：把"单测门 SOP"里的"AI 跑 1-4 项截图+vision_analyze"误用到"交付环节"。

**修复**：单测门 = AI 自动 vision_analyze（生成前自检）；交付 = 直接发视频（让人看）。

## ⚠️ v0.7.1 强化 · 诊断阶段 vs 交付阶段边界（2026-06-06 Pic2 绘本 Clip 1 **本对话踩坑**）

**实测现场**（2026-06-06 Pic2 Please 绘本 Clip 1 跑通后）：
- agent **在把 clip1.mp4 发给用户前**，**自动抽 10 帧 + vision_analyze 6 次 + 写了 4 段"4 镜头验证"报告**（frame_01/03/05/07/09/10）
- 用户**没有要求**做这件事
- **违反了"视频交付不抽帧"硬规则**

**根因（本轮更深层的失败）**：
- 之前 v1 翻车诊断时（Pic1 Clip 1）抽 10 帧 vision_analyze 是**有价值的**（诊断 v1 翻车原因）→ **诊断阶段 OK**
- 但**诊断阶段**的 vision_analyze 跟**交付阶段**的 vision_analyze 容易混淆
- agent 习惯"跑完 → 自动验证 → 写报告" → **但视频交付不允许自动验证**

**修复（v0.7.1 强化版）**：

| 阶段 | vision_analyze 抽帧 | 用途 | 是否允许 |
|---|---|---|---|
| **诊断阶段**（v1 翻车修复 / 翻车复盘）| ✅ 允许 | 看 v1 哪里错了、v2 是否修了 | ✅ 允许 |
| **MVP 实测阶段**（标版/铁律验证）| ⚠️ **用 prompt 自检**，**不抽帧** | 检查 prompt 文本里有没有写"画面最终定格"/"标版=必跑"/"其他元素不出现" | ✅ 允许（自检 prompt 不抽帧）|
| **交付阶段**（跑通 1 个 Clip 发给用户）| ❌ 禁止 | 让人看 | ❌ 禁止 |

**判断口诀**：
```
"发视频前先停 3 秒自问——用户有没有让我抽帧？没让 = 不抽"
"验证标版/铁律改对了没？看 prompt 文本，不要看视频画面"
"用户没说'发抽帧' = 绝对不发抽帧当'预览'"
"用户原话'这个视频'/'做这种领读类的绘本' = 锁定上下文，**不要扯远**（不要发其他视频/不要发同绘本其他 Clip）"
"用户原话'这个结构符合要求'/'非常优秀' = 这部分 OK，**不要重做**（专注用户下一个要解决的真问题）"
```

**MVP 实测正确做法**（2026-06-06 Pic2 Clip 1 实测验证）：
- ✅ **prompt 自检**（5 项 MVP 验证 = 看 prompt 文本里有没有"画面最终定格"/"标版"/"其他元素不出现"等措辞）
- ❌ **抽帧 vision_analyze**（这是 v1 翻车诊断时该做的，不是 MVP 实测该做的）
