# v8.1 动作驱动 Prompt 模板

> **v5.0.10 核心改动** — 替换 v5.0.9 的"运镜模板"。问题：v5.0.9 prompt 让画面"几乎静止"，主体本身没动。
> **v5.0.10.1 核心修正** — Potato 实测发现 v5.0.10 强时间锁（`00:00.0-00:01.0`）违反 Seedance 官方"模型对精确时间不稳定"原则。改为"按事件分镜"（镜头 1/2/3），时间戳为可选项。

---

## v5.0.9 旧模板 vs v8.1 新模板对比

### 旧 v8（运镜思维 · 4 段）

```
段1 主体: "一颗绿色生菜，中景"
段2 动作: "镜头缓慢推近"
段3 声音: "旁白: ..."
段4 收尾: "镜头持续循环不切断"
```

**结果**：镜头在动但生菜本身是死的 = 画面静止感。

### 新 v8.1（动作思维 · 4 段 · v5.0.10.1 按事件分镜修正）

```
段1 主体+视觉基底:
  "一颗绿色生菜，褶皱叶片铺满画面，中景（参考图 @Image1）"

段2 主体动作（按事件分镜 · 不卡秒数 · v5.0.10.1 修正）:
  "镜头 1（开场）：生菜居中，叶片边缘有微风轻摆，叶脉上光斑缓慢移动
   镜头 2：一只小手从画面右上伸入，捏住最外层叶子轻微摇晃后松开
   镜头 3（特写）：光斑聚焦到生菜中心，叶片表面水珠缓慢滑落"

段3 关键节点对照（不写精确时间戳 · v5.0.10.1 修正）:
  "念到'生菜'时镜头特写聚焦叶片
   念到'Lettuce'时长略长，画面慢速拉远展示整体"

段4 收尾（不凝固 · v5.0.10 不变）:
  "镜头 4（收尾）：回到开场状态，叶片继续微风轻摆 + 光斑缓慢移动"
```

**关键差异 (v5.0.10.1)**：
1. **按事件分镜** — 镜头 1 / 镜头 2 / 镜头 3，不写 `00:00.0-00:01.0` 强时间锁
2. **主体本身动** — 叶片摆/光斑跳/手伸入/水珠落，不只镜头动
3. **时间戳不强制** — 跟 Seedance 官方"模型对精确时间不稳定"对齐 (提示词指南 §2)
4. **段 3 改"事件对应"** — 写"念到 X 词时 Y 动作"，不写 `旁白 00:00.066-...` 精确锚点
5. **收尾不凝固** — 保持运动状态过渡

## v8.1 模板结构（4 段骨架 · 动作版）

### 段 1 · 主体 + 视觉基底（1-2 句）

**必含**：
- 主体（来自参考图 vision 识别）
- 景别（全身/中景/半身/特写）
- `@ImageN`（参考图锚定）
- 关键视觉特征（颜色/形状/姿态）

**文字元素动效分流（v5.0.13+ · 2026-06-26 Luffa 用户偏好）**:
- 参考图含文字（领读型几乎每张都有目标词文字）→ 视觉基底里**显式列出**文字 + 动效关键词
- 用户没要求文字动 → 保持原写法"保留参考图中的 X 文字作为画面元素"
- **6 种安全动效关键词**（Luffa 3-clip 实测全 ok=true）:
  - `轻微弹跳` — 字母逐个上下蹦
  - `轻微摇摆` — 文字跟主体一起轻晃
  - `轻微闪烁光效` — 文字颜色微微变化
  - `缩放呼吸` — 文字大小缓慢呼吸
  - `轻微摇曳` — 文字左右摆动
  - `波浪浮动` — 数字/文字带波浪感
- ❌ 反模式：`打字机效果` / `逐字缩放` / `字母依次出现` 等复杂时序 → seedance 易跑飞成"文字乱码"

**反模式**：
- ❌ "一颗漂亮的生菜"（形容词无信息）
- ❌ "中景" 单独出现（必须配合视觉特征）

### 段 2 · 主体动作序列（核心 · 4-6 行 · v5.0.10.1 按事件分镜）

**必含 (v5.0.10.1 修正)**：
- **按事件分镜**描述主体动作（`镜头 1：... / 镜头 2：... / 镜头 3：...`）
- 主体本身的动作（不是镜头动）
- **不卡秒数** — 不写 `00:00.0-00:01.0` 这种强时间锁（官方"不强制限制每段时长"）
- 动作要"具体可执行"（seedance 能渲染）
- 可选：句末加"约 X 秒"近似值（参考用）

**动作来源推导**（v5.0.10 新方法论）：
```
问自己 3 个问题：
  1. 参考图里的主体"能做什么"？（生菜: 摆动/被摘/被洗/被切）
  2. 旁白这段在"说什么动作"？（"摘生菜"= 主体动作就是"被摘"）
  3. 时间段里"有没有停顿"？（SRT pause_after >= 0.8s = 画面可以呼吸/切换）
```

**反模式**：
- ❌ "镜头持续 5s 循环"（无具体动作）
- ❌ "画面自然流动"（无信息）
- ❌ "保持稳定"（凝固语）
- ❌ `00:00.0-00:01.0 主体居中` 强时间锁（官方"模型对精确时间不稳定"）

### 段 3 · 关键节点对照（v5.0.10.1 修正）

**必含 (v5.0.10.1 修正)**：
- **不写精确 SRT 时间戳**（如 `00:00.066-00:01.800`）— 跟官方"不强制限制每段时长"对齐
- 写"念到 X 词时 Y 动作"的事件对应（如"念到'生菜'时镜头特写聚焦叶片"）
- 关键动作 vs 旁白关键词的对应（不写秒数，只写事件呼应）
- 可选：附 SRT 时间参考（用 `约 X 秒处` 近似值）供 agent 内部对照

### 段 4 · 收尾（不凝固 · v5.0.10 核心 · v5.0.10.1 按事件分镜修正）

**v5.0.9 旧收尾**：
- "镜头持续循环不切断" → 让画面趋向凝固

**v8.1 新收尾 (v5.0.10.1 修正)**：
- 用 `镜头 N（收尾）：...` 按事件分镜，不写 `00:0X.0-00:0Y.0` 强时间锁
- **必须保持运动状态**（哪怕微动）
- 跟起始状态呼应（"回到 X 状态"）

**禁止的收尾语**（verify_prompt.py R3 硬规则，regex `持续.*?循环`）：
- ❌ "持续循环" / "持续不切断"
- ❌ "保持微动状态循环过渡"（⚠️ 隐蔽变体：不含"持续"但仍命中 `.*循环`，firefighter 实测踩坑）
- ❌ 任何含"循环"的收尾语（R3 regex 覆盖所有"循环"出现）
- ❌ "保持稳定" / "保持静止"
- ❌ "自然流动" / "有运动感"（空话）

**正确收尾示例 (v5.0.10.1 修正版)**：
- ✅ "镜头 4（收尾）：回到开场状态，叶片继续微风轻摆 + 光斑缓慢移动"
- ✅ "镜头 4（收尾）：水珠继续滑落到叶尖 + 镜头微摇，自然过渡到下一段"

## 制作流程整合（v5.0.10 完整路径）

```
1. 接收 SRT 文件（用户从剪映导出）
2. python3 scripts/srt_parser.py <srt> → timeline.json
3. python3 scripts/clip_merger.py <timeline.json> → clips.json
4. 对每个 clip，按 v8.1 模板手写 4 段 prompt
5. python3 scripts/verify_prompt.py <prompt_dir> → 8+3=11 项硬规则检查
6. delegate agents/prompt-reviewer/ → L3 视觉逻辑审查
7. 通过 → Step 6 seedance 提交
```

## 与 v5.0.9 的兼容性

- v5.0.9 没 SRT 也能跑（估算路径保留为 fallback）
- v5.0.10 优先 SRT 路径 → 时间精度从估算秒级提升到真实毫秒级
- v8.1 模板与 v5.0.9 兼容（同样 4 段结构），但段 2 / 段 4 必须按新写法

## 反模式速查（v5.0.10.1 修正）

| # | 反模式 | 后果 | 修复 |
|---|---|---|---|
| R1 | 段 2 只写"运镜"无主体动作 | 画面静止 | 至少 3 个事件分镜写主体动作 (镜头 1/2/3) |
| R2 | 段 2 用"持续 X 秒"含糊 | 时间锚点丢失 | 改为"镜头 N：...动作"按事件分镜 |
| R3 | 段 4 用"持续循环/保持稳定"等含"循环"的语 | 画面凝固(regex `持续.*?循环`) | 改为"镜头 N（收尾）：回到开场状态+具体微动+自然过渡"（⚠️任何含"循环"的收尾都会被R3捕获） |
| R4 | 段 3 强写 `MM:SS.mmm` 精确时间戳 | 官方反模式(模型对精确时间不稳定) | 改"念到 X 词时 Y 动作"事件对应 (WARN) |
| R5 | 段 1 缺 @ImageN | 主体跟参考图脱钩 | 必含 `@ImageN` |
