# seedance prompt 翻车 → 系统化修复方法论（v1.0.0 · 2026-06-10 Pic10 Hamster v2→v3 实战沉淀）

> **核心问题**：seedance 生成的视频出现**非预期的视觉/听觉**（莫名说话声 / 画面死板 / 故事性 = 0 / 镜头呆板），怎么**系统化定位根因 + 修复 prompt**？
>
> **来源**：Pic10 Hamster v1→v2→v3 端到端实战 + Rabbit newclip1 v1→v2 + Pic8 Rabbit Clip4 v1→v2 翻车链沉淀
>
> **本文件不是**：seedance 操作手册 / 翻车案例合集（**是**翻车后**怎么修**的方法论）

---

## 0. 核心方法论（5 步系统化修复路径）

### Step 1 · **症状分类**（先看清翻的是什么）

把症状分到 4 个**互相独立**的类别，每类的诊断方法不同：

| # | 症状类别 | 典型表现 | 影响维度 |
|---|---|---|---|
| A | **画面/镜头问题** | 镜头死板 / 故事性 = 0 / 镜头太碎 / 末帧"微动"不动 / 单图整段推镜头 | 视觉 |
| B | **音频问题** | 莫名说话声 / 莫名音效 / 完全静默 / 双重否定失声 | 听觉 |
| C | **角色/风格漂移** | 主角变形 / 风格漂移 / 出现不存在的角色 | 视觉+一致性 |
| D | **时长/节奏问题** | 实测时长 < 设计时长 / 镜头被压缩 / 末帧被吞 | 节奏 |

> **关键原则**：**先分清症状类别**——A 和 B 的修复方向**完全不同**（A 改"4 步原则"，B 改"末尾约束段 4 词"）。混在一起看 = 修错地方 = 越改越烂。

---

### Step 2 · **定位根因**（按症状类别走对应诊断树）

#### A 类（画面/镜头）诊断树

```
画面/镜头问题
├─ 镜头死板 = 故事性 = 0？
│  └─ ✅ 套了"建立/单词/收势"3 镜固定模板（= 镜头凑数）
│     → 修复：看参考图实际画面 → 按动作步骤数定 1-3 镜（不固定 3 镜）
│  └─ ❌ 单图 + 整段"镜头缓慢平稳推进"（v6 简化版翻车）
│     → 修复：走 v15 6 段骨架 + 多镜头时间线（3 镜以上）
│
├─ 末帧"微动"实际不动？
│  └─ ✅ 末帧写了"画面继续微动"（v15 旧版措辞）
│     → 修复：写"画面不再动 · 整段留出静默 · 让 XX 在小朋友心中沉淀"（真正静默）
│
├─ 镜头太碎糊？
│  └─ ✅ 5s 5 镜头 = 每镜头 1s（Rabbit 旧版翻车）
│     → 修复：总镜头数 ≤ 时长 ÷ 1.5s（每镜头至少 1.5s）
│
└─ 凭印象写 prompt，没看参考图？
   └─ ✅ 不看参考图就套"喊出来"模板（Hamster Clip 1 v1 翻车）
      → 修复：Step 1 必须 vision_analyze 参考图（看 p1 实际画面再写）
```

#### B 类（音频）诊断树（**Pic10 v2→v3 实战沉淀**⭐）

```
音频问题
├─ 莫名说话声 / 莫名人声（非核心词领读）？
│  └─ ✅ prompt 里有"TTS 音轨占位 / 音轨占位 / 朗读时长"等元数据术语（Pic10 v2 翻车根因）
│     → 修复：**完全删除**元数据术语 → 用官方原话"无人声、无歌唱、无配音、无朗读"
│  └─ ✅ 音效写得太情绪化（"撒娇/满足/紧张"等）
│     → 修复：去掉情绪词 → 写客观音效（"一声短促的吱声"而非"像小宠物的撒娇"）
│  └─ ✅ 末尾约束段写得太口语化（"无旁白人声 / 无哼唱"等）
│     → 修复：用官方 doc2 line534 末尾约束段原话"无人声、无歌唱、无配音、无朗读"
│
├─ 完全静默（无声）？
│  └─ ✅ 双重否定"无任何背景音乐" + 又写 BGM 描述（Pic8 v1 翻车）
│     → 修复：只写"保持无字幕、无水印、无 Logo，无人声、无歌唱、无配音、无朗读"一句
│
└─ 音效无效 / 音效与画面不同步？
   └─ ✅ 音效没写 `<...>` 符号（官方 doc2 §440-451 特殊字符规范）
      → 修复：所有音效用 `<>` 包裹，音效**内联**到每个镜头（不集中末尾）
```

#### C 类（角色/风格）诊断树

```
角色/风格漂移
├─ 主角变形 / 主角颜色变了？
│  └─ ✅ 没用主体命名标签（"它/小兔子"模糊指代）
│     → 修复：主体定义段写"将图片1中的 X 定义为<主角名>"，每镜头都用<主角名>引用
│
├─ 风格漂移（绘本→写实 / 2D→3D）？
│  └─ ✅ 风格定义散落在多段，模型漂移
│     → 修复：段 1 素材角色声明一次性说清（"children's picture book 2D paper-cut collage style, soft pastel palette of..."）
│
└─ 出现不存在的角色？
   └─ ✅ 参考图有 2 个角色，prompt 没区分（官方 doc2 §8.2 双胞胎坑）
      → 修复：给每个角色命名（<主角A> / <配角B>）+ 明确动作归属
```

#### D 类（时长/节奏）诊断树

```
时长/节奏问题
├─ 实测时长 < 设计时长？
│  └─ ✅ 整数档选错（5/6/7s · seedance ceiling 7.04s）
│     → 修复：实测时长规律 = 设计时长 + 0.04s ceiling
│
├─ 末帧被吞 / 镜头被压缩？
│  └─ ✅ 镜头太多（5s 5 镜头 = 每镜头 1s）
│     → 修复：总镜头数 ≤ 时长 ÷ 1.5s
│
└─ 节奏太快/太慢？
   └─ ✅ 没看参考图实际动作复杂度
      → 修复：看图推断动作步骤数 → 按动作复杂度定 5/6/7s
```

### Step 3 · **修复方向对照**（根因 → 改哪里）

| 症状 | 修复位置 | 修复措辞（原话）|
|---|---|---|
| 镜头死板 | §1.1 主体定义段 + §1.3 多镜头时间线段 | 按 4 步原则重写（看图→推断→写动作→留静默）|
| 末帧微动不动 | §1.3 多镜头时间线段 · 末帧镜头 | "画面不再动 · 整段留出静默 · 让 XX 在小朋友心中沉淀" |
| 莫名说话声 | §1.4 整段音频约束段 | **完全删除** "TTS 音轨占位 / 音轨占位 / 朗读时长"，改用官方原话 "无人声、无歌唱、无配音、无朗读" |
| 音效情绪化 | §1.3 多镜头时间线段 · 每个镜头内联音效 | 去掉 "撒娇/满足/紧张" 等情绪词，写客观音效（"一声短促的吱声"）|
| 主角变形 | §1.2 主体定义段 | "将图片1中的 X 定义为<主角名>" + 后续每镜头用 <主角名> 引用 |
| 风格漂移 | §1.1 素材角色声明段 | 一次性说清风格（"children's picture book 2D paper-cut collage style, soft pastel palette of..."）|
| 完全静默 | §1.4 整段音频约束段 | 删除所有 BGM 描述 + 双重否定，只留一句官方原话约束 |
| 镜头太碎 | §1.3 多镜头时间线段 | 总镜头数 ≤ 时长 ÷ 1.5s（每镜头至少 1.5s）|

### Step 4 · **修复后必须重跑**（Pic10 v3 验证）

- **单 Clip 端到端先验**（铁律 #65 + #79）：修复后**先跑 1 个 Clip**，目检通过再批量跑剩下
- **目检清单**：
  - A 类 → 抽帧看 6 帧，看镜头是否有故事感
  - B 类 → 听音频，**反复听**是否还有莫名说话声（Pic10 v3 验证关键：用户反复听了 3 遍确认修复）
  - C 类 → 抽帧对比参考图，主角五官/颜色/风格是否一致
  - D 类 → 看实测时长 vs 设计时长，确认每镜头 ≥ 1.5s
- **修复有效** → 用同款修复方案批量应用到所有未跑 Clip
- **修复无效** → 回 Step 2 重新诊断（**不要**继续在错的方向上叠加修复）

---

### Step 5 · **同步沉淀到 skill**（避免下次再踩）

修复验证后**必须**：
1. 把修复方案**反推**到 v15director 规范（§1.4 / §1.3 对应章节）
2. 在 SKILL.md 铁律区**加铁律**（Pic10 修复 = 铁律 #107）
3. 在 v15director.md §5 / §7.3 反模式库**加新反模式条目**（Pic10 v2 = "写 TTS 音轨占位"反模式）
4. **本文件**（seedance-prompt-翻车修复方法论）也加对应条目到 Step 2 诊断树

> **反模式**：修完不沉淀 = 下次再踩 = 用户 3 轮纠错 = 浪费时间
> **铁律 #76 关联**：发飞书视频附件 = 必附完整证据链（md5 + task_id + seed + 时长 + 修复 diff）

## 1. Pic10 Hamster v2→v3 实战案例（⭐ 必看 · 用户目检修复通过）

### v2 翻车现场

- **症状**：clip2 v2 视频出现"莫名说话声"（**不是** hamster 等核心领读词，是奇怪的非核心词声音）
- **用户原话**："这视频里有莫名的说话声"（不是 hamster 也不是旁白词）

### v2 prompt 问题点（4 个错）

1. ❌ **元数据术语触发人声**："TTS 音轨占位，时长匹配旁白朗读时长（2 秒）"
2. ❌ **口语化约束失声**："无背景音乐、无旁白人声、无哼唱、无歌唱"（"无旁白人声"被模型理解为"无其他声音 = 留出空间给我生成人声"）
3. ❌ **音效情绪化**："像小宠物的撒娇" → seedance 自我补全人声填补情绪
4. ❌ **音效与画面描述混在一起**："画面保持无字幕..."（约束段被分散到多处 = 模型理解混乱）

### v3 修复方案（diff 全文）

```diff
- 全程无背景音乐、无旁白人声、无哼唱、无歌唱。保留 TTS 音轨占位，时长匹配旁白朗读时长（2 秒）。
- 画面保持无字幕、无水印、无 Logo。镜头全程严格遵守"1 镜头 1 种运镜"红线（不堆叠推拉摇移）。
+ 保持无字幕、无水印、无 Logo，无人声、无歌唱、无配音、无朗读。
+ 镜头全程严格遵守"1 镜头 1 种运镜"红线（不堆叠推拉摇移）。
```

```diff
- 镜头 1（...）：... <一声轻柔的微风>。
+ 镜头 1（...）：... <一声轻柔的微风声>。
- 镜头 2（...）：... <一声细软的"吱"叫，像小宠物的撒娇>。
+ 镜头 2（...）：... <一声短促的吱声>。
```

### v3 修复原理（4 步诊断 → 1 步修复）

| 步骤 | 动作 | 命中 v15director 规范 |
|---|---|---|
| Step 1 症状分类 | "莫名说话声" = B 类音频问题 | — |
| Step 2 定位根因 | prompt 里有"TTS 音轨占位 / 音轨占位 / 朗读时长"（元数据术语）| B 类诊断树第 1 条 |
| Step 3 修复方向 | §1.4 整段音频约束段 · 完全删除元数据术语 + 用官方原话 | Step 3 对照表第 2 行 |
| Step 4 验证 | 跑 clip2 v3，**反复听 3 遍**确认无说话声 | Pic10 v3 md5 `1d5a6466a4d51875c01f0c568877acc5` ✓ |

### v3 实测数据

- task_id: `cgt-20260610160820-xbx5g`
- 耗时：4 分 4 秒（244 秒）
- 实际时长：6.08s
- 音轨：✅ AAC 立体声 6.12s（无莫名说话声）
- md5: `1d5a6466a4d51875c01f0c568877acc5`
- **用户目检**："这次修复了音频问题" ✅（**修复通过信号**）

## 2. Pic10 Hamster v1→v2 实战案例（画面问题）

### v1 翻车现场

- **症状**：clip1 v1 镜头死板 = 故事性 = 0（用户原话："镜头设计不连续"）
- **典型镜头**："喊出来+挥手+收势"（**3 镜固定模板**）= 镜头凑数
- **末帧问题**："画面定格在仓鼠准备享用葵花籽的最后一帧"（没写"画面不再动"）= 实际不微动

### v1→v2 修复：4 步原则重写

- Step 1 症状分类：镜头死板 = A 类画面问题
- Step 2 定位根因：套了"建立/单词/收势"3 镜固定模板 + 末帧"画面定格"措辞
- Step 3 修复：按 p1 实际画面（仓鼠捧种子）→ 推断动作步骤 → 2 镜（建立+静默消化） + 末帧写"画面不再动"
- Step 4 验证：跑 clip1 v2，task_id `cgt-20260610154600-rzf6g`，md5 `8d5888b3c5745c71e81aee07b149435b` ✓

## 3. Pic8 Rabbit Clip4 v1→v2 实战案例（双重否定失声）

### v1 翻车现场

- **症状**：clip4 v1 视频**完全静默**（无任何音效）
- **prompt 问题**："全程无任何背景音乐"（双重否定）+ 又写"环境音：森林"（BGM 描述冲突）= 模型不知道怎么生成音效

### v1→v2 修复

- **删除**"无任何背景音乐" + 删除"环境音"描述
- **改用**：只写一句"保持无字幕、无水印、无 Logo，无人声、无歌唱、无配音、无朗读"
- **音效内联**到每个镜头（`<远处鸟叫声渐弱>` 等）

### 沉淀反模式

❌ **双重否定"无任何 BGM" + 又写 BGM 描述** = 完全静默（Pic8 v1 翻车）
✅ **正向约束**：只写"无人声/无歌唱/无配音/无朗读"一句（不写"无任何"等双重否定）

## 4. 翻车修复优先级（哪个最常踩）

| 排名 | 翻车类型 | 出现频率 | 修复成本 | 总案例数 |
|---|---|---|---|---|
| 1 | **B 类莫名说话声**（元数据术语）| 🔥🔥🔥 高 | 低（改 1-2 行）| 1+（Pic10 v2）|
| 2 | **A 类镜头死板**（3 镜模板）| 🔥🔥🔥 高 | 中（重写整段）| 2+（Pic10 v1 + Rabbit newclip1 v1）|
| 3 | **A 类末帧"微动"不动** | 🔥🔥 中 | 低（改末帧措辞）| 1+（Pic10 v1）|
| 4 | **B 类完全静默**（双重否定）| 🔥 中 | 低（删 BGM 描述）| 1（Pic8 v1）|
| 5 | **C 类风格漂移** | 🔥 低 | 中（重写段 1）| 0-1 |
| 6 | **D 类时长压缩** | 🔥 低 | 低（减镜头）| 0-1 |

> **最高频 = B 类元数据术语触发人声**（Pic10 v2 翻车）—— **必检** prompt 里有没有 "TTS / 音轨占位 / 朗读时长" 这 3 个关键词

## 5. 一句话 TL;DR

> **翻车修复 5 步法**：
> 1. **分症状**（A 画面 / B 音频 / C 角色 / D 时长）
> 2. **走诊断树**（每个症状类别有独立根因树）
> 3. **改对应段**（A → §1.3 多镜头时间线 / B → §1.4 整段音频约束段 / C → §1.2 主体定义 / D → §1.3 镜头数）
> 4. **单 Clip 端到端先验**（目检通过再批量）
> 5. **沉淀到 skill**（反推 v15director 规范 + 加反模式库条目 + 加铁律）
>
> **黄金口诀**：
> - **"莫名说话声" = 看 prompt 里有无 "TTS/音轨占位/朗读时长"** → 有就删 + 用官方原话
> - **"镜头死板" = 看 prompt 套了 3 镜模板没** → 套了就改 4 步原则
> - **"末帧不动" = 看末帧写了 "画面继续微动" 没** → 写了就改 "画面不再动"
> - **"完全静默" = 看 prompt 有无双重否定 + BGM 描述冲突** → 有就只留一句官方原话
