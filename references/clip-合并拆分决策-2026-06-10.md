# Clip 合并/拆分决策树 · 2026-06-10

> **本文档 = 顶层决策入口**。绘本级 → 范式级 → 段级 → 特殊场景，**逐级往下走**。每个分支指向 references/ 下的具体范式模板。
> **核心问题**：拿到 1 本绘本，怎么决定"分几个 Clip、每个 Clip 几张图"？
> **来源**：Pic2-Pic8 实战沉淀（2026-06-06 ~ 2026-06-10）·用户原话："什么时候合并/拆分要看场景密度，不是看时间"。

---

## 决策树（自顶向下 4 层）

```
Step 1 · 绘本级判定  ─── 是不是领读型？
   ├─ 是 → Step 2A · v7 范式
   └─ 否 → Step 2B · v15 4 段范式
                │
                ↓
Step 3 · 段级判定  ─── 单段时长 / 语义密度
   ├─ 单段旁白 < 8s + 弱情节 → 2图=1Clip 合并（v7 范式）
   ├─ 单段 > 15s 或 高密度段（家族词组≥3词） → 独立 Clip
   └─ 总时长对不齐 → 长旁白拆分规范（v15.1）
                │
                ↓
Step 4 · 特殊场景
   ├─ 家族词组集合（≥3 词 + 强文字持续可见） → 独立 Clip + --generate-audio false
   ├─ 收势/封面段 → 6-8s 极短档（v5 公式）
   └─ TTS 后期精准匹配 → 总时长让位给 TTS 时长
```

## Step 1 · 绘本级判定（"是不是领读型"）

### 判定 4 问

| 问 | 答案 → 范式 |
|---|---|
| 1. 主体是"单词/字母/认知集合"还是"角色冒险/情节推进"？ | 单词集合 → 领读 → **v7** ；角色冒险 → **v15** |
| 2. 风格跨图是否统一（同色系/同主体/相邻景别）？ | 全统一 → **v7** ；有 3+ 张风格跳变 → **v15**（拆开单图） |
| 3. 总图数 6-10 张？ | 是 → **v7**（5 条件全过的甜区） ；>10 张 → **v15**（细切） |
| 4. 旁白是"每段独立单词/认知"还是"贯穿叙事"？ | 独立单词 → **v7** ；贯穿叙事 → **v15** |

### 范式速查

| 维度 | **v7 范式** | **v15 4 段范式** |
|---|---|---|
| 适用 | 领读/认知/认字绘本 | 叙事/冒险/收势向 |
| 切分 | 2图=1Clip 合并 | 单图 1Clip |
| 典型 | Cactus 4 Clip 37s / Horse 5 Clip 48s / Rabbit 5 Clip 55s | Pic2 Please 8 Clip 47s |
| 调 C 子 agent？ | ❌ 不调（主 agent 直拼 8 段 prompt）| ✅ 调 C 产原料 |

**详细规则 → [`v7-vs-v15-paradigm-routing.md`](./v7-vs-v15-paradigm-routing.md)**

## Step 2A · v7 范式下的合并/拆分

### v7 范式判定 5 条件（**必全过 · 铁律 #89**）

1. ✅ **领读/认知/认字绘本**（不是叙事型）—— 例：单词/字母教学、家族词组集合、动物认知
2. ✅ **弱情节**（无明显"前一幕→后一幕"情节推进）—— 例：同一主角不同动作展示
3. ✅ **旁白每段 < 8s**（按 0.3s/字算）—— 英文 0.3s/词
4. ✅ **图片风格统一**（同色系/同主体/相邻景别）—— 例：Eric Carle 拼贴风全 8 张
5. ✅ **总图数 6-10 张**（多了需要更细分段）

**5 条件任一不满足 → 走 v15 4 段范式 + 调 C**。

### v7 范式下的"何时合并、何时拆"

| 情况 | 处理 | 时长公式 | 实测 |
|---|---|---|---|
| 普通段（旁白短+单焦点+弱情节） | **2图=1Clip 合并** | TTS1+TTS2+0.5-1.5s 缓冲 | Cactus Clip 1-3（8/9/10s）|
| 收势/封面段 | **独立 Clip** | 6-8s 极短档（v5 公式）| Cactus Clip 4（10s）|
| **高密度段**（家族词组≥3词 / 全文+微动画 / 强文字持续可见）| **独立 Clip** | 14s 起步 | **Rabbit Clip 4 IT家族 14s** ⭐ |

### 关键判断口诀

> **"单段内容密度高 = 独立 Clip + 14s；普通段 = 合并"**

### ❌ 反模式（Pic8 Rabbit 实战翻车）

- ❌ 5 词家族塞 2图=1Clip 合并（11-12s 装不下 5 词+微动画+静默 = 末帧 < 2s）
- ❌ 给家族段 8s 上限（误读 v5 默认档为硬上限）
- ❌ 用 v15 范式填 v7 合并（v15 模板只支持单图 @ImageN，不支持 @Image1 + @Image2 双图）

### 详细规则 →

- **v7 范式 8 段固定结构 + 11 项自检清单** → [`leading-reading-4clip-pattern.md`](./leading-reading-4clip-pattern.md)
- **v7 vs v15 路由决策** → [`v7-vs-v15-paradigm-routing.md`](./v7-vs-v15-paradigm-routing.md)

## Step 2B · v15 范式下的合并/拆分

### v15 范式判定 4 问

| 问 | 答案 → 处理 |
|---|---|
| 1. 单图朗读时长 + 必要留白 > 15s？ | 是 → **必须拆**（铁律 #37 语义块拆分）|
| 2. 旁白是按时间均分还是按语义块分？ | **按语义块**（铁律 #37） |
| 3. 末帧是否需要"读完后留白"？ | **必须**（铁律 #36 用户原话："音频一读完画面就结束"是错的） |
| 4. 镜头数 ≥ 4 + 每镜头有朗读+静默节奏？ | 是 → 走 v15.1 节奏公式（2-1-3-3-3）|

### v15 范式"必须拆"触发

| 情况 | 单图朗读 | + 必要留白 | 总时长 | 处理 |
|---|---|---|---|---|
| 不需要拆 | < 5s | + 5s | < 10s | 1 Clip 搞定 |
| 临界 | 5-8s | + 7-9s | 12-17s | 看是否 ≤ 15s |
| **必须拆** | **> 8s** | **+ 10s+** | **> 15s** | **多 Clip 拆** |

**判定公式**：`朗读总时长 + 单词数 × 2.5s 留白 > 15s` → 必须拆

### 拆分原则 · 按"语义块"不按"时间"

**❌ 错**（按时间均分）：6 个 AT 词 = 朗读 7.5s + 留白 13s = 20.5s → 按时间均分 7.5s+7.5s → 后半装不下
**✅ 对**（按语义块）：6 个 AT 词按概念分 3 块（cat+hat+mat / bat+fat+rat / 收势组）→ 12+12+6s

### 详细规则 → [`长旁白拆分规范-v15.1.md`](./长旁白拆分规范-v15.1.md)

## Step 3 · 段级判定（单段时长 / 语义密度）

| 单段内容 | 推荐做法 | 时长 |
|---|---|---|
| 单焦点 + 旁白 < 8s | 合并到相邻段 | 8-10s 合并 Clip |
| 家族词组 ≥ 3 词 + 强文字持续可见 | **独立 Clip** | 14s 起步 |
| 全文画面 + 微动画（如 Rabbit Clip 4 IT家族 5词）| **独立 Clip** | 14s 起步 |
| 收势/封面 | **独立 Clip** | 6-8s 极短档 |
| 末帧（朗读完后必须留白）| 必须留 ≥ 2-3s | 末帧时长 ≥ 朗读时长 1/2 |

## Step 4 · 特殊场景

### 场景 1：家族词组集合（≥3 词 + 强文字持续可见）

**触发条件**：
- 同字母家族 ≥ 3 词（AT/IT/AN 家族）
- 文字需"全程可见 + 伴随微动画"（不是闪现式）

**特殊处理**：
- 独立 Clip（**不要合并**到相邻段）
- 时长 **14s 起步**（5 词需 5×2.5s+ 缓冲 = 14s）
- **`--generate-audio false`**（铁律 #86，音效密集时静音留给 TTS 后期）
- 末帧必须 ≥ 2-3s 留白（5 词全部出现 + 鸟叫渐弱）

**实测**：[`2026-06-10-rabbit-clip4-it-family.md`](./2026-06-10-rabbit-clip4-it-family.md) ✅

### 场景 2：TTS 后期精准匹配

**触发条件**：用户明确说"我后期用 TTS 旁白匹配画面"

**特殊处理**：
- **总时长让位给 TTS 时长**——不为了对齐"标准 35s"而压缩
- 合并 Clip 时长 ≥ TTS1 + TTS2 + 0.5-1.5s 缓冲
- 算每段 TTS 耗时（中文 0.3s/字 + 英文 0.3s/词）

**实测**：[`leading-reading-4clip-pattern.md`](./leading-reading-4clip-pattern.md) 2026-06-02 用户原话："合并 clip 时，你要注意画面的切换时间"

### 场景 3：收势/封面段

**特殊处理**：
- 6-8s 极短档（v5 公式）
- 独立 Clip（不合并到主段）
- 末帧"读完后留白 ≥ 2-3s"（铁律 #36）

### 场景 4：跨风格混合（部分统一部分跳变）

**特殊处理**：
- 风格统一的 2 张 → v7 合并
- 风格跳变的 1 张 → 独立 Clip
- 整本走 v7.1 混合合并范式（不是"全合并"或"全独立"）

## 单 Clip 时长边界（铁律 #1 + #90 升级版）

| 边界 | 值 | 依据 |
|---|---|---|
| **下限** | 4s | Seedance 2.0 物理最低（官方 doc3 line1705） |
| **上限** | 15s | Seedance 2.0 物理最高（官方 doc3 line1705） |
| **甜区** | 8-12s | 节奏稳、TTS 后期好对齐 |
| **极短档** | 6-8s | 收势/封面专用（v5 公式）|
| **长档** | 14s | 家族词组集合专用 |

**8s < x ≤ 15s 完全合法**——v5 公式档位 5/6/7/8s 是参考范围，**不是上限**。
