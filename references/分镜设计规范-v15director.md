## 0. 心智模型（区别于"会画图"思维）

> **Seedance 2.0 本质上是一个多模态 AI 导演**（官方 doc2 §进阶公式 原话）

模型内部建模 = **空间层**（画面里有什么） + **时间层**（事情如何随时间变化）**两个维度**。

| 层 | 内容 | prompt 对应字段 |
|---|---|---|
| **空间层** | 主体 + 场景 + 风格 | 段 1 素材角色声明 + 段 2 主体定义 + 段 6 风格 |
| **时间层** | 动作 + 镜头 + 时序 | 段 3 多镜头时间线（核心） |

**反模式**（Rabbit 旧版踩过）：prompt 只写"镜头缓慢平稳推进 + 不切镜 + 不剧烈运动" = 空间层完整、时间层空白 = 生成"死板推镜头"——**故事性 = 0**。

## 1. prompt 6 段骨架（v15 导演思维版 · Rabbit 验证）

### 1.1 骨架总览

```
[1] 素材角色声明段      @image1 + 风格定义
[2] 主体定义段          将图片1中的 X 定义为 <主角>
[3] 多镜头时间线段      镜头 1-N（按事件切，不按秒数）
[4] 整段音频约束段      全程无 BGM + 音效 <...> + TTS 占位
[5] 画面禁令段          无字幕无水印无 Logo
[6] Storyboard 引导句   模型自行设计合理的呈现节奏
```

### 1.2 各段详细规范

#### [1] 素材角色声明段（开篇 1 段 · 必加）

**作用**：开篇就说"谁是什么/参考什么"，再进入分镜。

```
@image1 as the only visual reference for the entire video,
children's picture book 2D paper-cut collage style,
soft pastel palette of mint green, cream, white, and warm yellow,
paper texture and torn edges clearly visible.
```

**关键点**：
- ✅ `@image1` 明确角色（`reference_image`）
- ✅ 风格一次性说清（防止漂移）
- ❌ 不要直接进分镜（缺素材角色声明 = 官方 doc2 §6 案例对比中 v14 缺失的项）

#### [2] 主体定义段（开篇 1 段 · 必加 · 官方 §4.2 支持）

**作用**：给主角起名字标签，每次提及都绑同一标签。

```
主体定义：将图片1中的小兔子定义为<主角小兔>。
```

**官方支持**（doc2 §1 主体定义 · line157）：
> "将图片1中穿红色连衣裙、戴草帽的女人定义为**张红**（**注意：可以给角色起名字**——我们之前从未给主角命名）"

**关键点**：
- ✅ 主角 = **物种名 + 个性名**（小兔子 → 主角小兔），不是抽象"小兔子"
- ✅ 后续每次提主角都用 `<主角小兔>` 标签（doc2 §4.3 第 1 条红线）
- ⚠️ 绘本项目**不踩"双胞胎"坑**（doc2 §8.2 —— 多角色场景才需要人名区分）

#### [3] 多镜头时间线段（⭐ 核心 · 导演思维的灵魂）

**作用**：按事件分镜，每镜头齐全 **运镜 + 动作 + 位置 + 音频** 4 逻辑（官方 doc2 §1.2 原话）。

**格式**：
```
镜头 1（建立 · 0-Xs）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
镜头 2（单词动作 · X-Ys）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
镜头 3（收势 · Y-Zs）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
```

**4 逻辑**（官方 doc2 §1.2 原话 · 缺一不可）：

| # | 顺序 | 内容 | Rabbit 范例 |
|---|---|---|---|
| 1 | **运镜或镜头切换方式** | "全景缓慢推近""固定机位""镜头切至…" | "中景拉远定格""镜头切到侧面中景特写""缓推回到正面中景" |
| 2 | **主体动作与表情** | 核心角色/物体的关键动作、神态变化 | "<主角小兔>缓缓从草丛里探出半个脑袋，胡须轻颤，眼睛眨了两下" |
| 3 | **位置或空间变化** | 主体所处的场景、位置或空间关系 | "草丛""侧面中景""花田中央" |
| 4 | **音频信息** | 音效、人声或背景音乐（**每个镜头内联**，不集中末尾） | `<一声轻巧的"叮"，像铃铛一响>` |

**镜头数算法**（v15 + 官方 · 实际可执行版）：

| 时长 | 旁白密度 | 推荐镜头数 | 节奏公式 |
|---|---|---|---|
| **5s** | 1 短句（5-7 字） | **2-3 镜头** | 1.5s 建立 + 1.5s 单词动作 + 2s 收势 |
| **8s** | 1 短句 | 3-4 镜头 | 2s 建立 + 1s 跃入 + 3-4s 单词 + 1s 收势 |
| **12s** | 3 词家族 | 5 镜头 | 2-1-3-3-3（Cat 4a v3 标版） |
| **14s** | 5 词家族 | 5 镜头 | 2-1-3-3-3-3（Pic4 v2 标版，5 词 IT 家族） |
| **15s** | 6+ 词 | 6-7 镜头 | 2-1-3-3-3-3-...（Pic7 Horse OR 家族实测） |

> ⚠️ **2026-06-10 Hamster 实战 · 4 步原则重写镜头数判定**（用户强纠错"镜头设计不连续"）：
> - ❌ 旧版：套"建立/单词/收势"3 镜模板（凑数）= 故事性 = 0
> - ✅ 新版：**按参考图实际动作步骤定镜头数（1-3 镜 / 段 · 不固定 3 镜）**
> - 钻木屑 = 1 个动作 = 1 个钻的动作镜头 + 1 个静默消化 = **2 镜**
> - 跑轮 = 1 个跑的动作 = **1-2 镜**（不凑 3 镜）
> - 末帧 = **真正静默**（"画面不再动"）· 整段 30-40% = 静默消化
> 详细见 §7 4 步原则

**镜头数判定口诀**（**v1.2.0+pic21 新版**）：
- **看参考图 → 数动作步骤 → 决定镜头数**（1-3 镜 / 段自由组合）
- 末帧必加 1 个静默消化镜头（**写"画面不再动 · 整段留出静默"**——真正静默不是微动）
- **总镜头数 ≤ 时长 ÷ 1.5s**（每镜头至少 1.5s，少于则糊）

**反模式**：
- ❌ 5s 5 镜头 = 每镜头 1s（Rabbit 旧版踩过）→ 镜头太碎糊
- ❌ 5s 1 镜头 = 单推镜头无切镜（v6 简化版踩过）→ 故事性 = 0
- ❌ 套"建立/单词/收势"3 镜固定模板（Hamster v1 踩过 · 用户 2026-06-10 强纠错）→ 镜头凑数
- ❌ 末帧写"画面继续微动"（v15 旧版踩过 · 模型理解为"定格"）→ **改"画面静默停留 + 静默消化"**
- ❌ 按秒数硬切（"0-2s/2-4s/4-6s"）→ 官方 doc2 §1.3 红线"模型对精确时间支持不稳定"
- ❌ 不看参考图就套"喊出来"模板（Hamster Clip 1 v1 踩过）→ 故事性 = 0
- ❌ **默认"镜头 1 = 中景拉远定格 + 一声轻柔的微风声"模板**（**Pic10 Hamster 5-8 段踩过** · 6/6 段 = 100% 套用）→ 用户 2026-06-10 反馈"分镜设计过于模板化"· 实际应该是**按画面真实状态决定运镜 + 音效**（仓鼠蹲伏 = 固定中景 / 仓鼠从门洞探出 = 侧面机位看门洞 / 跑轮 = 正面平视 / 草地 = 拉远 / 静态 = 不动）
- ✅ **按事件切**（建立/单词/收势）—— 时长让模型自己安排

#### [4] 整段音频约束段（**v1.2.0+pic21 官方原话版**）

**作用**：明示"无 BGM + 无朗读 + 无配音 + 无水印"4 条红线（**Pic10 Hamster 修复"莫名说话声"后沉淀**）。

```
保持无字幕、无水印、无 Logo，无人声、无歌唱、无配音、无朗读。
```

**关键点**：
- ✅ **用官方原话**："无人声、无歌唱、无配音、无朗读"（来自官方 doc2 line534 末尾约束段 + 实际修复经验）
- ✅ 不用元数据术语（**不要**写"TTS 音轨占位 / 时长匹配旁白朗读时长"——seedance 看到"音轨占位"+"朗读时长"会自我生成人声填补，导致"莫名说话声"）
- ✅ 音效写 `<...>` 包裹（官方 doc2 §440-451 特殊字符规范）

**音效特殊字符规范**（官方 doc2 §440-451 原话）：

| 信息类型 | 符号 | 范例 |
|---|---|---|
| **音效** | `<>` | `<远处传来狗叫声>` |
| **音乐** | `（）` | （背景中播放着快节奏的摇滚乐） |
| **台词** | `{}` | {你好，世界} |
| **字幕** | `【】` | 【第一章：启程】 |

**绘本项目音效策略**（Pic10 Hamster v3 沉淀）：
- ✅ 音效每个镜头内联（不集中末尾）
- ✅ BGM = 绘本红线（Pic8 用户原话："不要乱发挥，不生成 BGM 是红线"）
- ✅ CLI 参数 `--generate-audio true`（让 seedance 生成音效）
- ✅ **不写元数据术语**（TTS/音轨占位/朗读时长 全部禁止出现在 prompt）

**反模式**：
- ❌ "环境音效" "声音" "BGM" 全纯文字（缺符号规范 = 模型理解模糊）
- ❌ 双重否定"无任何背景音乐" + 又写 BGM 描述（Pic8 v1 翻车 → 完全静默）
- ❌ 写"TTS 音轨占位，时长匹配旁白朗读时长（X 秒）"（**Pic10 Hamster v2 翻车** → seedance 看到"音轨占位"+"朗读时长"激活人声生成路径，产生"莫名说话声"）

**翻车修复方法论**（**v3 修复后沉淀** · 5 步法 + 4 大症状诊断树）：
→ 详见 [`seedance-prompt-翻车修复方法论-2026-06-10.md`](./seedance-prompt-翻车修复方法论-2026-06-10.md) ⭐ 必看

#### [5] 画面禁令段（必加）

```
画面保持无字幕、无水印、无 Logo。
镜头全程严格遵守"1 镜头 1 种运镜"红线（不堆叠推拉摇移）。
```

**关键点**：
- ✅ "1 镜头 1 种运镜" = 官方 doc2 §4 原话"不要同时要求推拉摇移，会增加画面的不稳定性"
- ✅ 无字幕/水印/Logo = seedance 防幻觉

#### [6] Storyboard 引导句（末尾 · 必加）

```
This is a storyboard reference image sequence, designed for picture book
reading - viewers should clearly see the rabbit scene unfold with gentle
micro-animations, holding the final pose for natural reading rhythm.
```

**作用**：让 seedance 看图自行设计微动效果，不写具体动效（Cat 范本精神）。

## 2. 运镜术语库（直接抄进 prompt）

> 官方 doc2 §127-141 + director-mode.md line131-141 · **模型认识以下术语**：

**镜头运动**（**1 镜头只 1 种**，红线）：
- `tracking shot` — 摄像机跟随主体移动
- `dolly zoom` — 后退+镜头推进（眩晕效果/希区柯克变焦）
- `push in / pull out` — 推进/拉远
- `whip pan` — 快速摇镜
- `crane shot` — 升降镜头
- `360° rotation` — 360度旋转
- `steady tracking shot` — 平稳跟拍
- `fixed shot` — 固定机位
- `缓慢推镜` / `平稳横移` / `固定镜头`（中文 seedance 也认）

**景别**（绘本项目最常用）：
- `Extreme close-up` — 特写（眼睛、嘴巴等细节）
- `Close-up` — 脸部特写
- `Medium close-up` — 头部+肩部
- `Medium shot` — 腰部以上
- `Full shot` — 全身
- `Wide / Establishing shot` — 环境全景
- `Bird's eye` — 俯视镜头
- `First-person POV` — 主观镜头

**绘本推荐组合**（Rabbit 验证）：
- 建立镜头：`中景拉远定格` / `全景推入`
- 单词/事件镜头：`侧面中景特写` / `正面近景`
- 收势镜头：`缓推回到正面中景` / `拉远至全景`

## 3. 动作量化公式（官方 doc2 §3 · 4 条铁律）

### 3.1 肢体细化 + 程度量化

> 官方原话："动作需具体到手、腿、头部、肩背等肢体部位，同时补充**幅度、速度、力度**描述"

**反例**：`小兔子动了`
**正例**：`缓缓抬手到胸口高度，胡须轻颤，眼睛眨了两下`

### 3.2 优先低缓连续小动作

> 官方原话："优先选用缓慢、轻柔、连贯的细微动作，**尽量规避狂奔、大跳、剧烈翻滚等高爆发、大动态动作**"

**绘本项目**：绘本本来就是"温柔讲故事"—— 优先用"缓缓""微微""轻轻"等词。

### 3.3 补充动作过渡衔接（⭐ 官方强调）

> 官方原话："写明前后动作的**惯性与承接关系**，保证画面动作连贯自然"

**范例**：
- ❌ "兔子抬头 + 兔子转头"（无承接）
- ✅ "**借着转身惯性顺势抬头**"（有承接）

**绘本应用**：每个镜头 2 切换时，**镜头 2 开头承接镜头 1 末尾**——"从停顿状态自然过渡到举手"。

### 3.4 情绪具象外化表达（⭐⭐⭐ 官方"情绪字典"是金矿）

> 官方原话："用具体的身体细节表现情绪，替代 '很悲伤' '非常愤怒' 这类抽象词汇"

| 抽象情绪 | 外化为动作与细节（官方原话） |
|---|---|
| **悲伤** | 低头、肩膀微微颤抖、眼眶泛红、手指无意识地攥紧衣角、泪水在眼眶里打转但没有落下 |
| **喜悦** | 嘴角抑制不住地上扬、眉眼舒展、脚步变得轻快、下意识地哼起小曲、忍不住原地转个圈 |
| **紧张 / 焦虑** | 频繁地看手表、手指不停敲击桌面、呼吸急促、眼神闪躲、无意识地啃咬指甲 |
| **愤怒** | 双拳紧握、下颌线紧绷、胸口剧烈起伏、眼神如刀般锐利、从牙缝里挤出话语 |
| **释然** | 长长地舒了一口气、紧绷的肩膀完全放松下来、脸上露出久违的、淡淡的微笑、抬头望向远方 |

**绘本项目应用**：绘本主角"高兴/好奇/失落"等情绪**直接查这张字典挑细节**，**不写 "happy" "sad" 这种抽象词**。

## 4. 完整 prompt 模板（Rabbit newclip1 v2 验证通过 · 1657 字符 · 5s · 3 镜头）

> ⚠️ **本模板适用范围限制**（**Pic10 Hamster 5-8 段踩坑后补充**）：
> - **仅适用于"找一找"类场景**（主角藏在草丛/门洞/木屑里 = 需要"拉远"交代主体在场景中的位置）
> - **不适用**于"主角在画面中央"类场景（仓鼠蹲伏在木屑上、仓鼠在跑轮里跑、仓鼠在草地上吃）= **不**需要"拉远"运镜
> - **新铁律**（**v1.2.0+pic22 · 2026-06-10 用户反馈"分镜设计过于模板化"后沉淀**）：**禁止把本模板的"中景拉远定格"作为默认开场运镜**· 必须按参考图实际画面状态决定运镜（见 §1.3 反模式库 + §7 4 步原则）

```
@image1 as the only visual reference for the entire video, children's picture book 2D paper-cut collage style, soft pastel palette of mint green, cream, white, and warm yellow, paper texture and torn edges clearly visible.

主体定义：将图片1中的小兔子定义为<主角小兔>。

整段视频呈现"找一找"主题。旁白朗读："小兔子 rabbit，藏在哪里？"。参考图原有的"rabbit"与"兔子"字样作为画面元素自然融入场景，不重新生成任何新文字。
```

## 7. 镜头设计新原则（2026-06-10 Hamster 实战 · 用户强纠错）⭐ 必读

> **用户原话**（2026-06-10 强纠错"镜头设计不连续"原话复盘 4 步）：
> 1. 以我给你的参考图作为首帧，推断后面 clip 时长
> 2. 结合旁白内容，推断 clip 时长接下来可能发生的事情
> 3. 根据旁白的故事情节来设计镜头，不要按照固定的模式来设计
> 4. 留出呼吸感的镜头衔接的静默内容（让观众消化的画面）
>
> **关键澄清**：首帧的意思**不是**让你用首尾帧模式（`--image`+`--last-frame` 已禁用 · 仍用 v15 多图参考）
>
> **仍然使用导演思维编写提示词**

### 7.1 4 步原则详解

**Step 1 · 以参考图作为首帧推断后面 clip 时长**
- ❌ 反模式：套"5s=2-3 镜头 / 8s=3-5 镜头"等固定公式
- ✅ 正解：先看 p1 实际画面（仓鼠捧种子 / 跑轮 / 木屑）→ 推断"仓鼠可能接下来钻进木屑 / 跑上转轮 / 啃种子" → 根据动作复杂度定 5/6/7s
- 时长决定权 = 动作内容（不是公式）

**Step 2 · 结合旁白内容推断接下来可能发生的事情**
- ❌ 反模式：看到 "happy hamster eats" = 写"仓鼠笑 + 仓鼠吃"（文字表面）
- ✅ 正解：看参考图实际场景（草地 + 木碟 + 葵花籽）→ 推断"眯眼笑 + 双手捧瓜子 + 木碟备用" → 把画面已有元素**当故事起点**写 prompt
- 旁白 + 画面 = 双源推断

**Step 3 · 根据故事情节设计镜头 · 不按固定模式**
- ❌ 反模式：v15 范式被理解为"6 段骨架 = 6 段文字" + 每段固定"3 镜头 = 建立/单词/收势"
- ✅ 正解：镜头数 = 动作步骤数（钻木屑 = 1 个钻的动作镜头 / 跑轮 = 1 个跑的动作镜头 / 探头 = 1 个探头的动作镜头），**不**为凑数加空镜
- 每 Clip 镜头数：1-3 个，**不**硬套 3 镜

**Step 4 · 留出呼吸感的静默内容**
- ❌ 反模式：末帧写"画面继续微动"（v15 旧版踩过 = 实际不微动）
- ✅ 正解：末帧镜头**写"画面不再动 · 整段画面留出静默 · 让 XX 在小朋友心中沉淀"**—— 真正的静默（不是"微动"）
- 静默时长 = 整段时长的 30-40%（5s 留 1.5-2s 静默 / 6s 留 2-2.5s 静默 / 7s 留 2-2.5s 静默）

### 7.2 Hamster 8 段实战 · 4 步原则落地实例

### 7.4 "首帧" ≠ "首尾帧" 关键澄清

| 维度 | 4 步原则的"首帧" | seedance "首尾帧范式" |
|---|---|---|
| **含义** | 视觉设计起点 = 参考图 | API 参数 = `--image` + `--last-frame` 锁住首/尾两帧 |
| **作用** | 推断情节 + 决定镜头内容 | 锁帧 = 模型只能在两帧之间插值 |
| **是否禁用** | ✅ 必用 | ❌ 禁用（v1.2.0+pic21 红线）|
| **使用方式** | "看 p1 仓鼠捧种子 → 推断 p2 仓鼠在户外草地" | "锁 p1 仓鼠 + 锁 p2 仓鼠 → 模型在中间过渡" |

**判断口诀**：
- 看到"首帧" = 思路起点 = "以参考图为基准来设计" ✅

### 7.6 运镜 + 音效决定规则（**v1.2.0+pic22 新增 · 2026-06-10 用户反馈"分镜设计过于模板化"后沉淀**）

> **新铁律**（**⭐ 必读**）：**禁止默认"镜头 1 = 中景拉远定格 + 一声轻柔的微风声"模板**。Pic10 Hamster 5-8 段 = 6/6 段 100% 套用 = 翻车。**运镜和音效必须按参考图实际画面状态决定**。

**Step A · 看图决定景别 + 运镜**（不写"中景拉远"作为安全帽）：

| 画面状态 | 镜头 1 推荐运镜 | 反例（❌ 模板化） |
|---|---|---|
| **主角在画面中央静态**（仓鼠蹲伏、仓鼠站桩）| **固定中景** + 不动 = "仓鼠就是舞台中心" | ❌ "中景拉远定格"（仓鼠本来就在中央 = 拉什么？）|
| **主角从洞口/草丛探出**（木屋门洞、草丛藏身）| **侧面机位看洞口**（找一找）| ❌ "中景拉远"（洞口才是视觉焦点）|

| **主角在开阔户外**（草地、空地、花田）| **中景拉远**（交代户外空间感）| ❌ 拉远 = OK · **但不是默认** |
| **主角在角落小场景**（钻木屑、藏角落）| **近景特写**（捕捉细节动作）| ❌ 拉远（看不到细节）|

**Step B · 看图决定音效**（不写"<一声轻柔的微风声>"作为默认兜底）：

| 画面场景 | 推荐音效 | 反例（❌ 模板化） |
|---|---|---|
| **户外草地 / 花田** | <微风声> / <鸟叫> / <远处铃声> | ✅ OK（自然户外）|
| **木屑堆 / 笼舍里** | <细碎刨花声> / <仓鼠呼吸声> / 静默 | ❌ "微风声"（笼舍里没风）|
| **跑轮 / 跑动中** | <跑轮踏板吱呀声> / <脚步轻踏声> | ❌ "微风声"（跑动是动态声）|
| **木屋门洞 / 室内** | <木屋吱呀声> / <仓鼠的呼气声> / 静默 | ❌ "微风声"（室内没风）|
| **静态画面**（仓鼠呆萌蹲着）| **静默**（让画面安静）| ❌ 任何声音 = 喧宾夺主 |

**Step C · 镜头命名不写"建立场景"标签**：

- ❌ "镜头 1（建立场景·0-2s）"（每个镜头 1 都长这样 = 模板化）

## 8. 一句话 TL;DR（v1.2.0+pic22 升级版）

**镜头设计 4 步原则 + 3 个 Step 决定**（**Pic10 Hamster 实战沉淀 · 2026-06-10 用户强纠错**）：

1. **看参考图**（以 p1 实际画面为基准）
2. **推断情节**（旁白 + 画面 = 双源推断）
3. **写动作步骤数**（按动作复杂度 1-3 镜 / 段 · 不固定 3 镜）
4. **留呼吸感静默**（末帧写"画面不再动 · 让 XX 在小朋友心中沉淀"——真正静默不是微动）

**Step A 看图决定景别 + 运镜**（v1.2.0+pic22 新增）：

- **主角在画面中央静态** → **固定中景**（仓鼠蹲伏/站桩）
- **主角从洞口/草丛探出** → **侧面机位看洞口**（木屋门洞/草丛藏身）
- **主角在跑轮/木屋里动** → **正面平视**（跑轮/跑动/玩耍）
- **主角在开阔户外** → **中景拉远**（草地/花田/空地）
- **主角在角落小场景** → **近景特写**（钻木屑/藏角落）

**Step B 看图决定音效**（v1.2.0+pic22 新增）：

- **户外草地** → <微风声>/<鸟叫>（✅ 合理）
- **木屑堆/笼舍里** → <细碎刨花声>/<仓鼠呼吸声>/静默（❌ 不写微风声）
