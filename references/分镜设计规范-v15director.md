> **来源**：Seedance 2.0 官方 3 份原文档（doc1/doc2/doc3）+ v15 4 段骨架 + v15.1 长旁白拆分规范 + director-mode.md + Pic8 Rabbit Clip4 v2 实战 + **Rabbit newclip1 v2（2026-06-10 用户目检通过）**
>
> **状态**：✅ 标版（Rabbit newclip1 v2 = 验证通过）
>
> **核心原则**：**镜头设计 = 服务于故事、服务于叙事**。导演思维 = 摄像机在哪、走多快、近景远景、光线从哪来。**不是**"单图配 5 秒推镜头"。

---

## 0. 心智模型（区别于"会画图"思维）

> **Seedance 2.0 本质上是一个多模态 AI 导演**（官方 doc2 §进阶公式 原话）

模型内部建模 = **空间层**（画面里有什么） + **时间层**（事情如何随时间变化）**两个维度**。

| 层 | 内容 | prompt 对应字段 |
|---|---|---|
| **空间层** | 主体 + 场景 + 风格 | 段 1 素材角色声明 + 段 2 主体定义 + 段 6 风格 |
| **时间层** | 动作 + 镜头 + 时序 | 段 3 多镜头时间线（核心） |

**反模式**（Rabbit 旧版踩过）：prompt 只写"镜头缓慢平稳推进 + 不切镜 + 不剧烈运动" = 空间层完整、时间层空白 = 生成"死板推镜头"——**故事性 = 0**。

## 1. prompt 6 段骨架（v15 导演思维版 · Rabbit 验证）

### 1.1 骨架总览

```
[1] 素材角色声明段      @image1 + 风格定义
[2] 主体定义段          将图片1中的 X 定义为 <主角>
[3] 多镜头时间线段      镜头 1-N（按事件切，不按秒数）
[4] 整段音频约束段      全程无 BGM + 音效 <...> + TTS 占位
[5] 画面禁令段          无字幕无水印无 Logo
[6] Storyboard 引导句   模型自行设计合理的呈现节奏
```

### 1.2 各段详细规范

#### [1] 素材角色声明段（开篇 1 段 · 必加）

**作用**：开篇就说"谁是什么/参考什么"，再进入分镜。

```
@image1 as the only visual reference for the entire video,
children's picture book 2D paper-cut collage style,
soft pastel palette of mint green, cream, white, and warm yellow,
paper texture and torn edges clearly visible.
```

**关键点**：
- ✅ `@image1` 明确角色（`reference_image`）
- ✅ 风格一次性说清（防止漂移）
- ❌ 不要直接进分镜（缺素材角色声明 = 官方 doc2 §6 案例对比中 v14 缺失的项）

#### [2] 主体定义段（开篇 1 段 · 必加 · 官方 §4.2 支持）

**作用**：给主角起名字标签，每次提及都绑同一标签。

```
主体定义：将图片1中的小兔子定义为<主角小兔>。
```

**官方支持**（doc2 §1 主体定义 · line157）：
> "将图片1中穿红色连衣裙、戴草帽的女人定义为**张红**（**注意：可以给角色起名字**——我们之前从未给主角命名）"

**关键点**：
- ✅ 主角 = **物种名 + 个性名**（小兔子 → 主角小兔），不是抽象"小兔子"
- ✅ 后续每次提主角都用 `<主角小兔>` 标签（doc2 §4.3 第 1 条红线）
- ⚠️ 绘本项目**不踩"双胞胎"坑**（doc2 §8.2 —— 多角色场景才需要人名区分）

#### [3] 多镜头时间线段（⭐ 核心 · 导演思维的灵魂）

**作用**：按事件分镜，每镜头齐全 **运镜 + 动作 + 位置 + 音频** 4 逻辑（官方 doc2 §1.2 原话）。

**格式**：
```
镜头 1（建立 · 0-Xs）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
镜头 2（单词动作 · X-Ys）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
镜头 3（收势 · Y-Zs）：<景别><运镜 1 种>，<主体动作>，<位置/空间>，<音频内联>。
```

**4 逻辑**（官方 doc2 §1.2 原话 · 缺一不可）：

| # | 顺序 | 内容 | Rabbit 范例 |
|---|---|---|---|
| 1 | **运镜或镜头切换方式** | "全景缓慢推近""固定机位""镜头切至…" | "中景拉远定格""镜头切到侧面中景特写""缓推回到正面中景" |
| 2 | **主体动作与表情** | 核心角色/物体的关键动作、神态变化 | "<主角小兔>缓缓从草丛里探出半个脑袋，胡须轻颤，眼睛眨了两下" |
| 3 | **位置或空间变化** | 主体所处的场景、位置或空间关系 | "草丛""侧面中景""花田中央" |
| 4 | **音频信息** | 音效、人声或背景音乐（**每个镜头内联**，不集中末尾） | `<一声轻巧的"叮"，像铃铛一响>` |

**镜头数算法**（v15 + 官方 · 实际可执行版）：

| 时长 | 旁白密度 | 推荐镜头数 | 节奏公式 |
|---|---|---|---|
| **5s** | 1 短句（5-7 字） | **2-3 镜头** | 1.5s 建立 + 1.5s 单词动作 + 2s 收势 |
| **8s** | 1 短句 | 3-4 镜头 | 2s 建立 + 1s 跃入 + 3-4s 单词 + 1s 收势 |
| **12s** | 3 词家族 | 5 镜头 | 2-1-3-3-3（Cat 4a v3 标版） |
| **14s** | 5 词家族 | 5 镜头 | 2-1-3-3-3-3（Pic4 v2 标版，5 词 IT 家族） |
| **15s** | 6+ 词 | 6-7 镜头 | 2-1-3-3-3-3-...（Pic7 Horse OR 家族实测） |

**镜头数判定口诀**：
- **建立镜头 = 1**（必加，交代场景）
- **单词/事件镜头 = N**（每词/每事件 1 镜头，2-3s/镜头）
- **收势镜头 = 1**（必加，定格末态）
- **总镜头数 ≤ 时长 ÷ 1.5s**（每镜头至少 1.5s，少于则糊）

**反模式**：
- ❌ 5s 5 镜头 = 每镜头 1s（Rabbit 旧版踩过）→ 镜头太碎糊
- ❌ 5s 1 镜头 = 单推镜头无切镜（v6 简化版踩过）→ 故事性 = 0
- ❌ 按秒数硬切（"0-2s/2-4s/4-6s"）→ 官方 doc2 §1.3 红线"模型对精确时间支持不稳定"
- ✅ **按事件切**（建立/单词/收势）—— 时长让模型自己安排

#### [4] 整段音频约束段（Pic8 Rabbit 沉淀 · 2026-06-10）

**作用**：明示 BGM 红线 + 音效写 `<...>` + TTS 占位。

```
全程无背景音乐、无旁白人声、无哼唱、无歌唱。
保留 TTS 音轨占位，时长匹配旁白朗读时长（X 秒）。
```

**音效特殊字符规范**（官方 doc2 §440-451 原话）：

| 信息类型 | 符号 | 范例 |
|---|---|---|
| **音效** | `<>` | `<远处传来狗叫声>` |
| **音乐** | `（）` | （背景中播放着快节奏的摇滚乐） |
| **台词** | `{}` | {你好，世界} |
| **字幕** | `【】` | 【第一章：启程】 |

**绘本项目音效策略**（Pic8 沉淀）：
- ✅ 音效每个镜头内联（不集中末尾）
- ✅ BGM = 绘本红线（Pic8 用户原话："不要乱发挥，不生成 BGM 是红线"）
- ✅ CLI 参数 `--generate-audio true`（让 seedance 生成音效）

**反模式**：
- ❌ "环境音效" "声音" "BGM" 全纯文字（缺符号规范 = 模型理解模糊）
- ❌ 双重否定"无任何背景音乐" + 又写 BGM 描述（Pic8 v1 翻车 → 完全静默）

#### [5] 画面禁令段（必加）

```
画面保持无字幕、无水印、无 Logo。
镜头全程严格遵守"1 镜头 1 种运镜"红线（不堆叠推拉摇移）。
```

**关键点**：
- ✅ "1 镜头 1 种运镜" = 官方 doc2 §4 原话"不要同时要求推拉摇移，会增加画面的不稳定性"
- ✅ 无字幕/水印/Logo = seedance 防幻觉

#### [6] Storyboard 引导句（末尾 · 必加）

```
This is a storyboard reference image sequence, designed for picture book
reading - viewers should clearly see the rabbit scene unfold with gentle
micro-animations, holding the final pose for natural reading rhythm.
```

**作用**：让 seedance 看图自行设计微动效果，不写具体动效（Cat 范本精神）。

## 2. 运镜术语库（直接抄进 prompt）

> 官方 doc2 §127-141 + director-mode.md line131-141 · **模型认识以下术语**：

**镜头运动**（**1 镜头只 1 种**，红线）：
- `tracking shot` — 摄像机跟随主体移动
- `dolly zoom` — 后退+镜头推进（眩晕效果/希区柯克变焦）
- `push in / pull out` — 推进/拉远
- `whip pan` — 快速摇镜
- `crane shot` — 升降镜头
- `360° rotation` — 360度旋转
- `steady tracking shot` — 平稳跟拍
- `fixed shot` — 固定机位
- `缓慢推镜` / `平稳横移` / `固定镜头`（中文 seedance 也认）

**景别**（绘本项目最常用）：
- `Extreme close-up` — 特写（眼睛、嘴巴等细节）
- `Close-up` — 脸部特写
- `Medium close-up` — 头部+肩部
- `Medium shot` — 腰部以上
- `Full shot` — 全身
- `Wide / Establishing shot` — 环境全景
- `Bird's eye` — 俯视镜头
- `First-person POV` — 主观镜头

**绘本推荐组合**（Rabbit 验证）：
- 建立镜头：`中景拉远定格` / `全景推入`
- 单词/事件镜头：`侧面中景特写` / `正面近景`
- 收势镜头：`缓推回到正面中景` / `拉远至全景`

## 3. 动作量化公式（官方 doc2 §3 · 4 条铁律）

### 3.1 肢体细化 + 程度量化

> 官方原话："动作需具体到手、腿、头部、肩背等肢体部位，同时补充**幅度、速度、力度**描述"

**反例**：`小兔子动了`
**正例**：`缓缓抬手到胸口高度，胡须轻颤，眼睛眨了两下`

### 3.2 优先低缓连续小动作

> 官方原话："优先选用缓慢、轻柔、连贯的细微动作，**尽量规避狂奔、大跳、剧烈翻滚等高爆发、大动态动作**"

**绘本项目**：绘本本来就是"温柔讲故事"—— 优先用"缓缓""微微""轻轻"等词。

### 3.3 补充动作过渡衔接（⭐ 官方强调）

> 官方原话："写明前后动作的**惯性与承接关系**，保证画面动作连贯自然"

**范例**：
- ❌ "兔子抬头 + 兔子转头"（无承接）
- ✅ "**借着转身惯性顺势抬头**"（有承接）

**绘本应用**：每个镜头 2 切换时，**镜头 2 开头承接镜头 1 末尾**——"从停顿状态自然过渡到举手"。

### 3.4 情绪具象外化表达（⭐⭐⭐ 官方"情绪字典"是金矿）

> 官方原话："用具体的身体细节表现情绪，替代 '很悲伤' '非常愤怒' 这类抽象词汇"

| 抽象情绪 | 外化为动作与细节（官方原话） |
|---|---|
| **悲伤** | 低头、肩膀微微颤抖、眼眶泛红、手指无意识地攥紧衣角、泪水在眼眶里打转但没有落下 |
| **喜悦** | 嘴角抑制不住地上扬、眉眼舒展、脚步变得轻快、下意识地哼起小曲、忍不住原地转个圈 |
| **紧张 / 焦虑** | 频繁地看手表、手指不停敲击桌面、呼吸急促、眼神闪躲、无意识地啃咬指甲 |
| **愤怒** | 双拳紧握、下颌线紧绷、胸口剧烈起伏、眼神如刀般锐利、从牙缝里挤出话语 |
| **释然** | 长长地舒了一口气、紧绷的肩膀完全放松下来、脸上露出久违的、淡淡的微笑、抬头望向远方 |

**绘本项目应用**：绘本主角"高兴/好奇/失落"等情绪**直接查这张字典挑细节**，**不写 "happy" "sad" 这种抽象词**。

## 4. 完整 prompt 模板（Rabbit newclip1 v2 验证通过 · 1657 字符 · 5s · 3 镜头）

```
@image1 as the only visual reference for the entire video, children's picture book 2D paper-cut collage style, soft pastel palette of mint green, cream, white, and warm yellow, paper texture and torn edges clearly visible.

主体定义：将图片1中的小兔子定义为<主角小兔>。

整段视频呈现"找一找"主题。旁白朗读："小兔子 rabbit，藏在哪里？"。参考图原有的"rabbit"与"兔子"字样作为画面元素自然融入场景，不重新生成任何新文字。

镜头 1（建立 · 0-1.5s）：中景拉远定格，画面中央的草丛只露出<主角小兔>的两只长耳朵，纸艺拼贴的质感和纸纹清晰可见，<远处有微风轻轻吹过的声音>。
镜头 2（单词动作 · 1.5-3s）：镜头切到侧面中景特写，<主角小兔>缓缓从草丛里探出半个脑袋，胡须轻颤，眼睛眨了两下，<一声轻巧的"叮"，像铃铛一响>。
镜头 3（收势 · 3-5s）：缓推回到正面中景，<主角小兔>从草丛完全钻出坐在花田中央，前爪轻轻放下，<主角小兔>抬头望向镜头方向微笑，<远处鸟叫声渐弱>，画面定格在<主角小兔>微笑的最后一帧。

全程无背景音乐、无旁白人声、无哼唱、无歌唱。保留 TTS 音轨占位，时长匹配旁白朗读时长（5 秒）。
画面保持无字幕、无水印、无 Logo。镜头全程严格遵守"1 镜头 1 种运镜"红线（不堆叠推拉摇移）。

This is a storyboard reference image sequence, designed for picture book reading - viewers should clearly see the rabbit scene unfold with gentle micro-animations, holding the final pose for natural reading rhythm.
```

## 7. 一句话 TL;DR

**镜头设计 = 服务于故事、服务于叙事**。5s 3 镜头（建立/单词/收势）+ 主体命名标签 + 1 镜头 1 运镜 + 音效 `<>` 包裹 + 动作量化 + 情绪具象化 = 绘本镜头设计 6 大要素。**不要单图 + 整段推镜头 = 死板无故事性**。
