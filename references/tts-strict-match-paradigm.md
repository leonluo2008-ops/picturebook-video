# TTS 严格匹配范式 · 视频总时长 = TTS(不加冗余)

> **核心命题**:用户给 TTS 时,**视频总时长 = TTS 严格匹配**,不额外加 5s 冗余,不加末帧静默。

适用场景:**所有"用户提供 TTS 数值的视频创作场景"**(绘本视频 / 漫剧分镜 / 教学视频 / 短视频脚本 / 任何"声音-视觉"对齐场景)。

## 2. 判定优先级反转(关键)

**默认**(用户没说):视频总时长 = TTS + 5s 冗余(= 整数化代价 + 末帧微动空间)
**用户原话"按 TTS 严格算"/"TTS 已经有冗余"/给具体数字**:视频总时长 = TTS 严格匹配(不加 5s 冗余)

| 用户原话 | 采纳方案 | 反模式 |
|---|---|---|
| "TTS 36s" + 不加话 | TTS + 5s = 41s(默认) | ❌ 用户说"TTS 有冗余"还 + 5s |
| "TTS 36s + 已经有冗余" | 36s 严格(不加 5s) | ❌ 强行 + 5s = 多余 5s = 节奏拖沓 |
| "38s"(具体数字) | 38s(用户已给最终值) | ❌ 算"装得下" 41s = 偏多 3s |

---

## 3. 每 Clip 拟时长公式(整本节奏派生)

**当视频总时长 = TTS 严格匹配时**,每 Clip 拟时长:

```
拟时长 = 段 TTS 真实值(向上取整到 4-15 整数) + 0.5s 末帧微动空间
```

**示例**(2026-06-15 Seal 海豹 · TTS 36s · 1.6/4.0 算的段 TTS 总和 35.15s):

| Clip | 段 TTS 真实值(1.6/4.0) | 拟时长(整数 + 0.5s 余量) |
|---|---|---|
| 1 | 10.26s | 11s |
| 2 | 4.38s | 5s |
| 3 | 5.25s | 6s |
| 4 | 9.51s | 10s |
| 5 | 5.75s | 6s |
| **总** | **35.15s** | **38s**(整数化代价 2.85s) |

**38s ≈ 用户给 38s** ✅ = 严格匹配 TTS + 整数化代价

**整数化代价** = 整数时长必须向上取整造成的 1-3s 余量,**不是额外加的静默**。

**反模式**:
- ❌ 4s 凑 8s(为了"显得饱满")= 多余 4s = 节奏拖沓
- ❌ 段 TTS 5.25s 用 7s(向上多取整 2s)= 多余 2s = 节奏拖沓
- ❌ 整数化代价 > 5s(说明速率算错了,不是"额外加冗余")

## 7. 速查(一眼能用)

**Step 3 算视频总时长时**:

```
用户给 TTS = 36s
├─ 用户原话"按 TTS 严格算"/"TTS 有冗余" / 给具体数字(38s)?
│   ├─ 是 → 视频总时长 = 36s(不加 5s 冗余) = 用户原话优先
│   └─ 否 → 视频总时长 = 36s + 5s 冗余 = 41s(默认)
└─ 整本节奏派生:每 Clip 拟时长 = 段 TTS(向上取整整数) + 0.5s 末帧微动空间
   └─ 整数化代价 1-3s(在 5s 容忍内) = 不算额外冗余
```

**反问自检**(写完拟时长表必跑):
- [ ] 用户给具体数字(38s)? → 视频总时长必须 = 38s,不能"装得下"算 41s
- [ ] 用户说"TTS 有冗余"? → 视频总时长 = TTS,不加 5s
- [ ] 整数化代价 > 5s? → 速率算错了,重算
- [ ] 末段凑整 8s? → 真实 TTS = 多少秒就多少秒(铁律 #18)

## 8. 反模式(必避)

| 反模式 | 后果 | 修复 |
|---|---|---|
| ❌ **用户说"TTS 有冗余"还 + 5s** | 实际比 TTS 长 = 多余空镜 = 节奏拖沓 | 视频总时长 = TTS 严格匹配 |
| ❌ **用户给 38s 还按"装得下"算 41s** | 偏多 3s = 节奏拖沓 | 视频总时长 = 用户给具体数字 |
| ❌ **段 TTS 5.25s 凑 7s** | 多余 1.75s = 节奏拖沓 | 拟时长 = 5.25s 向上取整 6s = 整数化代价 0.75s |
| ❌ **末段凑整 8s** | 多余 2-3s = 节奏拖沓 | 末段 = 真实 TTS 整数秒 |
