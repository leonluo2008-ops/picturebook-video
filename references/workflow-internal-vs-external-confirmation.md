# 流程内不确认 · 流程外才让用户介入（**用户硬性元偏好 · 2026-06-16 验证**）

> **用户原话**（爷爷 Grandpa 绘本）："**看图你不要跟我确认，直接全看**"

---

## 核心命题

> **拿到批量输入（多图 / 多文件 / 多数据）= 必 1 次性全部处理完 = 流程内部不向用户确认 = 只有"流程外"节点（提交后目检 / 不可逆决策）才让用户介入。**

**底层逻辑**（用户元偏好）：

```
1. 用户来 = 让我做事，**不是让我做问卷**
2. 流程内部节点 = 可逆程度高 = 我自己拿主意 = **不确认**
3. 流程外节点（提交后目检 / 不可逆决策）= 不可逆 = **等用户拍板**
4. "我打算 X 因为 Y" ≠ "OK 吗？"（报告 ≠ 询问）
```

---

## 禁止反模式（必避）

- ❌ 看完前 3 张后停下来发"已 vision 3 张，是否继续看剩下 5 张？"
- ❌ 看完一批后发"是否开始写 prompt？"
- ❌ 任何把"流程内部节点"伪装成"决策点"反问用户的行为
- ❌ 1 张图 vision 后问"这张是什么风格？"
- ❌ "我打算 X 因为 Y"式询问（应该改成"我做了 X，因为 Y"+ 直接发结果）
- ❌ 流程跑得通但中途问"接下来是否继续？"

**根因**：用户已经给了输入 + 给定了 4 件事需求（简介/图/平台/TTS）= **信任 agent 跑完**。中途打断 = 把流程节点决策权硬塞回给用户 = 增加用户负担 + 减慢进度 + 违背用户偏好。

---

## 正确范式

```
收到 N 张图（或 N 个文件 / N 段数据）
   ↓
1 次性全部处理完（按 ≤ 3 张/批拆批，**不**在批间跟用户确认）
   ↓
直接写到产物文件（image-inventory.md / json / 任何业务文件）
   ↓
直接进下一步（不确认）
   ↓
跑完整条流程
   ↓
**流程外节点**（提交视频后目检 / 不可逆决策）= 这才是用户该介入的节点
   ↓
发结果 + 等用户目检反馈
```

---

## 判定口诀（**什么时候该跟用户确认？**）

```
1. 我要确认的事 = 流程内还是流程外？
   ├─ 流程内（vision / TTS / Clip 划 / prompt 写 / 数据处理）→ 不确认 · 直接做
   └─ 流程外（提交后目检 / 不可逆决策 / 删数据 / 关服务）→ 等用户拍板

2. 我要确认的事 = 可逆还是不可逆？
   ├─ 可逆（再跑一个 Clip / 改 prompt / 重算）→ 直接做 + 报告
   └─ 不可逆（删业务数据 / 关服务 / 删文件）→ 一次性问清楚

3. 我有没有"问错了的代价"？
   ├─ 问错了代价低（凭印象写 1 段 prompt）→ 不问
   └─ 问错了代价高（删业务数据 / 错拍决策）→ 一次性问
```

**简化口诀**：

- 流程内 = **不问**
- 流程外 = **等目检**
- 可逆 = **不问 + 报告**
- 不可逆 = **一次性问**

|---|---|

| **漫剧分镜** | 收到 N 个角色图 = 1 次性 vision 全 N 个 = 不分批确认 · 全部镜头序列 + 全部 prompt 一次写完 |

| **通用批量输入** | 任何"拿到 N 个 X 一次性处理"的场景 = 本元方法适用 |

## 实战案例（2026-06-16 爷爷 Grandpa 绘本）

**用户输入**：`爷爷 Grandpa.zip`（8 张图 + 1 个 readme.txt · 8 段中英双语旁白 · TTS 32s）

**我之前犯的错**（被用户纠错）：

```
Step 2.1 vision 第 1 批 3 张
  ↓
发消息"看图你不要跟我确认，直接全看" ← 用户纠错
```

**正确做法**（用户元偏好）：

```
Step 2.1 vision 第 1 批 3 张（不跟用户确认）
   ↓
直接进第 2 批 3 张（不跟用户确认）
   ↓
直接进第 3 批 2 张（不跟用户确认）
   ↓
8 张全看完 + 写到 image-inventory.md
   ↓
直接进 Step 3 算 TTS（不确认）
   ↓
直接进 Step 4 划 Clip（不确认）
   ↓
直接进 Step 5 写 5 个 prompt（不确认）
   ↓
Step 5.5 跑 4 项手动 grep 自检（不确认）
   ↓
Step 6 提交 Clip 1 → 发飞书（**这才是用户该介入的节点**）
   ↓
等用户目检反馈
```

**判定**：本绘本 8 张图 + TTS 32s 全部 = **流程内节点** = agent 自己拿主意跑完 = 不打断用户 = **直到提交第 1 个 Clip 才让用户介入（目检）**。

## 反模式（必避）

| 反模式 | 触发 | 修复 |
|---|---|---|
| ❌ "已 vision 3 张，是否继续？" | 拿到 N 张图分批后问 | 1 次性 vision 全 N 张 · 中间不问 |
| ❌ "接下来是否开始写 prompt？" | 流程跑完一个节点问 | 跑完整条流程 · 提交后让用户介入 |
| ❌ "我打算 X 因为 Y，OK 吗？" | 报告伪装成询问 | 改成"我做了 X，因为 Y" + 直接发结果 |
| ❌ "你的 TTS 32s 我看下来可能要 35s，是否加 5s 冗余？" | 把"流程内可逆"决策问用户 | 自己做 = TTS 32s + 5s 冗余 = 37s · 报告 + 跑 |
| ❌ "我先跑 1 个 Clip 试试水？" | 把"目检节点"伪装成"流程内" | 1 个 Clip 跑完直接发飞书 = 让用户目检 = **这才是用户该介入的节点** |

## 自检命令（写完流程节点后必跑）

```bash
# 1. 流程内不向用户确认的检查 = 产物文件应一次写完整
ls -la /tmp/<project>/image-inventory.md
# 期望：一次写完 = 包含全部 N 张图的 5 项 + 有/没有两列
# 失败 = 多次写入 = 中间跟用户确认过

# 2. 用户介入节点检查 = 飞书消息历史
session_search query="绘本视频" limit=3
# 期望：每本绘本 = 1-2 条用户消息（输入 + 反馈）· 不是 5+ 条每节点确认
```
