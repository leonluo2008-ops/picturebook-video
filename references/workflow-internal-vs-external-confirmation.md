# 流程内不确认 · 流程外才让用户介入

> **跨 skill 通用元方法论** — 绘本视频 / 漫剧分镜 / 账号运营 / 任何批量输入处理任务
> **首次沉淀**：2026-06-16 picturebook-video skill 沉淀期
> **第 1 次跨本验证**：2026-06-16 爷爷 Grandpa 绘本（v4.0 首秀）
> **第 2-3 次待验证**：需 ≥ 3 次跨场景才能升 SKILL.md 铁律（按元教训 M）

## 核心命题

```
用户给我 N 张图 + 旁白 + TTS
   ↓
我在流程内 = 全跑（vision N 张 / image-inventory / 叙事弧 / 时长 / 5 项必查）
   ↓
流程边界 = 提交方案给用户
   ↓
用户目检 / 反馈 → 我继续
```

**关键区分**：
- ❌ 流程内问用户"我接下来要 vision 几张" / "要不要先写 image-inventory" = **打扰用户** = 反模式
- ✅ 流程边界（"Clip 1 跑完，要不要我连跑剩下 4 个"）= 拍板项 = **问** = 正常
- ✅ 流程边界（"Clip 1 跑完，结果如何"）= 报告 = **不**问 = 正常

## 触发器

读到以下场景立即警觉：

```
"接下来要不要 / 能不能 / 是不是先..."
"我打算 X 因为 Y，你 OK 吗"
"需要我先做 X 还是先做 Y"
```

**判断流程**：
1. 这是流程内还是流程边界？→ 流程内 = **不问** = 全跑
2. 我问的这个问题是不是"询问"伪装成"报告"？→ 是 = 反模式
3. 是不是"怕你不同意"才问？→ 是 = 反正模式

## 决策树

```
用户在流程中问我"要不要 X / 接下来做什么"？
├─ 是 → 一次性报方案（不 = 问 · 是 = 报告）
│       "我打算 X 因为 Y，跑下去 OK"
└─ 否 → 流程内 = 全跑 + 流程边界（= 提交/目检/换方向）= 问

我已经做了 N 步，要不要问"对吗"？
├─ 是 → 流程边界 = 报告（"做了 X / Y / Z · 下一步打算 W"）
└─ 否 → 流程内 = 不回头看 = 继续
```

## 实战对照表

| 场景 | 反模式 | 正模式 |
|------|--------|--------|
| 收到 8 张图 | "我打算分 3 批 vision 你 OK 吗" | 直接 vision 3 批 + 写 image-inventory |
| 写完 image-inventory | "清单写好了，要继续吗" | 继续标叙事弧 + 合并 Clip |
| 写完 prompt | "5 个 prompt 都写好，要提交吗" | 提交 Clip 1 + 报告 |
| Clip 1 跑完 | "Clip 1 OK，要不要连跑剩下 4 个" | 报告时长/md5 + **问**（拍板项 = 流程边界）|
| 5 个 Clip 都跑完 | "全部跑完，要不要发飞书" | 直接发飞书 + 报告 |

## 跟其他元方法论的关系

- **元教训 J 主动追问 = 我对自己问**：内部推理，不打扰用户
- **元教训 N 冲突时降级**：本方法论 = 跟 "superpowers prefer multiple choice" 不冲突（superpowers 推荐的是 brainstorm 阶段列 2-3 方案，不是流程内每步问）
- **元教训 M 反模式不擅自加**：本方法论目前是 references/ 层（跨场景 1 次验证）= **不升 SKILL.md 铁律** = 等 ≥ 3 次跨本验证

## 验证记录

### 第 1 次 · 2026-06-16 爷爷 Grandpa 绘本（picturebook-video）

**用户原话**："**看图你不要跟我确认，直接全看**"

**执行**：
- 我原本打算在 vision 前问"分 3 批还是 4 批"= 立即纠错
- 直接 vision 8 张（3+3+2 串行）+ image-inventory 一气写完 + 叙事弧 + 5 Clip 合并 + v8 prompt 5 个
- 流程边界 = 提交 Clip 1 跑完后**才**问"要不要连跑剩下"

**结论**：流程内全跑 = 用户满意。**但是用户提醒**才触发 = 反元教训 D "装了没用" = 下次开局就要装这个反射

# 流程边界 = 拍板项 = 问（一次问完，不反复）

# 流程内 = 0 问
# 流程边界 = ≤ 3 拍板项（superpowers 推荐）

# 反模式自检
# "我接下来打算 X 因为 Y" 是 = 报告 不是 = 询问
