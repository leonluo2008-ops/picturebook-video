## 4. bird 特殊配色处理（en_color_pattern → 流畅词组）

### 4.1 原图视觉特征

每页都有顶部双语标题：
- 英文 **bird**（小写）：**b=鲜艳红色 / i=鲜艳蓝色 / r=鲜艳红色 / d=鲜艳蓝色**（红蓝交替）
- 中文 **鸟**（单字）：**鲜艳橙色**

### 4.2 C 子 agent 输出结构

C 子 agent 在 `text_position.en_color_pattern` 填了**结构化数据**：

```json
{
  "en_color_pattern": {
    "b": "鲜艳红色",
    "i": "鲜艳蓝色",
    "r": "鲜艳红色",
    "d": "鲜艳蓝色"
  },
  "zh_color": "鲜艳橙色"
}
```

### 4.3 主 agent 脚本清洗

**Pic4 翻车教训**（铁律 #62）：C 输出的结构化数据不能直接拼进 prompt。

**Bird 清洗函数**：

```python
def _parse_bird_color(en_color_pattern):
    """把 {"b": "鲜艳红色", "i": "鲜艳蓝色", ...} 转成 'b=鲜艳红色 / i=鲜艳蓝色' 流畅词组"""
    if not en_color_pattern:
        return "红蓝交替配色"
    parts = []
    for k, v in en_color_pattern.items():
        v = v.rstrip('色')  # 去掉末尾"色"避免重复
        parts.append(f"{k}={v}色")
    return " / ".join(parts)
```

**实测输出**：
- 8 段全部清洗为 `b=鲜艳红色 / i=鲜艳蓝色 / r=鲜艳红色 / d=鲜艳蓝色`
- 0 段出现 `b=鲜艳红色 / i=鲜艳蓝色 / r=鲜艳红色 / d=鲜艳蓝色 英文` 语法断裂

### 4.4 启示

| 数据类型 | C 输出 | 主 agent 必清洗 |
|---|---|---|
| 颜色（结构化）| `{"b": "鲜艳红色", ...}` | `_parse_bird_color` → `b=鲜艳红色 / i=鲜艳蓝色 / ...` |
| 颜色（简单）| `"红色"` | 加 "色" 字结尾 → "红色色" |
| 位置 | "画面顶部正中央" | **已是流畅词组** · 不清洗 |
| 表情 | "眉眼弯弯+嘴角上扬+慈爱温柔" | **已是流畅词组** · 不清洗 |

**新约定（铁律 #62 强化）**：C 输出的所有"颜色/位置/特征"字段都应该是**句法片段**（可直接拼入句子的词组），不是**键值对**。**判断口诀**：**"C 产原料 ≠ prompt 词组 · 主 agent 必清洗"**

## 5. 8 task 并行轮询（铁律 #50 边界优化）

### 5.1 实测批次策略

| 批次 | task 数 | 提交方式 | 轮询方式 | 实测耗时 |
|---|---|---|---|---|
| 验证 | 1 (clip1) | 单 task | 同步 25 轮 | ~2 min |
| 批量 1 | 2 (clip2+3) | 串行 | 后台并行 30 轮 | ~1 min |
| 批量 2 | 2 (clip4+5) | 串行 | 后台并行 30 轮 | ~1 min |
| 批量 3 | 2 (clip6+7) | 串行 | 后台并行 30 轮 | ~1.5 min |
| 批量 4 | 1 (clip8) | 单 task | 后台并行 30 轮 | ~1.5 min |
| **总** | **8** | **8 串行提交 + 8 并行轮询** | — | **~7 min** |

### 5.2 8 task 并行 vs 2/批 顺序

| 策略 | Pic5 实测 | Pic4 v6 实测 |
|---|---|---|
| 2/批 × 4 批 + 顺序轮询 | — | ~15 min（4 批顺序）|
| **8 串行提交 + 8 并行轮询** | **~7 min** ✅ | — |

**结论**：**8 task 并行轮询比 2/批 顺序快 50%**。

### 5.3 铁律 #50 边界（关键）

**8 task 并行可行的边界**：
- ✅ D 阶段 = 提交 + 轮询 + 下载（**不抽帧+不 vision**）
- ❌ D + 抽帧 + vision = 8 并行会撞 600s timeout（Pic2 v1.0.0 教训）

**新约定（铁律 #85 候选）**：
- D 仅做生成（不抽帧不 vision）= 8 并行可行
- D + 抽帧 + vision = 仍走 ≤2/批
- 8 task 并行轮询 shell 模板：

```bash
poll_task() {
  local TID=$1
  for i in $(seq 1 30); do
    sleep 15
    STATUS=$(python3 seedance.py status $TID 2>&1 | grep -oE '"status":\s*"[a-z]+"' | head -1 | grep -oE 'succeeded|failed|running|queued')
    if [ "$STATUS" = "succeeded" ] || [ "$STATUS" = "failed" ]; then
      return 0
    fi
  done
}
for TID in ${TASK_IDS[@]}; do
  poll_task $TID > /tmp/poll_${TID}.log 2>&1 &
done
wait
```
