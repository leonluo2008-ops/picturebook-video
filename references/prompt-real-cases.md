### 修复 2 · 角色指代对齐子代（段 2 约束第 2 行）

**v1 翻车**：镜头一/二/三混合写 "小兔子@图1" + "小兔子@图2" → **主语乱用** → 模型自由发挥。

**v2 修复**：

**Pic 5 Clip 全部按此规则**：镜头主语的 @ImageN **严格等于该镜头对应的图**。

### 修复 3 · 铁律 #39 领读节奏元原则（用户原话驱动 · 2026-06-06）

**用户原话**（2026-06-06）：
> "v15 标准节奏这个我记得是有用的，这个节奏是根据领读绘本的 TTS 节奏，让领读保持呼吸感。根据旁白的内容，由 AI 确定合适的领读节奏，原则是让听众有足够的时间去消化，这是一个灵活的处理原则，而不是固定的框架。"

**用户原话**（续）：
> "铁律的使用就是最后的静默那个是，也是根据故事的情节来，比如说最后需要有停顿，让画面慢慢的消失，静默让观众慢慢的退出的感觉，其实所有的故事画面都是按照这个原则来处理的，而不是说一定怎样，原则就是根据故事情节，让画面有足够的时间，让听众和观众去感受消化，而不是戛然而止。"

**v15 标版节奏 ≠ 硬规则**（这是用户明确说法的核心）：
- 之前 SKILL.md 写"末帧静默 2-3s"是**常见情况默认**
- 新铁律 #39 = **元原则**（主导）：节奏灵活，**根据故事需要**，**不戛然而止**
- 数字标版（2-3s / 2-1-3-3-3）= **参考**，**实际操作按元原则灵活调整**

**Pic 绘本 5 Clip 末帧全部 = 5s 退场感**（按 #39 元原则）：
- 故事主题 = 温暖午后 + 友好问候 + 友谊收束 → **节奏舒缓、末帧 5s 退场感**
- 不是默认 2-3s 也不是 0.5s 收势 → **5s 退场感**（"让观众慢慢退出"）
- 跨 5 个 Clip 一致（绘本"调性"统一）

**沉淀**：铁律 #39 + 铁律 #36 降级为"默认 2-3s，按 #39 灵活调整"。

### 修复 4 · v3 字保留强约束（Pic Clip 2 v2 翻车驱动 · 2026-06-06）

**Pic Clip 2 v2 翻车**（铁律 #14 v2 措辞不够强）：
- v2 措辞："参考图原有的所有文字...作为画面元素自然融入场景"
- 双图 `--ref-images 3.jpg 4.jpg` 模式 → 末帧 11s 彩色字 **完全被吃掉**
- v2 措辞在单图首帧模式（`--image`）有效（Good Morning/Good Night 验证通过），**但在双图 / 单图首参模式不够强**

**v3 措辞升级**（Pic Clip 3/4/5 全部命中彩色字）：
```
参考图原有的所有文字（<列举具体文字>）必须完整保留作为画面元素，

```

**v2 vs v3 关键差异**：
| 维度 | v2 | v3 |
|---|---|---|
| "保留"语气 | "作为画面元素自然融入场景" | "**必须完整保留**作为画面元素" |
| 明确禁止 | 隐含"不要重画" | "**模型不得删除或替换**" |
| 列举具体文字 | 泛指"所有文字" | "**彩色字母Good afternoon + 棕色手写下午好**" |
| 适用 | `--image` 单图首帧 | `--ref-images` 双图/单图首参（绘本默认）|

## 5 Clip 节奏 2-1-3-5 模板（Pic 绘本范式 · 跨 Clip 一致）

```
节奏 2-1-3-5 = 2s 建场 + 1s 跃入 + 3s 朗读（朗读 1s + 静默 2s）+ 5s 末帧（朗读 1s + 静默 4s 退场感）
```

- Clip 2-5 11s = 2-1-3-5（1 词 + 末帧 5s）
- **末帧 5s 一致**（绘本调性：温暖退场感）
- **节奏模板 11s 跨 4 个 Clip 统一**（领读型 5s 末帧退场感是 Pic 绘本"调性"）

## 反模式 6 条（Pic 绘本全过程沉淀）

| ❌ 反模式 | 后果 | 正确做法 |
|---|---|---|
| 1. 写 `@图N` 中文别名 | 模型不识别，分镜绑定全失效 | 写 `@ImageN`（官方语法）|
| 2. 镜头主语不绑子代（@图1/2/3 混用）| 画面自由发挥、飘移 | 每个镜头主语 = 该镜头对应的图 |
| 3. 写"其他元素不出现"隔离句 | 元素增/减受限，违反剧情 | 元素增/减 = 故事剧情需要 |
| 4. 用 v2 字保留措辞（"自然融入"）| 双图模式末帧彩色字被吃 | 用 v3 措辞（"必须完整保留" + "不得删除"）|
| 5. 用 V13 范式分离"主体外观@ImageN + 场景@ImageM" | 视觉权重压过文本约束，@ImageM 末段视觉胜出 | 走铁律 #40 A 方案（`--image` 锁首帧）|
| 6. 主体一致性在 prompt 里二次声明（"同一个奶白色小兔子"）| 干扰视觉权重，冗余 | 段 1 主体定义对每张图各说一次，不写"同一个" |
