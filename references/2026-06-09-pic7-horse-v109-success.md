## 🎯 v1.0.9 A+B 混合镜头设计（成功的部分）

### 设计思路

**核心问题**：v1.0.8 纯 A 切镜头（7 镜头）= 每词给 1 个镜头（2.5s 朗读 + 2s 静默）= 14s 偏赶 + 5 词不充分

**解决思路**：每词 = 2 镜头（A 切 + B 持续）= 节奏更舒缓 + 给足消化时间

### 镜头设计（10 镜头）

| 镜头 | 类型 | 时长 | 作用 |
|---|---|---|---|
| 1 | 全景建立 | 2s | 6 卡片 + OR Family 一起亮起 |
| 2 | A 切 | 1s | horse 卡片轻柔浮现（1.1x + 边框柔光 + 1 次呼吸 0.5s）|
| 3 | B 持续 | 2.5s | horse 卡片脉动 + 马嘶鸣 + 朗读+静默 |
| 4 | A 切 | 1s | fork 卡片轻柔浮现 |
| 5 | B 持续 | 2s | fork 卡片脉动 + 叉子闪 + 朗读+静默 |
| 6 | A 切 | 1s | horn 卡片轻柔浮现 |
| 7 | B 持续 | 1.5s | horn 卡片脉动 + 号角响 + 朗读+静默 |
| 8 | A 切 | 1s | corn 卡片轻柔浮现 |
| 9 | B 持续 | 1s | corn 卡片脉动 + 玉米摆 + 朗读+静默 |
| 10 | A 切+末帧 | 1s | pork 卡片轻柔浮现 + 末帧定格 |

**A 切机制**：保证参考图准（每切 = 重新看图 = seedance 跟随）
**B 持续机制**：让每词有持续 2s 强调时间（柔光脉动 + 图标微动 + 朗读静默）

### 朗读节奏（轻压缩）

- horse 2.5s + fork 2s + horn 1.5s + corn 1s + pork 1s = 总 8s 朗读 + 5s 静默 = 13s + 1s 末帧共享 = 14s 装得下

## 🎯 给未来绘本/绘本项目的指导

### 1. 画面对齐就 1 条铁律

> **以参考图为基准 = 不约束 + 让 seedance 看图 = 任何镜头设计都 OK**

### 2. 镜头设计 = 任意合理描述都 OK

- ✅ A 切镜头（v1.0.8 现状）
- ✅ A+B 混合（v1.0.9 创新）
- ✅ 全景慢慢自然流动（用户原话）
- ✅ 弹跳/高亮/脉动/平移/旋转/缩放 = 任何动效都 OK

- 5+ 词家族 + 朗读 + 静默 + 弹跳 = 14s 装不下
- 走 Cat 4a/4b 拆 2 Clip = 节奏舒缓 + 给足消化

### 4. C agent 必考虑"以参考图为基准"（流程问题）

- C 看图时必填 image_structure（不是写具体 N 张 = 而是写名称列表）
