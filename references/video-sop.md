# video-sop - 即梦（Dreamina）视频创作标准工作流程

## 技能描述
即梦（Dreamina）视频创作标准工作流程，涵盖从创意发散到最终成片的9个完整阶段，支持**故事成片**（故事短片、动画故事、小说漫改、情感短片、电影感短片）和**营销广告**（产品广告、带货视频、品牌宣传、营销短片）两大类内容创作。

本SOP是所有视频创作的主流程框架，必须严格遵循阶段顺序执行，每个阶段完成后进入下一阶段，不允许跳步。

---

## 创作整体流程（9个阶段）

```
Phase 0: 创意发散 → 
Phase 1: 内容确认 → 
Phase 2: 内容大纲 → 
Phase 3: 剧本创作 → 
Phase 4: 素材挖掘 → 
Phase 5: 分镜设计 → 
  ↳ Step 1 分镜切分 (script-chunk)
  ↳ Step 2 分镜计时 (shots-timing)
  ↳ Step 3 分镜组合 (shots-assembly)
  ↳ Step 4 连贯性校验 (scene-reflection)
Phase 6: 需求确认 → 
Phase 7: 参考素材生成 → 
Phase 8: 分镜生视频 → 
Phase 9: 视频剪辑/成片 → 
```

---

## 各阶段详细说明

### Phase 0：创意发散
**适用场景**：用户只有一个模糊想法/主题/一句话灵感，没有具体内容
**调用技能**：
- 故事成片 → `story-idea`（故事短片大纲创作）
- 营销广告 → `ecom-idea`（营销广告创意构思）

**输出内容**：
- 故事成片：故事大纲方向+风格类型+情感主题
- 营销广告：3个不同创意方向供用户选择

**用户确认**：用户选定一个创意方向后进入下一阶段

---

### Phase 1：内容确认
**核心工作**：和用户确认以下关键信息
1.  目标总时长：1分钟/2分钟/3分钟
2.  视频宽高比：16:9/9:16/1:1/4:3等
3.  整体风格：写实/二次元/国风/科幻等
4.  其他用户特殊要求

**输出**：整理成《创作需求确认表》，用户确认无误后进入下一阶段

---

### Phase 2：内容大纲
**调用技能**：
- 故事成片 → `story-idea`（基于选定创意产出完整故事大纲）
- 营销广告 → `ecom-idea`（基于选定创意产出完整创意大纲）

**输出内容**：结构化完整大纲，包含：
- 故事概览
- 角色列表
- 三幕式结构/创意结构
- 主题/卖点

**用户确认**：用户确认大纲后进入下一阶段

---

### Phase 3：剧本创作
**调用技能**：
- 故事成片 → `story-script`（故事短片剧本创作）
- 营销广告 → `ecom-script`（营销广告剧本创作）

**输出内容**：完整可拍摄的剧本，包含：
- 基本信息（时长、风格、场景数）
- 角色/产品列表
- 分场景正文（场景+动作+对话）

**用户确认**：用户确认剧本后进入下一阶段

---

### Phase 4：素材挖掘
**调用技能**：`ref-extract`（原子素材识别 + 素材深化设计）

**输出内容**：结构化素材清单，包含：
- 角色列表（主体+变体）
- 道具/关键物品/产品列表
- 场景列表
- 每个素材标注主体来源（用户参考主体/用户普通参考图/待生成主体）

**用户确认**：用户确认素材清单后进入下一阶段

---

### Phase 5：分镜设计（四步走）
必须按顺序完成这四个步骤，不跳步：

#### Step 1：分镜切分 → `script-chunk`
- 将完整剧本切分为单个分镜
- 每个分镜标注：镜号、景别、画面描述、台词/音效

#### Step 2：分镜计时 → `shots-timing`
- 给每个分镜计算合理时长
- 汇总总时长，调整到目标范围内

#### Step 3：分镜组合 → `shots-assembly`
- 将分镜组合为4-15秒的Clip（AI生成硬约束）
- 每个Clip时长严格控制在4-15秒
- 保持场景/情节完整性

#### Step 4：连贯性校验 → `scene-reflection`
- 检查人物/道具/场景/逻辑/色调一致性
- 识别并修复不连贯问题
- 输出修复后的Clip表+一致性注意事项

**输出内容**：修复后的最终Clip表

---

### Phase 6：需求确认
将以下内容汇总给用户做最终确认：
1.  最终剧本
2.  素材清单
3.  分镜Clip表（含时长）
4.  总时长、宽高比、风格确认

**用户确认**：用户确认所有内容无误后进入下一阶段

---

### Phase 7：参考素材生成
**调用技能**：
- 故事成片 → `story-ref-gen`（故事参考素材生成）
- 营销广告 → `ecom-ref-gen`（营销参考素材生成）

**核心工作**：
- 按素材清单批量生成角色主体/变体、道具、场景参考图
- 所有素材并行生成，用户选择最终满意的版本

**输出结果**：用户确认后的参考主体库，每个主体有唯一ID

---

### Phase 8：分镜生视频
**调用技能**：`video-prompt`（分镜生视频）

**核心工作**：
- 将每个Clip翻译成AI可执行的视频生成Prompt
- 所有Clip并行调用视频生成工具一次性提交
- 严格遵守每个Clip4-15秒时长约束

**输出结果**：所有Clip视频片段生成完成

---

### Phase 9：视频剪辑/成片
**核心工作**：
- 使用 `video_editor` 工具将所有Clip按顺序拼接合并
- 智能生成匹配风格的背景音乐
- 输出最终完整成片

**输出结果**：完整可播放的最终视频

---

## 适用场景分类
| 内容类型 | 对应技能路径 |
|---------|-------------|
| 故事短片/情感短片/电影感短片 | Phase0(story-idea) → story-script → ref-extract → 分镜四步 → story-ref-gen → video-prompt → video_editor |
| 小说漫改/IP改编 | 同上，Phase0先提炼故事大纲 |
| 产品广告/带货短视频 | Phase0(ecom-idea) → ecom-script → ref-extract → 分镜四步 → ecom-ref-gen → video-prompt → video_editor |
| 品牌宣传/营销短片 | 同上 |
| 已有完整剧本直接生成 | 跳过Phase0-Phase3，从Phase4开始 |
| 单个15秒以内片段生成 | 跳过分镜组合，直接单Clip生成 |

---

## 核心规则
1.  **不跳步原则**：必须按阶段顺序执行，每个阶段需要用户确认后再进入下一阶段
2.  **并行生成原则**：同一阶段多个生成任务（多张参考图/多个Clip视频）必须一次性并行提交，不串行分批
3.  **时长硬约束**：每个AI生成的Clip必须在4-15秒之间，绝对不能超限
4.  **一致性原则**：参考主体生成后，所有后续生成必须保持一致性，不能随意变化
5.  **最小修改原则**：用户反馈问题时，只修改有问题的部分，不重新生成全部

---

## 单Clip直达规则（例外情况）
当且仅当同时满足以下两个条件，可以跳过完整SOP，直接生成：
1.  用户目标是生成**单个Clip**（单个4-15秒视频片段）
2.  需求已经足够明确，可直接生成

此时：
- 纯文字 → 直接 `text2video`
- 已有首帧图 → 直接 `image2video`
- 多参考素材 → 直接 `multi_modal2video`

---

## 检查清单（完成所有阶段后）
✅ 每个阶段都经过用户确认，没有跳过
✅ 所有素材都已生成并得到用户确认
✅ 每个Clip时长都在4-15秒范围内
✅ 所有一致性问题都已修复
✅ 所有Clip已经并行生成完成
✅ 最终拼接和背景音乐已完成

---

## 总结
本SOP是即梦平台视频创作的标准流程，从0到1覆盖了创意→大纲→剧本→分镜→素材→生成→成片全链路，只要严格遵循流程，就能稳定产出结构完整、风格统一、逻辑连贯的优质视频作品。
