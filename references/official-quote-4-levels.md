# 官方原话 4 档分级法（绘本实战沉淀 · 反过度推论）

> **本文件是引用 seedance 官方文档（doc1/doc2/doc3）的标准化方法**——任何 prompt 写法/prompt 优化的规则都从官方原文档找依据，但**引用前必标注语气级别**。**适用场景**：写任何绘本 / 视频 prompt 之前 / 修改 prompt 范式时 / 用户质疑某条规则时 / 跟子 agent 讨论 prompt 设计时。
>
> **核心问题**：agent 容易把官方"小字说明/建议"读成"红线/铁律"，包引号包装成"官方依据"强推给用户 = **过度推论 = 翻车**。

---

## 1. 元教训（根因）

agent 的两个高频错误模式：

1. **"建议→红线"升档**：把"建议/参考/可以"读成"必须/不能/禁止"
2. **凭印象引用**：编"doc2 §X.X 说……"但**实际没这话**（凭记忆复述 = 漏字/断章）

**根因**：agent 跳过了"先回原文档 grep 验证 + 标上下文"这一步，直接信"上次我查过"记忆。

---

## 2. 官方原话分 4 级（核心方法论）

| 级别 | 关键词 | 性质 | 违反后果 | 引用规范 |
|---|---|---|---|---|
| **L1 红线** | "必须/不能/禁止/红线/必须" | 硬约束 | 必然翻车 | ✅ 可直接当铁律推 |
| **L2 强建议** | "注意/应该/建议/推荐" | 软硬边界 | 大概率翻车 | ⚠️ 引用时标"建议"语气 |
| **L3 软建议** | "说明/参考/可以/推荐" | 可选 | 可能不稳定 | ⚠️ 标"参考语气"，**不当铁律** |
| **L4 描述** | "例如/比如/如" | 例子不是规则 | 例子 ≠ 规则 | ✅ 仅作举例 |

**判断口诀**：
- 原文带【说明/建议/参考/可以】= L3 软约束 = **可违反，违反效果可能差**
- 原文带【注意/必须/不能/红线】= L1 硬约束 = **必走，违反必然翻车**

---

## 3. 凡引用官方原话前必跑 3 问

**任何"官方说 X 是错的 / 必须 Y" 类结论前必跑**：

1. **官方原话真存在吗？**
   - 必先 `grep` 原文档验证
   - 必复制原文（不缩略）
   - 必标上下文（所在章节、行号、是否小字说明）

2. **官方语气是"建议"还是"红线"？**
   - 看关键词（必须/不能/禁止 vs 说明/参考/可以）
   - 标 L1-L4 级别
   - 上下文是"主要规则"段还是"其他技巧"段

3. **有实战翻车数据支持吗？**
   - 没数据 = **L3 软建议**，不当铁律用
   - 有数据 = 升 L1 红线档（但需验证数据真伪）

## 4. "按事件拆" vs "按秒数拆" 实战决策表（case study）

| 场景 | 写法 | 理由 | 级别 |
|---|---|---|---|
| **首次跑**（无翻车数据）| 按事件拆（`镜头 1：xxx`）| 官方首选 + 不堆风险 | L2 强建议 |
| **按事件拆翻车** | 改按秒数拆 | 显式约束模型在指定秒数做指定事 | L2 强建议 |
| **按秒数拆也翻车** | 回到按事件拆 + 1 种运镜 + 1 个动作 | "不约束不必要的细节"原则 | L1 红线（基础） |
| **按秒数拆成功** | 沿用 | 不为换而换 | — |

**绘本 clip1 v1 → v2 实战验证**：
- v1：按秒数拆（0.0-3.0s / 3.0-8.0s / 8.0-13.0s）+ 4 动作堆叠 → 末段抽搐
- v2：按事件拆（镜头 1/2/3）+ 每镜 1-2 动作 → 流畅

**翻车真凶 = 动作堆砌**（v1 一句塞 4 动作），**不是**"按秒数"。

**保守结论**：按事件拆是**首选 + 命中"少约束"思路**，但**不**等于"按秒数拆是错的"。**两种都允许**——选哪个看**实战效果**，不看官方语气强弱。

## 5. "1 镜 1 运镜" 漏字案（case study）

**官方原话**（doc2 §4 · 红线）：

> "**一个镜头里尽量只指定 1 种运镜方式，不要同时要求推拉摇移**"

**漏字误读**（agent 翻车模式）：把"运镜方式"读成"动作" → 推论"1 镜 1 动作" → 限制镜头叙事

**正解**：
- 1 镜头 = 1 种**运镜**（中景跟拍 / 切至 / 推近 / 横移）= L1 红线
- 1 镜头 = N 个**连贯动作**（点头→低头→抬头 是 1 镜 3 动作 OK）= 没限制

**漏字根因**：agent 用缩写记忆（"1 镜 1 ..."）= 漏"运镜"4 字 = 错全意。

**修复**：凡引用官方原话 = **完整 5 维属性**：
- 原文（不缩略）
- 位置（§X · 行号）
- 语气（关键词）
- 上下文（小字说明 vs 主要规则）
- 级别（L1-L4）

---

## 6. 反模式（必避）

- ❌ "**官方说按秒数拆是错的**"（L3 软建议当红线 + 强加因果）
- ❌ "doc2 §X.X 说……"——**没说 §X 真有这话**（凭印象引用）
- ❌ 4 动作堆砌翻车 → 甩锅给"按秒数"（**真因**是动作堆砌，**不是**秒数）
- ❌ "**官方说不要 X**"（泛泛否定）= 看上下文是 L1 还是 L3
- ❌ 缩写复述"1 镜 1 X"——X 必须 = 原文关键词

---

## 7. 实战 SOP（提交规则前必跑）

```
[准备写"必须 / 禁止 / 红线"类规则前]
   ↓
1. 必先回原文档 grep 验证原文
   ↓
2. 标 5 维属性（原文/位置/语气/上下文/级别）
   ↓
3. L3 软建议 → 降级为"建议/参考"
   ↓
4. L1 红线 → 必加"违反 = 翻车"实测数据
   ↓
5. 数据不足 → 标"未验证 · 待实测"·不当铁律用
```
