## 用户原话（2026-06-07）

> "整个的设计逻辑是这样的：优先保障旁白的朗读领读感。"
> "比如说读完这个旁白，我们预计时长是2秒钟。那么说在增加预留的镜头拆解，可能要扩展到3秒，就是说拆解镜头过后。把旁白的时长包含在里面，3秒是可以把这个旁白讲完的。那么说我们最低就是3秒。"
> "但是再增加我们预留的用户消化、读者消化的时长，可能要预留2秒。那么说这个视频可能就是5秒钟。"

**核心逻辑**（两步推导）：
1. **第一步：朗读完成时长** = 旁白能讲完 + 拆镜头后的实际时长（**最少 3s** 起步）
2. **第二步：+ 2s 末帧静默消化**（让读者消化）

## v5 公式的"为什么这样"

**用户两步推导的核心洞察**：

### 步骤 1：朗读完最低时长

**"拆解镜头过后，把旁白的时长包含在里面，3秒是可以把这个旁白讲完的"**

含义：
- 拆镜头 = 把一段长旁白拆成多个镜头（建立 + 跃入 + 朗读 + 末帧）
- 拆完镜头后**总时长 ≥ 朗读时长**（不能比朗读本身还短）
- 例：0.7s 朗读 + 0.5s 铺垫 = 1.2s（但 **不能 1.2s**），**最低 3s**（用户底线）

**为什么是 3s 不是 1.2s**？
- 朗读过程中画面要有**足够时间**让小朋友**听到 + 看到表情/动作**
- 1.2s 朗读结束画面切走 = 小朋友**还没消化画面** = 体验差
- 3s = 朗读 + 视觉段 + 缓冲 = **画面讲完语义，朗读强化读音**

### 步骤 2：+ 2s 末帧静默

**"再增加我们预留的用户消化、读者消化的时长，可能要预留2秒"**

含义：
- 朗读完到视频结束 = **末帧静默**
- 末帧静默 = **小朋友消化时间**（听完 + 看到 + 理解）
- **2s 是用户底线**（不能再短，否则小朋友跟不上）

**为什么是 2s 不是 1s**？
- 1s 静默 = 画面定帧 1s（小朋友觉得"卡了"）
- 2s 静默 = 有微动（眨眼+下移+微笑）= 看起来是"画面还在继续"
- 2s = **小朋友读懂 + 准备下一句**

## v5 公式应用

**主 agent 算总时长**：

```python
# 输入：旁白朗读时长 (duration_seconds)
# 输出：v5 总时长

def calc_v5_duration(duration_seconds, is_closure=False):
    """v5 两步推导"""
    # 步骤 1：朗读完最低时长（最少 3s）
    base_min = max(duration_seconds + 1.5, 3)  # 朗读 + 简短铺垫 ≥ 3s
    
    # 步骤 2：+ 2s 末帧静默（用户底线）
    silence_min = 2.0
    
    return base_min + silence_min
```

**B 旁白量化子 agent 必填字段**：
- `silence_recommendation_seconds`: 末帧静默时间（≥ 2s）
- `silence_rationale`: 必填公式来源（"v5 公式步骤 2 用户底线 ≥ 2s"）
- `total_duration_seconds`: v5 公式算出
