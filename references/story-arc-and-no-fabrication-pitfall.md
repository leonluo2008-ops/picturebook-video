## v8 一次过验证 + 深层根因（2026-06-16 Run 跑 · 本次会话新增）

### v8 一次性过的组合公式

v8 Clip 1（9s 狗三连）= **v1-v7 全部失败后**第一次过用户目检 = 关键里程碑。组合公式：

```
参考图驱动（铁律 #26）
+ 故事弧 4 步（出现 → 转折 → 高潮 → 收尾 · 反模式 1 修复）
+ 动词方向感（"toward the right edge" / "exits the frame on the right" · 新增 RP-26a）
+ 情绪具象化（"耳朵竖起/身体前倾/尾巴飘起"= 1 组身体动作 · 新增 RP-26b）
+ 不凭空添加参考图没有的元素（反模式 2 修复）
+ 不擅自加防御性约束（铁律 #28 + 元纪律）
= v8 一次过
```

**判定**：6 个维度**缺任一个** = 原地循环 / 平铺 / 跟参考图冲突 = 翻车。**全有** = 一次过。

### v8 prompt vs v7 prompt 真实对比

| 维度 | v7（翻车）| v8（一次过）|
|---|---|---|
| 镜头数 | 3 镜平铺（欢快地小跑 / 飞奔 / 跑过）| 3 镜故事弧（耳朵竖起 → 前倾跑向右侧 → 跑出画面）|

| 镜头 | 静止/无方向 | "from behind 跟拍" = 镜头也位移 |

### 2 类新增深层根因（RP-26a / RP-26b · 本次会话发现）

> v7 → v8 的真正修复点**不只是**"加了故事弧"和"删了凭空添加"，**更深层根因**：

#### RP-26a · 原地动词陷阱（**深于"凭空添加"**）

**症状**：prompt 里写的动词 = **不带方向的原地动作** = 模型只能"原地循环"模拟该动作

| 原地动词（v7） | 有方向动词（v8 修复） |
|---|---|
| "欢快地小跑"（没方向）| "跑向画面右侧"（方向明确）|
| "飞奔"（没方向）| "exits the frame on the right"（离开画面）|
| "笑着跑过草地"（一过性 = 不动）| "追着蝴蝶跑远 / 跑出画面"（有目标有位移）|

**判定口诀**：prompt 里的动词 = "在 X 里 Y"（"在草地里跑 / 在花丛里飞"）= **原地动词** = 模型只能"在原地做 Y 动作循环" = 必改 "Y 向 Z / Y 出去 / Y 远" = 有方向 = 强制空间位移。

**反模式**：❌ "小狗在草地上小跑" = 原地动词 ❌ "小狗飞奔" = 无目标 ❌ "小狗跑过草地" = 一过性 = 不动
**正模式**：✅ "小狗跑向画面右侧" ✅ "小狗跑出画面" ✅ "小狗追着蝴蝶跑远" ✅ "小狗从 behind 跟拍中跑远"

#### RP-26b · 情绪修饰词陷阱（**深于"凭空添加"**）

**症状**：prompt 里写的情绪 = **抽象修饰词** = 模型不知道具体怎么做 = 只能"原地循环"模拟情绪

| 抽象修饰词（v7）| 具象化情绪外化（v8 修复）|
|---|---|
| "笑着" | "耳朵竖起（被叫到）/ 身体前倾（准备跑）"|
| "欢快地" | "尾巴翘起（跑得欢）/ 步伐轻盈" |
| "脚步轻快" | "腿抬高 / 步频变密" |
| "越来越快" | "从画左跑到画右越跑越小（视差感）" |

**判定口诀**：prompt 里的情绪 = 抽象词（笑着/欢快/开心/兴奋）= **修饰词陷阱** = 模型无对应身体动作 = 必改 1 组身体动作。

**反模式**：❌ "小狗笑着跑" ❌ "欢快地小跑" ❌ "开心地跳跃"
**正模式**：✅ "小狗耳朵竖起，尾巴翘起，身体前倾" ✅ "小狗腿抬高步频变密" ✅ "小狗头部微微上扬，眼睛睁大"

**根因（深层）**：

- "凭空添加" 是**表层反模式**（"参考图没有的不要写"）
- "原地动词 / 情绪修饰词" 是**深层反模式**（"即使参考图有的，写错了也会触发原地循环"）
- 修复顺序：① 删凭空添加（表层）② 替换为有方向动词 + 具象化情绪（深层）

## 反模式 1 · 缺故事弧（"平铺式"镜头）

### 症状

- 3 个镜头 = 3 个同方向动作 = 节奏平 = 视频"原地循环"无故事感
- 例：v7 Clip 1 狗三连 = 镜头一"欢快地小跑"+ 镜头二"越来越快地飞奔"+ 镜头三"笑着跑过草地"= 3 个动作 = 3 次"快" = 节奏平 = 没故事弧
- 例：v7 Clip 2 猎豹 = 镜头一"矫健有力地奔跑"+ 镜头二"越跑越快"+ 镜头三"消失在远方扬起金色尘烟"= **有故事弧** = "出现→加速→消失"= 完整 = 用户认可

### 修复范式：4 步故事弧模板

每个 Clip 的镜头序列 = 完整 4 步故事弧（出现→转折→高潮→收尾）= **或**简化 3 步（出现→高潮→收尾）：

```
镜头 N 写 prompt 前必走 4 步自检：

① 这个 Clip 在整本故事的哪个位置？（开篇/中段/高潮/收势）
② 这个 Clip 的旁白进度里有几个"叙事转折点"？
   - 1 个转折 = 1 个镜头（但要有起承转合 = 3-4 步）
   - 多个转折 = 多个镜头 = 每镜头 1 个转折 + 转折之间有节奏变化
③ 故事弧的 4 步是否齐全？
   - 出现（开场画面）→ 转折（变化/加速/换方向）→ 高潮（最强烈/最远/最快）→ 收尾（画面落定）
   - 缺任何一步 = 节奏断裂 = "原地循环"感
④ 3 个镜头是否同方向？（同方向 = 缺故事弧）
   - 3 个"快"= 平铺
   - "快 → 更快 → 消失"= 有故事弧
   - "慢 → 快 → 慢"= 有起伏
```

### 故事弧模板（4 步通用）

| 步骤 | 镜头动作 | 节奏 | 视觉特征 |
|---|---|---|---|
| **出现** | 主体在画面里"开始动作" | 慢 / 安静 | 静态启动 / 起步姿态 |
| **转折** | 主体开始"换动作 / 换方向 / 换节奏" | 中 / 加速 | 转换姿态 / 加速 / 转身 |
| **高潮** | 主体"最强烈 / 最快 / 最远"的动作 | 快 / 剧烈 | 巅峰姿态 / 跑出画面 / 扬起尘烟 |
| **收尾** | 画面落定 / 主体离开 / 情绪留白 | 慢 / 静 | 远去 / 定格 / 画面淡出 |

### 实战对比（绘本 v7 Run 跑）

| Clip | 镜头数 | 故事弧 | 用户反馈 |
|---|---|---|---|
| Clip 1 狗三连 | 3 个平铺（欢快地小跑 / 飞奔 / 跑过）| ❌ 平铺 = "原地循环" | "有很大改善但还有原地循环" |
| Clip 2 猎豹 | 3 个故事弧（矫健有力奔跑 / 越跑越快 / 消失在远方扬起金色尘烟）| ✅ 出现→加速→消失 | "完全 OK 没有问题" |

**判定口诀**：

- 3 个镜头 = 同方向动作（如"小跑/飞奔/跑过"3 个"快"）= 缺故事弧 = 必拆/必重写
- 3 个镜头 = 故事弧 4 步 = OK
- 1-2 个镜头 = 故事弧精简版 = OK（短 Clip 简化）

### 跨场景适用

- 绘本视频（本场景）
- 漫剧分镜（多镜头对白场景）
- AI 短剧（多场景剧情推进）
- 短视频（多镜头剪辑）

## 反模式 2 · 凭空添加参考图没有的元素

### 症状

- prompt 写了参考图里**没有**的物品 / 动作 / 表情 / 修饰词 = 模型不知道怎么做 = **反向触发"原地循环"** = 模型做"循环模拟"
- 例：v7 Clip 1 prompt 写了"笑着""脚步轻快""飞奔"= 参考图没有"笑"的表情 + 没有"脚步轻快"的具体动作 + 没有"飞奔"姿态 = 模型只能"原地循环"模拟这些词

### 修复范式：3 步走

```
镜头 N 写 prompt 前必走 3 步自检：

① prompt 里**每个**名词 / 动作 / 表情 / 修饰词 = 参考图里**实际有**吗？
   ├─ 有 → ✅ 用
   ├─ 没有 → ❌ 必删 / 改用参考图里有的表达
   └─ 不确定 → 回 Step 2 vision 自检确认

② "修饰词 vs 具体动作"区分：
   - "笑着"= 表情词 = 参考图里没有"笑"的具体动作 = ❌ 必删
   - "飞奔"= 抽象速度词 = 参考图里没有"飞奔"姿态 = ❌ 必删 / 改"加速跑"+"跑出画面"
   - "欢快地"= 情绪修饰 = 参考图里看不到"欢快" = ❌ 必删 / 改用参考图里有的具体动作

③ "有但参考图里不存在"清单：
   - ❌ 凭空添加物品（蝴蝶 / 花瓣 / 气球 = 参考图里没有）
   - ❌ 凭空添加动作（飞奔 / 跳跃 / 拥抱 = 参考图里没有对应姿态）
   - ❌ 凭空添加表情（笑 / 哭 / 惊讶 = 参考图里没有对应面部）
   - ❌ 凭空添加速度（越来越快 / 极速 = 参考图里看不到速度感 = 模型只能循环加速）
```

### 实战对比（绘本 v7 Run 跑）

| v7 prompt 写法 | 参考图实际有 | 翻车原因 |
|---|---|---|
| "小狗欢快地小跑，**脚步轻快**" | ❌ 没有"脚步轻快"具体动作 | 模型只能"原地循环"做轻快步态 |
| "小狗越来越快地**飞奔**" | ❌ 没有"飞奔"姿态 | 模型只能"原地加速循环" |
| "小狗在雨幕中**笑着**跑过草地" | ❌ 没有"笑着"表情 | 模型只能"原地循环"做表情包 |
| "小狗追着蝴蝶跑"（**修复版**）| ✅ 有"花丛"= 蝴蝶可能存在 | ✅ 用参考图有的目标做引导 |

### 跟 #M3 旁白-镜头映射法的配合

- #M3 5 条核心规范第 2 条："**1 张参考图可被多镜头复用**（铁律 #4 跳过冲突图也是这个思路）"
- 第 5 条："**参考图选择 = 服务于'同步点'**（不是'每张图用一次'）"
- → **prompt 里**只描述参考图实际有的元素 = 模型能"看到"= 能做 = 不卡循环

### 跨场景适用

- 任何用参考图的 prompt 场景（绘本 / 漫剧 / AI 短剧 / 短视频 / 产品演示）

## 修复流程（用户反馈"视频跟参考图契合度不高"或"画面有原地循环"时）

```
用户反馈"画面原地循环 / 视频跟参考图契合度不高 / 镜头无故事感"
   ↓
Step 1 · 回 Step 2 vision 自检图 1-3（重新查参考图实际有什么）
   ↓
Step 2 · 标 image-inventory "有/没有"两列（写明参考图实际有 / 没有）
   ↓
Step 3 · 检查当前 prompt：
   ├─ 是否缺故事弧？（3 个镜头是否同方向？）
   ├─ 是否凭空添加？（"笑着/飞奔/脚步轻快"等）
   ├─ RP-26a · 是否用原地动词？（"在 X 里 Y"模式）
   └─ RP-26b · 是否用情绪修饰词？（抽象词）
   ↓
Step 4 · 修复 prompt（按 4 类反模式优先级）：
   ├─ 缺故事弧 → 按 4 步模板（出现→转折→高潮→收尾）重写镜头序列
   ├─ 凭空添加 → 删修饰词 → 改用参考图里有的具体物品/动作/目标
   ├─ 原地动词 → 改 "Y 向 Z / Y 出去 / Y 远" = 有方向 = 强制空间位移
   └─ 情绪修饰词 → 改 1 组身体动作（耳朵/尾巴/腿/头/眼睛）
   ↓
Step 5 · 重提新 task（绝不重提交原 task · 已扣费）
   ↓
Step 6 · 跑完 → 不抽帧 → 发飞书 → 等用户目检
```
