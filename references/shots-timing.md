# shots-timing - 分镜计时技能

> 来源：云盘 jimeng-2026-05-26-3727-shots-timing.md

---

## 技能描述

分镜计时技能。给每个镜头计算合理时长，然后汇总计算总时长，确保最终成片时长符合要求。

**它是 Phase 5「分镜设计」的第二步**：在完成分镜拆分后，计算每个镜头合理的展示时长，为后续分镜组合做准备。

---

## 计算原则

### 基础规则

| 类型 | 计算方式 |
|------|----------|
| 台词镜头 | 字数 × 0.3s/字（正常说话语速） |
| 最低保底 | 每镜 ≥1s |
| 快速节奏/动作戏 | 短镜头，1-3s 每个 |
| 慢节奏/情感戏 | 长镜头，3-8s 每个 |
| 空镜/转场 | 2-4s |

### 景别影响

| 景别 | 时长建议 |
|------|----------|
| 远景/全景 | 需要时间让观众看清楚环境，3-6s |
| 中景/中近景 | 叙事常用，时长中等，2-5s |
| 近景/特写 | 聚焦细节，时长偏短，1-4s |

### 总时长要求

- 故事短片目标总时长：60-180秒（1-3分钟）
- 单个Clip（AI生成片段）：4-15秒，**硬约束，绝对不能超限**

---

## 计算步骤

1. 按基础规则给每个分镜计算初始时长（秒，保留1位小数）
2. 汇总所有分镜得到初始总时长
3. 如果超出目标范围，对非关键镜头压缩时长
4. 如果小于目标范围，适当给关键情绪镜头增加时长
5. 最终总时长调整到合理范围内

---

## 输出格式

```markdown
# {故事名称}分镜计时表

| 镜号 | 景别 | 画面描述 | 台词/音效 | 时长(秒) |
|------|------|----------|-----------|----------|
| 1 | 全景 | 灰蒙蒙的办公室，落地窗被厚重窗帘遮住... | （环境音：时钟滴答声） | 3.0 |
| ... | ... | ... | ... | ... |

## 总时长
所有分镜总时长：XX.X 秒
```

---

## ⚠️ 重要：duration 参数必须是整数

`--duration` 参数必须是整数，浮点数（如 `7.5`）会导致 CLI 报错：

```
argument --duration: invalid int value: '7.5'
```

向上取整到最近的整数，或在 prompt 里写清楚实际需要的秒数。
