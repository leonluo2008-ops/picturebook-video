## 6 个重构根因

### 根因 7 · 短句用长档位（节奏凭印象）

**类型**：方法不当

**现象**：Pic2 Clip 1 旁白 "Please, Mama, read!" 实际朗读 ~2.5s，但 v0.7.1+pic7 时代主 agent 凭印象选 10s 档节奏 2-1-3-2-2，跑出来 10.1s 视频 = **浪费 7.5s**（4 倍时长）。

**根因**：
- v0.7.1+pic7 节奏决策表有"< 4s → 6s"档和"5-8s → 11s"档
- 主 agent 凭"上次跑通的是 11s"印象选错档
- 没有任何机制强制按档位表查

**v1.0.0 修复**：
- **B 子 agent 强制量化朗读时长**（TTS 实测 / 字数×0.15s 兜底）
- **C 子 agent 强制查档位表**（不凭印象，档位表 = 唯一决策依据）
- 主 agent 调度时**直接看 C 的 `rhythm_formula` 字段**，不重新决策

**Pic2 实测预期**：
- 旧（v0.7.1+pic7）：Clip 1 = 10.1s（浪费 7.5s）
- 新（v1.0.0）：Clip 1 = 6s（节省 40%）

---

### 根因 8 · 单 agent 上下文污染

**类型**：流程缺失

**现象**：主 agent 同时承担：
- 接收需求 + 启动前 6 必问
- 调 lark-cli 下载素材
- vision_analyze 8 张图
- 文本匹配简介旁白
- **自己算旁白朗读时长（凭印象）**
- **自己选节奏档位（凭印象）**
- **自己拼 v15 范式 prompt**
- seedance 提交
- ffmpeg 抽帧
- vision_analyze 抽帧诊断
- **自己判断翻车 + 自己改 prompt**

10 步串行在同一 agent 上下文里。**每步都被上一步污染**：
- 第 4 步凭语感估算时长 → 算错
- 第 5 步凭印象选档 → 选错
- 第 6 步拼 prompt → 多塞"建立+跃入"两个空镜（凭"上次标版 5 镜头"惯性）
- 翻车后第 11 步自己改 prompt → 改得更乱（上下文已带前面所有累积判断）

**根因**：**单一 agent 串行 = 每步都被上一步污染** + 翻车改 prompt 时无法回滚到"干净上下文"。

**v1.0.0 修复**：
- 拆 4 个子 agent（A/B/C/D），**每个子 agent 自己的上下文是干净的**
- 翻车时**重发某个子 agent**（如 C），**不重发全部**（因为输入没变）
- A 错只重发 A，B 错只重发 B，C 错只重发 C，D 错**不重发**（报错回主 agent）

**主 agent 退化为"调度员"**——不直接做 A/B/C/D 任一步。

---

### 根因 9 · 强制拆子 agent（用户原话驱动）

**类型**：方法不当（决策机制缺失）

**用户原话**：
- **"如果全部在一个agent里面可能会受上下文干扰，还有记忆或什么的干扰"**
- **"我认为 clip 时长的计算、题材风格的识别、节奏的设计、提示词的编写，这些都应该要设计合适的子 agent 去执行"**

**v1.0.0 架构**：

| 子 agent | 职责 | 隔离意义 |
|---|---|---|
| **A · 风格识别** | 调性 + 节奏倾向 + 风格锚定词 | 与算时长隔离（不受"上次跑过 10s"印象污染）|
| **B · 旁白量化** | 朗读时长 + 复杂度 + 静默推荐 | 与选节奏隔离（不受"上次 11s 档"印象污染）|
| **C · 分镜设计** | 节奏公式 + 镜头表 + v15 prompt | 与跑视频隔离（不受"上次 seedance 翻车"印象污染）|
| **D · 视频执行** | seedance 跑 + ffmpeg 抽帧 + vision 自检 | 与改 prompt 隔离（**D 不改 prompt**——翻车报回主 agent）|

**A+B 并行**（L1）→ **C 依赖 A+B**（L2）→ **D 依赖 C**（L3）。

**3 层流水线**：避免一次性下发 A+B+C+D 把单 agent 问题搬到子 agent 内部。

---

### 根因 10 · 短档位"拉远未完成"

**类型**：方法不当（节奏模板残留）

**用户原话**：
- **"再跟你说，前面浪费是 6s clip 里面前 2s 是空镜头"**
- **"你前面设计那么长的慢镜头空镜头来做什么？"**

**现象**：v0.7.1+pic7 范式 prompt 模板有"镜头一 0-2s · 场景建立：镜头从空白拉远到全景"。**1-2s 不够模型完成"拉远"动作**——Pic2 Clip 1 v1 frame_01 跑出"角色只露边缘"（拉远未完成）。

**根因**：v0.7.1+pic7 节奏模板用"拉远"做镜头一，假设"拉远"是单帧操作。实际是**渐变镜头**——需要 2-3s 才到位。

**v1.0.0 修复**：
- C 子 agent 档位表**短档位用"切到全景"**（"切" = 一帧到位，0s 完成）
- 长档位仍可用"拉远"（2-3s 够时间）
- self_check 第 8 项"镜头一 ≠ 拉远未完成"强制检查

**Pic2 实测预期**：
- 旧（v0.7.1+pic7）：镜头一 0-2s "拉远" → frame_01 拉远未完成
- 新（v1.0.0）：镜头一 0-1s "切到全景" → frame_01 一帧到位

---

### 根因 11 · "标版 = 必跑" 残留

**类型**：方法不当（用词语义错位）

**现象**：v0.7.1+pic7 节奏决策指南写：
> **"标版"= 常见情况默认参考**，不是必跑。

但实际 Pic2 Clip 1 v1 跑了 10s 节奏 2-1-3-2-2 = **"5 镜头标版机械套"**，证明：
- "标版"这个词**让 agent 误以为是必跑模板**
- 哪怕加了"不是必跑"说明，**词义本身暗示"标准答案"**

**v1.0.0 修复**：
- C 子 agent 决策表用**档位 + 数字公式**（`< 4s → 1-1-3-1` / `5-8s → 2-1-3-5`）
- **不出现"标版"字样**（彻底去除语义暗示）
- self_check 第 9 项"节奏_按_档位_选_不是_凭印象"强制检查

---

### 根因 12 · "凭印象算时长"（v0.7.1+pic7 时代根因）

**类型**：流程缺失

**现象**：v0.7.1+pic7 时代主 agent "凭语感"算朗读时长：
- "这句大概 2.5s"（实际 TTS 测出来可能是 3.2s）
- 误差 ±30%

**根因**：skill 没有强制"先 TTS 实测"流程。`references/旁白朗读时长计算.md` 有公式但**不是必跑**。

**v1.0.0 修复**：
- **B 子 agent 强制量化**：先 TTS，没 TTS 走兜底公式
- 主 agent **必问用户**"提供 TTS 吗"（提供 = TTS 实测，不提供 = 兜底公式 + warning）
- 兜底公式写在 B 子 agent 的 `duration_source` 字段 = `fallback_formula`（**不静默**）
