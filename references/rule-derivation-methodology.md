# 规则推导方法论 · Rule Derivation Methodology

> **元方法论（比 #M1 更底层）**：skill 规则应该从**决策逻辑**推导，不是从**特定情境的表象**总结。

## 2. 核心命题

**skill 里的规则 ≠ 特定情境的结论 = 从决策逻辑推导出来的判断方法**

| 写法 | 类型 | 鲁棒性 |
|---|---|---|
| "拟声符号 = 显式说消失" | **表层规则**（从 4.jpg 彩条 = 嘎嘎叫 = 叫完就消失 总结）| ❌ 下次遇到"持续在叫"的符号 = 误判 = 翻车 |
| "拟声符号的生命周期 = 它表达的瞬时动作的生命周期" | **决策逻辑**（找出"符号 → 动作 → 生命周期"的判断方法）| ✅ 模型用这个判断方法 = 任何场景都对 |

---

## 3. 4 步规则推导法

```
第 1 步：看到现象（用户反馈 / 翻车 / 任务完成）
   ↓
第 2 步：找出"决策逻辑"
        - 不是找"特定情境的结论"（如"嘎嘎叫 = 消失"）
        - 是找"判断方法"（如"符号生命周期 = 表达动作的生命周期"）
        - 关键：决策逻辑 = 跨场景通用的判断方法
   ↓
第 3 步：把"决策逻辑"写进 skill
        - 写法：决策逻辑本身 + 跨场景的应用方式
        - 不是：特定情境的结论 + 反模式
   ↓
第 4 步：模型用决策逻辑自己判断
        - 看到新场景 = 套用决策逻辑 = 自然处理
        - 不会因为"特定情境的结论"不适用而翻车
```

## 4. 反例 vs 正例对照

### 反例（表层规则化 · 必避免）

```markdown
❌ 反例 1: 拟声符号 = 显式说消失
   推出来自：4.jpg 头部彩条 = 嘎嘎叫 = 叫完就消失
   鲁棒性：❌ 下次"持续在叫" = 误判 = 翻车

❌ 反例 2: 时长 = 凑整到 8s / 10s / 12s
   推出来自：v1 clip 都用了 8s = 凑整 = 看起来饱满
   鲁棒性：❌ 4s 旁白 = 不需要凑整 = 凑整 = 拖沓

❌ 反例 3: 每个 clip 都要设计起承转合
   推出来自：电影感 = 每个镜头都有"起承转合"
   鲁棒性：❌ 绘本 = 节奏快 = 单 clip 直接切入 = 不需要"起"
```

### 正例（决策逻辑化 · 必走）

```markdown
✅ 正例 1: 拟声符号的生命周期 = 它表达的瞬时动作的生命周期
   推导逻辑：符号 = 视觉化的动作 = 动作结束 = 符号该消失
   鲁棒性：✅ "嘎嘎叫一下" = 瞬时 = 消失；"持续冒烟" = 持续 = 保留

✅ 正例 2: 时长 = 真实叙事秒数（不凑整）
   推导逻辑：时长是给叙事用的，不是给"显得饱满"用的
   鲁棒性：✅ 4s 旁白 = 提交 4s；8s 旁白 = 提交 8s；不需要"凑整"

✅ 正例 3: 整本节奏优先（#M1）
   推导逻辑：起承转合是整本故事的维度，不是单 clip 的维度
   鲁棒性：✅ 开篇可"起"；中段直接切入；高潮快切；收势留白
```

## 5. 4 步自检（写完规则必跑）

写完一条规则后，**必跑 4 问**：

| # | 问 | 期望 |
|---|---|---|
| 1 | **这条规则是从"特定情境的结论"总结的，还是从"决策逻辑"推导的？** | 决策逻辑 |
| 2 | **如果遇到不同情境，这条规则还成立吗？** | 成立 = 决策逻辑对；不成立 = 必重写 |
| 3 | **我能不能说出这条规则背后的"判断方法"？** | 能说 = 决策逻辑对；说不出 = 必重写 |
| 4 | **模型用这条规则 = 是会思考，还是会照搬？** | 思考 = 决策逻辑对；照搬 = 必重写 |

**判定口诀**：
- 4/4 全过 = 规则有效 = 保留
- 任 1 项失败 = 规则是表层规则 = **必重写为决策逻辑**
- 跳过自检 = 必翻车（用户下次会纠正"你不能看表面"）

## 8. 速查（一眼能用）

写规则前 1 问：
```
这条规则是从"决策逻辑"推导的，还是从"特定情境的表象"总结的？
├─ 决策逻辑 → 直接写
└─ 表象总结 → 必先找出决策逻辑 → 再写
```

写规则后 4 问：
```
1. 这条规则是从"特定情境的结论"总结的，还是从"决策逻辑"推导的？
2. 如果遇到不同情境，这条规则还成立吗？
3. 我能不能说出这条规则背后的"判断方法"？
4. 模型用这条规则 = 是会思考，还是会照搬？
```
