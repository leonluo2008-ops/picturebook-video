## 2. 反模式：反推档为什么错

**根因**（数学拟合的陷阱）：
- 默认速率 1.0 词/秒 和 3.0 字/秒 是**独立测速**（英文 1 词 = 1.0s / 中文 1 字 = 0.33s）
- 用缩放系数同时缩放两者 = **强制中英等比例减速** = 数学上满足总和 = 用户 TTS，但**单位时间内单词数 = 字数都同比缩小** = 真实英文读速不可能慢到 0.67 词/秒（每词 1.5s = 异常慢）
- **反推档的总和 = 用户 TTS 是拟合的，但单项速率毫无物理意义**

**修复**：1.4/4.0 是合理的"领读短句档"——实际英文 1.4 词/秒 = 比默认稍快（孩子读得快），实际中文 4.0 字/秒 = 比默认稍快（短句 + 标点少 + 节奏紧凑）。**这两档速率各自都有物理意义**，不是数学拟合出来的。

## 4. Step 3 必跑 5 档对比（决策树升级）

原本铁律只说"1.0/3.0 兜底公式"，但跨本验证显示**绘本短句领读型 = 1.4/4.0 更准**。**Step 3 必给 5 档对比**（不只是单档）：

| 档位 | 英文 | 中文 | 触发场景 |
|---|---|---|---|
| **慢估（兜底）** | 1.0 词/秒 | 3.0 字/秒 | 默认·长句+复杂词 |
| **领读短句** | 1.4 词/秒 | 4.0 字/秒 | **绘本短句 + 双语领读**（推荐默认）|
| **偏快** | 1.5 词/秒 | 4.0 字/秒 | 极短单词开篇 |
| **反推拟合** | 0.67 词/秒 | 2.0 字/秒 | **❌ 反模式·数学拟合≠真实速率** |
| **极慢估** | 0.8 词/秒 | 2.5 字/秒 | 长段描述句 + 形容词多 |

## 5. 反模式（必避）

- ❌ **用缩放系数同时缩放中英速率** = 数学拟合 = 看起来总和匹配但单项毫无物理意义
- ❌ **默认 1.0/3.0 慢估** = 绘本短句领读型必差 10s+ = 重算
- ❌ **只跑 1 档** = 不跑 5 档对比 = 必翻车（差 10s+ 才发觉）
- ✅ **5 档对比 + 选物理意义最强那档** = 1.4/4.0（领读短句型）/ 1.0/3.0（长句型）/ 1.5/4.0（极短单词型）

## 6. 修复流程（Step 3 必走 5 步）

```
1. 1.0/3.0 算段 TTS（慢估兜底）→ 看上限（可能 +10s）
2. 1.4/4.0 算段 TTS（领读短句档）→ 看下限（绘本短句常命中）
3. 1.5/4.0 算段 TTS（极短单词档）→ 看极短句情况
4. 反推档（用缩放系数）= 必跑但**仅参考** = 不选
5. 选物理意义最强那档 + 跟用户给 TTS 差 ≤ 5s
   ├─ ≤ 3s = 完美（无需报告）
   └─ 3-5s = 正常（需报告选档理由）
   └─ > 5s = 重算或报告用户
```
