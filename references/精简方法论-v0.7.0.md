# SKILL 精简方法论（v0.7.0 实践沉淀 · 2026-06-05）

> **背景**：picturebook-video SKILL 从 2070 行精简到 264 行（砍 87%），用户原话："保留 V15 版本系列，其他的因为 V15 已经综合了前面的长处，所以前面的版本我是想移除掉，但是 V15 系列版本包括 V15.1 是不要改的"
>
> **适用**：任何累积了多版本范式的 SKILL 优化场景

---

## 1. 核心原则

| 原则 | 含义 |
|---|---|
| **新版本综合了旧版本** | v15 已综合 v3-v14 的长处 → 旧版详细段 2 写法可砍 |
| **保留 = 主线 + 必要案例** | 主线（v15 + v15.1）+ 必要案例（Cat 6 Clip）= 必留 |
| **精简 ≠ 删干净** | 是"砍历史包袱"不是"砍干净"，git tag 备份完整版 |
| **精简必须可逆** | git tag 备份是必要条件，**不是 nice-to-have** |

---

## 2. 操作流程（5 步）

### Step 1：识别"当前默认版本"

问自己：
- 当前用户用哪个范式最多？（v15）
- 哪些版本是历史包袱？（v3-v14）
- 哪些是当前范式必需子版本？（v15.1）

**反问**：如果新版本没综合旧版本长处，**不能精简**——必须等新版本完全稳定。

### Step 2：git tag 备份完整版（必须）

```bash
# 在最新版本 commit 上建 tag
git tag v0.6.2-full  # 完整版名
git tag --list | grep v0.6.2-full  # 验证

# 恢复命令（精简后想找回）
git checkout v0.6.2-full -- SKILL.md
git checkout v0.6.2-full -- references/
```

**为什么必须**：
- 精简是不可逆操作
- 用户后来要查旧内容 = 0 成本恢复
- 不是 nice-to-have，是必要的安全网

### Step 3：新建分支（不动 main）

```bash
git checkout -b simplify-YYYYMMDD
```

**为什么**：精简前必须先在本地分支干，review 通过再合 main。

### Step 4：精简 SKILL.md（重写优于 patch）

| 操作 | 推荐 | 不推荐 |
|---|---|---|
| 写新 SKILL.md | **整文件 write_file 重写** | 100 个小 patch |
| 删 references/ | `rm` 旧文件 + git commit | 留在主文档但加"已废弃" |

**为什么 write_file 重写**：
- 重新组织结构和表达
- 避免"补丁式精简"产生新冗余
- hill-climbing 视角看 = 一次"探索性重写"

### Step 5：精简后 + 达尔文优化（recovery）

**问题**：精简 = 一次性砍 87% → 8 维分数下降 11+

**修复**：精简后**接达尔文优化循环**（3 轮 keep = +9.0）→ 回到 96.5

**操作**：

```bash
# 1. 简化后跑基线评估
darwin baseline  # 假设已加载 darwin-skill

# 2. 写新 test-prompts（基于精简后范式）
# 旧 test-prompts 触发不到 = 必须重写

# 3. 跑 3 轮棘轮优化
darwin optimize --rounds 3
```

## 3. 砍什么 / 留什么

### ✅ 留（主文档）

- 当前范式完整描述
- 当前范式 + 必要子范式（如 v15.1）
- 关键铁律（10-15 条）
- 1 个端到端实战案例
- 启动前自检清单
- references 索引（指向子文档）

### ❌ 砍（归档 git tag）

- 历史范式段 2 详细写法
- 历史范式选型决策树
- 历史范式铁律（#1-#31 类）
- 历史范式探索过程对话
- 历史 references 详细文档（如 versions/v3-v14.md）

### ⚠️ 判断（不砍不挪）

- 当前范式未稳定 = 不砍历史
- 历史还可能被引用 = 不砍
- git tag 备份失败 = 不砍

---

## 4. 反模式（不能这样精简）

| ❌ 反模式 | 后果 |
|---|---|
| 精简前没 git tag | 旧内容**无法恢复** |
| 在 main 直接精简 | 污染主分支，revert 痛苦 |
| 写 100 个 patch 删 | 新冗余产生，结构更乱 |
| 精简后不接达尔文 | 分数下降 11+ 没人发现 |
| test-prompts 没重写 | 旧 prompt 触发不到新范式 |
| 砍 references 但不删 | 主文档 264 行 + references 1 行指引 = 半成品 |
