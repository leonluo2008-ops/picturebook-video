# 参考图是起点，不是限制（reference-image-as-starting-point）

> **核心命题**：视频画面 = 读者脑补的动画 = 镜头语言 + 旁白进度的视觉化 + 自然流畅的镜头叙事
>
> **来源**：2026-06-16 Run 跑绘本 v1-v4 5 次翻车 → 用户原话级纠错（**第 1 次升错了·本轮降级**）

---

## 1 · 用户原话（必读 5 遍）

> **"视频的镜头画面设计,要充分建立在参考图和旁白的基础上,本来我发给你的参考图和旁白是具有强关联对应关系的,你在设计视频画面时,必须以参考图为重要依据,让视频流畅合理,尽量避免和参考图冲突。比如参考图中的人物所处的位置,朝向,画面布局等,视频应该是参考图的动态呈现,而不是重新设计视频"**

> **核心概念（第 2 次纠错补充）**：动态呈现 = **读者脑补的动画 = 镜头语言 = 有节奏感的运动** = 静态绘本的参考图是静止图片，读者看图时脑袋里会呈现运动的感觉
>
> **补充说明**：假如参考图画面是小狗作出跑的姿态，旁白是"小狗在奔跑"。那么视频的画面就以参考图为起点，设计成小狗在奔跑，能匹配旁白的意思，**唤起读者的共鸣**就可以了。重点是整个视频**流畅自然**，不要过多约束，可以使用任何方式，多镜头，单镜头，多机位，只要时长能容纳，都是允许的设计。

---

## 2 · 真实根因（5 次翻车的本质）

**第 1 次错误理解（v1）**：把"参考图驱动"理解成"凭印象 + 不看图" → 写"张臂迎面跑来"（参考图是侧身朝右）= 跟参考图每条冲突

**第 2 次错误修正（v2）**：改正了朝向/景别/招牌位置 = 但**过度补偿** = 写"必须保持参考图的整体构图...不重新设计画面布局" + "跑步动作循环" + "固定原景别不做推拉" = **把参考图当枷锁**

**第 3 次错误加深（v3）**：v2 视频用户反馈"画面非常糟糕+呆板" → 我**误归因为"末帧定格"反模式** → 加了"循环不切断不冻结"对抗约束 = 错上加错

**第 4 次错误加深（v4）**：v3 视频依然不对 → 我**继续加约束** = "持续交替/持续摆动" = 把动作描述塞进"持续+持续+持续"的死循环 = 极端僵化

**第 5 次真实理解（v5）**：用户明确纠正："动态呈现 = 读者脑补的动画"+"参考图是起点不是限制"+"可以跑出画面/转弯/姿态变化" = 真正理解

## 3 · 核心命题（本轮真实理解）

```
视频 = 读者脑补的动画 = 镜头语言 + 旁白进度的视觉化 + 自然流畅的镜头叙事
   ↓
参考图是**起点**，不是**限制** = 启发性参照
   ↓
镜头 / 动作 / 景别 / 朝向 = 自由设计
   ↓
只要能"唤起读者共鸣" + 旁白 + 时长能容纳 = 都允许
   ↓
明确允许（用户原话）：
✅ 跑出画面（往画面外跑 = 镜头有方向感）
✅ 转弯往回跑（动态 = 有变化 = 镜头有故事感）
✅ 跑 1-2 秒再换动作（动态 = 节奏点）
✅ 姿态一直在变（动画感来源 = 不是"保持参考图"）
✅ 多镜头 / 单镜头 / 多机位都允许（时长能容纳）
✅ 镜头推进拉远切换角度（自由运镜）
   ↓
明确禁止（v2-v4 误加）：
❌ 把参考图当枷锁（铁律 5 项必查"严格匹配"）
❌ 写"必须保持参考图的整体构图...不重新设计画面布局"
❌ 写"固定原景别不做推拉"
❌ 写"跑步动作循环"
❌ 写"跑步姿态保持...跑步循环"
❌ 写"末帧微动:定格在...瞬间"
❌ 写"持续跑步动作循环不切断不冻结"
❌ 写"持续交替/持续摆动"
```

## 4 · 写 prompt 范式（v5 真实成功写法）

**v1 反模式（错误）**：
```
主体定义：折纸小男孩 @Image6（黑鼻+粉鼻头+张开双臂+灿烂笑容），画面左侧
分镜绑定：必须保持参考图的整体构图不重新设计画面布局
镜头一：保持参考图原有中景景别不做推拉，固定中景，
       小狗保持侧身朝右跑步姿态，跑步动作循环（前后腿交替+尾巴摆动+耳朵轻摆），
       末帧微动：小狗保持跑步姿态定格在跨步瞬间
```

**v5 正确写法**：
```
主体定义：@Image6 一个折纸小男孩在草地上开心地飞奔
（黑色短发、粉色上衣、蓝色短裤、张开双臂、灿烂笑容）

分镜绑定：@Image6 作为参考起点，画面设计自由，参考图是起点不是限制

镜头一（0-5s · 跑向你）：从 @Image6 出发，画面是明媚蓝天下的草地，
小男孩在草地上开心地张开双臂飞奔，粉衣下摆迎风飘动，脚下草屑飞扬，
镜头从远处逐渐拉近，小男孩越跑越近，笑容越来越大，
自然地迎着镜头跑来，最终跑出画面或者停下来，
姿态自然欢快有孩子气的活力

参考图原有的招牌"run"和红"跑"字作为画面元素保留在合适位置
```

**关键对比**：

| 维度 | v1（错）| v5（对）|
|---|---|---|
| 主体定义 | 详细特征列表（黑鼻+粉鼻头+...）| 简短 + 起点式（"小男孩在草地上飞奔"）|
| 分镜绑定 | "保持参考图...不重新设计" = 枷锁| "作为参考起点，画面设计自由" |
| 镜头动作 | "跑步动作循环（前后腿交替+尾巴摆动）" = 死循环| "越跑越近，笑容越来越大，迎着镜头跑来" = 有故事感 |
| 末帧 | "末帧微动:定格在跨步瞬间" = 反暗示| **不写** = 信任模型自然结束 |
| 持续/不切断 | "持续跑步动作循环不切断不冻结" = 僵硬| **不写** = 不约束 |
| 景别 | "保持参考图原有中景景别不做推拉" | "镜头从远处逐渐拉近" = 自由运镜 |
| 朝向 | "保持侧身朝右跑步姿态" | 自然跑动，不锁定姿态 |

## 5 · 修复流程（用户反馈"画面呆板/糟糕/不自然"时）

```
用户反馈"画面呆板/不自然/静态"
   ↓
**第一步 = 必先重读用户原话**（重新理解"动态呈现"的真实含义）
   ↓
❌ 错误路径：写防御性约束（"持续/不切断/不冻结/末帧微动"）
   ↓
✅ 正确路径：识别哪些约束在僵化画面 = 全部删掉
   ↓
重新写自由镜头 prompt（v5 范式）
   ↓
重新跑（同 task 失败的话用新 task_id）
```

**用户已经拍板的反约束原则**：
- 用户说"不要急着把讲的东西都加到铁律啊" = **不要凭印象升级铁律**
- "你擅自增加这种规则，并且不对加入的规则进行检查校正，包括冲突处理" = **用户元偏好级** = 禁止擅自加规则
