# 领读型合并 · 2图=1Clip 连续运镜

> 本文档解决：领读绘本（弱情节、靠画面+旁白推）单图单 Clip 撑不到 4s 最低线的问题。
> 模式来源：2026-06-02 cactus 仙人掌绘本实测。
> 模式提取（不绑定具体内容）：**「2张图 + prompt 内连续运镜 + 首尾帧控制」**。

---

## 适用判断（同时满足才合并）

- 绘本类型 = **领读型**（弱情节、认字/认知/英语单词教学）
- 旁白 < 8s 单段（按 0.3s/字计算）
- 两图场景元素相似（同色系/同主体/相邻景别）
- 弱叙事弧（不需要明确的「前一幕→后一幕」情节推进）

**反例不合并**：
- 故事绘本（强情节、起承转合）→ 1图=1Clip
- 旁白 > 8s 单段 → 时长够了，不需要合并
- 两图视觉元素差太多（季节变/视角大幅切换）→ 合并后视觉割裂

---

## 合并的 3 个核心组件

### 组件 1 · 首尾帧控制（关键）

| 参数 | 作用 |
|------|------|
| `--image <前图>` | 锁住**起始画面**（clip 第一帧） |
| `--last-frame <后图>` | 锁住**结束画面**（clip 最后一帧） |

**两图不传 ref-image / video-refs**——领读绘本图间**没有**物理连续动作（不像漫剧人物奔跑），不需要参考图一致性。

### 组件 2 · prompt 内部连续运镜

整段 prompt **不用 `[00-XXs]` 时间轴切分**，而是**单线连续**：

```
[起始画面]@Image1 镜头从 [起始状态] 开始
  → [过渡动词]: 镜头推进 / 镜头缓拉 / 镜头暖光晕开
  → [中间动作]: 主体微动 / 主体摇摆 / 暖光变化
  → [结束画面]@Image2 镜头停在 [结束状态]
  → [收势]: camera locks completely, image becomes still, no fade
```

### 组件 3 · 收势必须明确三件事

> 来自既有 SKILL「关键教训 #8」+ 领读绘本实测验证。

1. **镜头锁住（Camera LOCK）**
2. **画面静止（No Motion）**
3. **不渐隐不淡出（No Fade / No Dissolve）**

英文句式直接复用：
> "Final frame: the camera locks completely, the image becomes still, no fade, no dissolve, holds to the last frame."

## 时长分配公式

| 段落位置 | 推荐时长 | 理由 |
|---------|---------|------|
| **Clip 1（开头）** | 8s | 标题+主形象登场，需要稳 |
| **Clip 2-N-1（中间段）** | 8-9s | 均匀推进，节奏稳定 |
| **Clip N（结尾）** | 10s | 情感核心（坚强/自信/友谊），给足余韵 |

**公式**：`总时长 ≈ 段落数 × 8-10s`，向上对齐到 8/9/10s 整数值。

**例 1**：8 张图分 4 个 Clip → `8 + 8 + 9 + 10 = 35s` ✅
**例 2**：6 张图分 3 个 Clip → `8 + 9 + 10 = 27s`
**例 3**：10 张图分 5 个 Clip → `8 + 8 + 9 + 9 + 10 = 44s`

## 合并后视觉节奏控制

| 节奏目标 | 写法 |
|---------|------|
| **活泼** | 镜头推进 + 暖光晕开 + 主体微动 + 沙粒飞动/花瓣飘落等粒子效果 |
| **舒缓** | 镜头缓拉 + 暖色渐变 + 主体缓慢摇摆 |
| **震撼** | 镜头仰角推近 + 暖光溢出 + 主体高耸占满画面 |

领读绘本默认走**活泼**风格——这是用户偏好（2026-06-02 二大爷确认：「整体氛围要活泼」）。

## 完整 prompt 模板（4 段式结构）

```
@Image1 as the opening frame,
[起始描述：场景+主体+起始状态];
[过渡描述：镜头运动+视觉变化];
continues to slowly push forward and transitions seamlessly to @Image2 as the second half,
[结束描述：场景+主体+结束状态];
[收势]: final frame: the camera locks completely, the image becomes still, no fade, no dissolve, holds to the last frame.
[风格]: Children's picture book illustration style, flat cartoon with thick black outlines, vibrant rainbow colors, warm and [lively/uplifting] atmosphere.
```

**注意**：
- `@Image1` `@Image2` 是**写给模型看的标记**，实际调用靠 `--image` + `--last-frame` 传参
- 真实生成的 prompt 写**第一/第二半**或**前段/后段**等中性词
- 风格段**必须**——领读绘本风格一致性靠这句话锁

## 与官方 SOP 的差异

| | 官方短剧 SOP | 领读绘本合并（本模式） |
|--|--|--|
| **图间关系** | 物理连续动作 | 独立叙事单元（无物理连续） |
| **clip 衔接** | 尾帧接力（action continuation） | 每个 clip 内部自闭合 |
| **分镜切分** | 拆动作 | 拆**叙事单元**（2图=1叙事单元） |
| **收势** | 动作进行中接下一个 clip | 镜头锁住+画面静止+不渐隐 |
| **prompt 写法** | 接力 + 转场 | 连续运镜 + 内部过渡 |

**反模式警告**：
- ❌ 领读绘本用「尾帧接力」写法 → 错（绘本图间没有物理动作）
- ❌ 用 `[00-XXs]` 时间轴切分 2 张图 → 错（领读绘本单线叙事，不要人为切分）
- ❌ 单图 1 Clip 撑 4s → 错（浪费 1-2s 缓冲，节奏拖沓）

## 实测 pitfall（2026-06-02）

1. **多 Clip 并行提交时 `--download` 是文件路径不是目录** → 多 clip 会互相覆盖。传**完整文件路径**（每个 clip 独立命名）。
2. **绘本默认 `--generate-audio true`**（2026-06-02 修正）→ 绘本**需要**音效（拟声+环境音配合画面），**不需要**人声旁白、**不需要** BGM。三件事在 prompt 里**严格区分**：
   - 音效：**写进 prompt**，由 Seedance 同步生成
   - 旁白：**不写**，走后期 TTS
   - BGM：**不写**，绘本默认不加
   - 写法三铁律见 `references/绘本音效-prompt写法.md`（独立 `[Sound effect: ...]` 块会切碎视觉，必须嵌入视觉句中间）
3. **总时长公式别照搬** → 不同绘本旁白长度差异大，先算「单页旁白时长」再决定是否合并。
