# v6 5 段骨架 · 模板（v1.0.3+pic12 · 2026-06-07）

> **v6 = v15 4 段 + 文字持续可见段**（铁律 #73 + #77）
>
> **触发条件**（必跑 v6 不用 v15）：
> 1. **领读型绘本**（彩色文字 = 领读锚点，全程可见是核心）
> 2. **目标词反复出现**（如 Bird "bird" 8/8 句全有强调）
> 3. **0-3 岁低幼**（需要强视觉锚点 + 慢节奏领读）
>
> 警示向/冒险向/广告向 = 走 v15 4 段（不加文字持续可见段）。

---

## 模板正文（v6 · 5 段 · 主 agent 必用）

```yaml
# v6 5 段骨架模板 · 填 12 变量即可（不要改结构）
# v6 = v15 4 段 + 段 5 文字持续可见段

段 1 · 主体定义
主体定义：{主角1}@Image{N}（{视觉特征1} + {视觉特征2} + {视觉特征3} + {姿态/动作} + {表情}），{背景}@Image{N}（{主色} + {次色} + {纹理} + {氛围}）；

段 2 · 分镜绑定 + 文字保留 v3
分镜绑定：@Image{N} 作为唯一参考帧；
{镜头1 描述}{拟声1}一响，{画面微动元素1}
{镜头2 描述}{拟声2}一响，{画面微动元素2}
{镜头3 描述}{拟声3}一响，{画面微动元素3}
{镜头4 描述}{拟声4}一响，{画面微动元素4}
{镜头5（可选 收势）}{拟声5}一响，{画面微动元素5}
{末帧策略}画面继续微动（{末帧微动具体动作1} + {动作2} + {动作3}），末帧 {X}s 静默消化时间，末帧 1s 内必须包含至少 1 个动作元素（{必含动作}），不得成为定格海报；
参考图原有的所有文字（顶部 1/6 画面的{EN_COLOR_DESC}英文 "{EN_WORD}" 和中文"{ZH_WORD}"字）必须完整保留作为画面元素，模型不得删除或替换这些文字，文字位置锁定在顶部 1/6 画面不要重新生成该区域内容，让文字自然融入场景；

段 3 · 风格锁定（v14/v15 继承）
风格：2D paper collage style, {具体风格词1}, {具体风格词2}, {具体风格词3}。

段 4 · BGM 段（v15 继承）
无任何背景音乐、无旁白人声、无哼唱。

段 5 · 文字持续可见段（v6 独有 · 铁律 #73 + #77）
参考图原有的所有文字（顶部 1/6 画面的{EN_COLOR_DESC}英文 "{EN_WORD}" 和中文"{ZH_WORD}"字）必须从 t=0 到 t=末帧 全程可见，模型不得删除或替换这些文字，文字位置和颜色全程锁定不重新生成该区域内容；
微动画（领读锚点 · 必含）：① 每字呼吸式明暗交替 0.5s/次（"{EN_WORD}" 4 字轮流 + "{ZH_WORD}" 字）② 按朗读节奏字符顺序浮现：{EN_WORD 第 1 字母}(0.3s) → {第 2}(0.6s) → {第 3}(0.9s) → {第 4}(1.2s) → {ZH_WORD}(1.5s) ③ 整体轻微浮动（0.3s/次·上下 2px 范围）；
字体：{字体风格描述}。
```

## EN_COLOR_DESC 两种格式（fill 脚本必兼容）

**C 子 agent 输出的英文颜色字段有两种格式**——fill 脚本必须兼容：

### 格式 A：单字符串（BIRD/Banana 简单目标词）
```json
"text_position": {
  "en_color": "N=鲜艳红色 / o=橙红色"
}
```
**清洗函数**：`_parse_en_color()`（已有，pic4 v1.0.2 沉淀）→ `"鲜艳红色色"`

### 格式 B：字母级字典（Bird 等多字母目标词·Pic5 Bird 实战 2026-06-07）
```json
"text_position": {
  "en_color_pattern": {
    "b": "鲜艳红色",
    "i": "鲜艳蓝色",
    "r": "鲜艳红色",
    "d": "鲜艳蓝色"
  }
}
```
**清洗函数**：`_parse_en_color_pattern()` → `"b=鲜艳红色 / i=鲜艳蓝色 / r=鲜艳红色 / d=鲜艳蓝色"`

**fill 脚本**（v1.0.3+pic12 升级版）必先判断字段存在性：
```python
if 'en_color_pattern' in tp:
    en_color_desc = _parse_en_color_pattern(tp['en_color_pattern'])
elif 'en_color' in tp:
    en_color_desc = _parse_en_color(tp['en_color'])
else:
    en_color_desc = '彩色'
```

**踩坑教训**（Pic5 Bird 实战 2026-06-07）：原 fill_v15_template.py 只支持 en_color 字符串格式，Bird 用了 en_color_pattern 字典，C 子 agent 输出不识别 = 字段空 = "彩色" 兜底。修复：fill 脚本升级必加 `_parse_en_color_pattern()` + 字段存在性判断。

## 字符顺序浮现时间表（v6 段 5 模板变量）

| 字母/字 | 时间锚点 | 说明 |
|---|---|---|
| 单词第 1 字母 | 0.3s | 朗读开始前 0.3s 浮出 |
| 单词第 2 字母 | 0.6s | 跟读第一个字 |
| 单词第 3 字母 | 0.9s | 跟读第二个字 |
| 单词第 4 字母 | 1.2s | 跟读第三个字 |
| 中文单字 | 1.5s | 跟读完整单词后浮出中文 |

**踩坑教训**（Pic5 Bird 实战）：原 C 子 agent 输出"按朗读节奏字符顺序浮现"但没具体时间锚点 = 模板无时间戳 = seedance 不会自动加微动画。**v6 段 5 必含时间锚点**（每个字母/字带 0.3s 步进）。

## 与 v15 4 段的区别

| 维度 | v15 4 段 | v6 5 段 |
|---|---|---|
| 段数 | 4 | 5（+ 文字持续可见段）|
| 适用 | 警示向/冒险向/广告/漫剧 | 领读型绘本/认知启蒙/低幼 |
| 文字可见性 | 段 2 文字保留（一次锁）| 段 2 锁 + 段 5 全程可见 + 微动画 |
| EN_COLOR_DESC | 必填 11 变量 | 必填 12 变量（+ EN_COLOR_DESC 清洗函数） |
| 字符顺序浮现 | 不需要 | 必填（时间锚点 0.3/0.6/0.9/1.2/1.5s） |
| 字体描述 | 段 2 附带 | 段 5 独立字段（必填）|

**判断口诀**：**领读型/认知启蒙/0-3 岁 = v6**；其他 = v15。

## 红线

- ❌ **v6 场景不能用 v15**（领读型缺段 5 = 文字后半段消失 = 翻车）
- ❌ **不能用 v13 / v10 范式**（v10 段太多 prompt 长且慢，v13 没文字保留）
- ❌ **段 5 时间锚点省略**（0.3/0.6/0.9/1.2/1.5s 必填，模型不会自动加微动画）
- ❌ **EN_COLOR_DESC 字段空**（脚本兜底 = "彩色" = 文字颜色失控 = 翻车）
- ❌ **C 子 agent 写 prompt_draft 字段**（v1.0.3+pic12 强约束，C 只产 11 维原料，主 agent 填模板）
- ❌ **段 2 不写"朗读 X 词"指令**（v1.0.5+pic14 实战新增 · Pic7 Horse R7 翻车）——seedance 把"朗读 '家族词组'"当必生成朗读指令 → 5 词家族词组 = 5 次独立朗读 → 抢节奏 + 重复发 + 错乱。**v6 段 2 = 镜头描述 + 拟声**（音效嵌入视觉句，用 `,` 串接），**不**是朗读台词表。**TTS 朗读 = 后期对齐**（不在 prompt 里指挥）。
- ❌ **段 4 不写具体音效清单**（v1.0.5+pic14 实战新增）——段 4 BGM 段 = **声音策略的兜底句**（A 档"无任何背景音乐、无旁白人声、无哼唱" / B 档"画面元素动作音效保留...，不发完整朗读，TTS 后期对齐"），**不**是拟声清单。**具体音效描述**属段 2 镜头描述末尾的拟声字段，**不**属段 4。
- ❌ **段 2 文字保留约束不重复 2 遍**（v1.0.5+pic14 实战新增）——v6 段 2 文字保留 = **1 段独立约束**（与段 5 文字持续可见段**互补不重复**——段 2 锁位置，段 5 锁全程可见 + 微动画），**不要**在 prompt 里复制 2 遍。
- ❌ **段 2/段 5 文字保留 = 模糊描述"参考图原有 · 不在 prompt 重写"（v1.0.7+pic17 实战新增 · Pic7 Horse R7 第 5 轮翻车）**——v6 比 v15 多了段 5 文字持续可见段，所以 v6 有 **2 处文字保留 + 1 处微动画 + 2 处字符顺序浮现 = 5 处** 文字字段必模糊化。**修复方向**同 v15 红线 #97：段 2 文字保留 + 段 5 文字持续可见 + 段 5 微动画 + 段 5 字符顺序浮现 en_word[0-3] + 段 5 字符顺序浮现 zh_word = **5 处全改"参考图原有 · 不在 prompt 重写"模糊占位**。**判断口诀**：**"v6 5 处文字字段模糊化 = 跟 v15 同样要求 · 段 5 文字持续可见段 = 必模糊"**。
- ✅ **主 agent 必填 12 变量**（v6 比 v15 多 EN_COLOR_DESC + 字体描述）
- ✅ **fill 脚本必兼容 en_color 字符串 + en_color_pattern 字典**（C 子 agent 输出格式有 2 种）

## 反模式

- ❌ "领读型用 v15 就够" → **不**允许，**领读型 = v6 必跑**（缺段 5 = 翻车）
- ❌ "段 5 太长，简化" → **不**允许，**3 个微动画必含**（呼吸明暗 + 字符浮现 + 整体浮动）
- ❌ "字符顺序浮现不重要" → **不**允许，**领读锚点 = 按朗读节奏逐步浮现**（小朋友跟读视觉提示）
- ❌ "字体描述省略" → **不**允许，**段 5 字体必填**（撕纸拼贴圆润无衬线 = 绘本默认风格）
- ❌ **"段 2 我写'朗读 家族词组/第2词/第3词'指令"** → **不**允许（v1.0.5+pic14 实战新增）——seedance 把每个"朗读 X 词"当独立朗读指令 → 5 词家族词组 = 5 次独立朗读 → 抢节奏 + 错乱。**TTS 朗读 = 后期对齐**（不在 prompt 里指挥）。
- ❌ **"段 4 我加'卡片高亮配叮/图标浮现配叮咚/...'"** → **不**允许（v1.0.5+pic14 实战新增）——拟声属段 2 镜头描述，不是段 4 BGM 段。**段 4 只写 B 档兜底标准句**。
- ❌ **"段 2 文字保留我复制 2 遍以强调"** → **不**允许（v1.0.5+pic14 实战新增）——**段 2 文字保留 1 遍（位置锁） + 段 5 文字持续可见段 1 遍（全程可见 + 微动画）= 各 1 遍 = 互补不重复**。不要把段 5 的"全程可见"复制到段 2 强化。

**实战教训**（2026-06-09 Pic7 Horse R7 v1.0.4 重跑）：clip4 prompt 同时违反上述 3 条 = 视频跑出"完全错乱的语音"。**修复**：删"朗读 X 词" + 段 4 改 B 档兜底句 + 段 2 文字保留只留 1 遍 → 重跑（task `cgt-20260609201914-stqmd` · 5.6MB · 14s）→ 声音错乱问题闭环。
