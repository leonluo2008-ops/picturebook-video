# 案例 #34 · 文档/代码 3 处对齐(防"全静音"类翻车)

**场景**: v5.0.4 Step 6.1 批量示例代码写 `--generate_audio true` + Step 6.2 参数表写 "true(绘本有声场景)"。

**翻车**: 但 `seedance_mcp/seedance_uploads.py:472` 代码默认值是 `body["generate_audio"] = False` + `mcp_server.py:100` docstring 写 "绘本默认 false" = **3 处自相矛盾** → 云服按文档跑 = 实际跑出来 = 跟用户期望相反 = "全静音"翻车。

**修复**: **写文档前必 grep 代码 / 写示例前必看默认值 / 3 处对齐必走 / 不齐先停问用户**。

**3 处对齐**(文档 / 代码注释 / 默认值):
1. 写示例代码前 = `grep -E "默认值|default" <code.py>` 查代码实际默认值
2. 写参数表前 = 对照 `mcp_server.py` docstring 里的 schema 默认值
3. 写注释前 = 看代码上下文确认注释还成立
4. **不一致时 = 先改代码再改文档 = 不允许"文档写一套 / 代码跑另一套"**

**反模式**:
- ❌ 凭"应该 = X"的直觉写参数表(不查代码默认 = 翻车)
- ❌ "代码注释说 X = 老代码 = 应该改 Y"(没用户拍板前不许动默认值)
- ❌ "文档和代码哪个对 = 看哪个新"(按时间判断 = 错)

**对应**: SKILL.md 流程纪律 / `agents/interface.yaml` 维护 / 任何 skill 改写前。
