# Clip 衔接问题：实战诊断与解决

> 2026-05-26 实战记录：火把节绘本视频Clip 4-6生成后发现各段视频之间无法有效衔接，总是突然结束。

---

## 问题根因

当前工作流每个Clip单独生成，结尾和开头之间没有「接力设计」。模型在生成每个Clip时从头开始动作（人物站好、场景静置），独立随机，导致连起来像10张GIF随机播放。

---

## 三种解决模式

| 模式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **模式A：独立Clip（问题根源）** | 每个clip单独生成 | 操作简单 | 衔接必断，不推荐 |
| **模式B：导演模式多镜头时间线** | 一个prompt写多个分镜 `[00-05s] Shot 1: ... [05-10s] Shot 2: ...` | 模型自动转场衔接，一致性最强 | 单次上限10-15s |
| **模式C：尾帧接力** | Clip N尾帧用 `--return-last-frame` 生成 → 作为Clip N+1首帧 `--image`驱动 | 物理保证连贯，跨API上限也可接 | 需要额外生成尾帧图 |

### 模式B + C结合（推荐）

当总时长超过即梦单次上限（约10-15s）时：
1. 每段prompt内部用时间线写法
2. 段间用 `--last-frame` 尾帧接力

---

## 分镜脚本衔接设计规范

生成Clip前，必须在分镜脚本里写清楚：

```
Clip N
├─ 开头：承接 Clip N-1 结尾 → 【具体动作描述】
├─ 中段：【当前场景核心动作】
└─ 结尾：【出动作】→ 下一Clip N+1开头要从这个动作接
```

示例（火把节Clip 4→5）：

```
Clip 4（庭院日常）
├─ 开头：承接Clip 3结尾 → 男人拿起松木准备制作火把
├─ 中段：女人制作扫帚，男人削木柴、绑火把、浸松脂
└─ 结尾：女人把火把举起 → 火把头朝右

Clip 5（黄昏出发）
├─ 开头：承接Clip 4 → 火把从画面右侧进入，小女孩从右侧接过火把
├─ 中段：小女孩举火炬率队列出发，夕阳下行军
└─ 结尾：小女孩回首望身后 → 镜头推进她的眼神POV
```

## 即梦API限制速查

| 参数 | 限制 |
|------|------|
| 单次生成时长 | 4-15s（超出报错） |
| 导演时间线上限 | ~15s（一个prompt内可写多段时间线） |
| 总时长67s如何处理 | 分4-5个prompt，每个内部用时间线，段间尾帧接力 |

---

## 关键教训

1. **先设计衔接再生成**：不能在clip生成完后再想办法拼接，必须在分镜阶段就设计好「出/入动作」
2. **即梦API上限是物理约束**：导演时间线单次上限约10-15s，超时分段+尾帧接力
3. **尾帧图生成**：用 `seedance.py create --return-last-frame` 生成尾帧，不接受手动上传尾帧图片
4. **分镜脚本是衔接设计的载体**：不是生成完clip再补救，而是生成前就在脚本里写清楚
