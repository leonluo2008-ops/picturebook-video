### 第 3 步：7 个视频全部翻车 = 5 秒慢推 + 单镜头 + 文字一动不动 ❌

**用户目检原话（最严厉的纠错信号）**：
> "**我看了你的这个视频，我明确告诉你，你生成这一批视频，使用了之前过时的镜头画面设计方式，导致了镜头非常的呆板，缺乏动感，没有故事性，非常垃圾，只有简单的推镜头，而且只有单镜头，这根本不能用！**"
>
> "**请一定不要使用这种镜头设计方式。我们在V15版本，包括后面的V9，就是V15这一系列的后续的镜头设计，都没有再使用过这种方式。我不知道你他妈从哪里翻出了这个东西，你把这个东西给老子。全部清理掉。**"
>
> "**使用正确的镜头设计方式。镜头是服务于故事，服务于叙事，不要用这么死板的镜头设计。你应该使用导演思维来设计镜头。我们V15系列里面。对镜头语言的设计就非常的好，你只要把V15里面关于镜头设计的东西，结合官方文档正确的写法，修正声音、音效。等一系列的东西，就可以了**"

**根因分析**：
1. v6 简化版"整段不分镜"= **零分镜时间线** = 单镜头死板推镜头
2. v6 简化版没"主体命名标签" = 模型不知道主角是谁 = 自由发挥
3. v6 简化版"镜头缓慢平稳推进" = 单一运镜堆叠 5 秒 = 死板
4. v6 简化版没"多镜头时间线" = 没故事性
5. v6 简化版 7 个 prompt 几乎一样 = 6 个几乎一样的视频 = 失去绘本节奏

### 第 5 步：写新规范 `分镜设计规范-v15director.md`（v1.0.0）✅

**核心规范 6 段骨架**：
```
[1] 素材角色声明段      @image1 + 风格定义
[2] 主体定义段          将图片1中的 X 定义为 <主角>
[3] 多镜头时间线段      镜头 1-N（按事件切，不按秒数）
[4] 整段音频约束段      全程无 BGM + 音效 <...> + TTS 占位
[5] 画面禁令段          无字幕无水印无 Logo
[6] Storyboard 引导句   模型自行设计合理的呈现节奏
```

**镜头数算法**（v15 + 官方 · 实际可执行版）：
| 时长 | 旁白密度 | 推荐镜头数 | 节奏 |
|---|---|---|---|
| **5s** | 1 短句（5-7 字） | **2-3 镜头** | 1.5s 建立 + 1.5s 单词动作 + 2s 收势 |
| 8s | 1 短句 | 3-4 镜头 | 2s 建立 + 1s 跃入 + 3-4s 单词 + 1s 收势 |
| 12s | 3 词家族 | 5 镜头 | 2-1-3-3-3（Cat 4a v3 标版）|
| 14s | 5 词家族 | 5 镜头 | 2-1-3-3-3-3（Pic4 v2 标版）|

**每镜头 4 逻辑齐全**（官方 doc2 §1.2）：
1. 运镜或镜头切换方式（1 镜头 1 运镜 · 红线）
2. 主体动作与表情（量化：肢体部位+幅度+速度+力度）
3. 位置或空间变化
4. 音频信息（音效 `<...>` 内联每个镜头 · 不集中末尾）

## 🔁 5 条核心教训（写新绘本 prompt 时必查）

1. **v6 简化版"整段不分镜"** = 14s IT 家族特例 OK = **通用绘本 100% 翻车**——别再瞎套
2. **多镜头时间线分镜** = 必填 = 按事件不按秒数（官方 doc2 §1.3 红线）
3. **镜头数 = 5s×2-3 / 8s×3-5 / 12s×4-5 / 14s×5-6**（v15 + 实战验证算法）
4. **每镜头 4 逻辑齐全** = 运镜+动作+位置+音频内联（不集中末尾）
5. **官方 doc2 §4 红线**：**1 镜头只 1 种运镜**（不要同时推拉摇移）
