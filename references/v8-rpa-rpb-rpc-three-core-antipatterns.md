## 反模式 1：原地动词陷阱（RP-26a）

### 反模式特征
提示词里写：
```
小狗在花丛里欢快地小跑
小狗越来越快地飞奔
小狗在雨中笑着跑过草地
```

### 失败机制
- "欢快地小跑" / "飞奔" / "跑过" = **不带方向的动词**
- 模型收到指令后只能**"原地循环"模拟这些动作**
- 结果 = 小狗在原地反复做"小跑姿态" / "加速姿态" / "跑过姿态"

### 正模式
```
小狗从画面左侧跑向右侧，耳朵竖起
小狗跑出画面，尾巴飘起
小狗跑过草地后离开镜头
```

**关键要素**：每个动词都必须带**空间目标**（向 X / 出画面 / 离开镜头）。

### 跨本验证
- Run 跑 Clip 1 v7 = 失败（原地循环）
- Run 跑 Clip 1 v8 = 通过（带方向 = 跑出画面）

## 反模式 2：情绪修饰词陷阱（RP-26b）

### 反模式特征
提示词里写：
```
欢快地跑
笑着跑
脚步轻快
兴奋地飞奔
```

### 失败机制
- "欢快" / "笑着" / "轻快" = **抽象情绪词**
- 模型不知道具体怎么做 = 只能"原地循环"模拟情绪包
- 结果 = 小狗在原地做"欢快表情包"循环

### 正模式
```
耳朵竖起（被叫到）→ 身体前倾（准备跑）→ 尾巴飘起（跑得欢）
腿抬起加快步频（越跑越快）→ 身体远离镜头（消失在远方）
```

**关键要素**：情绪 = **1 组具体身体动作**，不是 1 个修饰词。

### 跨本验证
- Run 跑 Clip 1 v6 = 失败（误删情绪外化）
- Run 跑 Clip 1 v7 = 失败（用"笑着"等修饰词）
- Run 跑 Clip 1 v8 = 通过（用"耳朵竖起/前倾/尾巴飘起" = 1 组身体动作）

## 反模式 3：故事弧缺失（RP-26c）

### 反模式特征
多镜头 Clip 里：
```
镜头 1：小狗欢快地跑
镜头 2：小狗越来越快地跑
镜头 3：小狗在雨中跑
```
3 个镜头 = **同一只狗 + 3 次"快"** = 节奏平 = 没有故事弧。

### 失败机制
- 镜头之间没有"开胃→刺激→满足"的关系
- 用户观看时感受不到**节奏起伏** = 觉得"平" = 不好

### 正模式
**4 步固定结构**：
```
出现（开胃）→ 转折（刺激）→ 高潮（满足）→ 收尾（回味）
```

**v8 Clip 1 应用**：
```
镜头 1 = 出现（耳朵竖起 = 被 RUN 叫到）
镜头 2 = 转折（身体前倾跑向右侧 = 开始跑）
镜头 3 = 高潮（雨幕中跑出画面 = 越跑越快）
```

### 跨本验证
- Run 跑 Clip 1 v7 = 失败（3 镜平铺 = 节奏平）
- Run 跑 Clip 1 v8 = 通过（出现→转折→高潮 = 完整弧）
- v7 Clip 2 猎豹 = 已具备完整故事弧（用户认可）
