# 达尔文 dry_run 实践（picturebook-video v0.7.0 · 3 轮全 keep）

> **背景**：达尔文 skill 跑 picturebook-video v0.7.0 精简版，子 agent 不可用降级 dry_run 评估，**3 轮全 keep · 总分 +9.0**（87.5 → 96.5）
>
> **适用**：其他 skill 跑达尔文缺子 agent 时的 dry_run 评估参考

## 2. test-prompts 重写关键

**问题**：旧 test-prompts 是"检索 v7/v14 范式"——v0.7.0 砍了 v3-v11 范式文件，**旧 prompt 触发不到**。

**修复**：基于 v15/v15.1 范式重写 5 个 test-prompts：

| 场景 | 触发铁律 | 命中点 |
|---|---|---|
| 1. v15 范式入口 | 启动前 6 必问 | "🎯 v15 系列范式"段 + 3 必问 + 3 默认 |
| 2. 长旁白拆分 | #36 #37 #38 | v15.1 增补 + 切分表自检 + 段 2 标版 |
| 3. 切分表自检 | #38 | 异常处理 fallback + 4 项清单 |
| 4. 末帧铁律 | #36 | v15.1 第 4 条 + 段 2 标版末帧约束 |
| 5. 旁白朗读时长 | #32 #33 | 4 必跑第 1 条 + 决策树 |

**JSON 格式**（`test-prompts.json`）：

```json
[
  {
    "id": 1,
    "prompt": "用户会说的话",
    "expected_should_trigger": "SKILL.md 应引导到 X 段",
    "expected_reason": "为什么这么触发",
    "scenario": "实测场景名",
    "difficulty": "标准/复杂",
    "covers": "覆盖什么能力"
  }
]
```

## 5. dry_run 评分规则

| 维度 | dry_run 评分方法 | 风险 |
|---|---|---|
| 1-7（结构）| 主 agent 直接读 SKILL.md 打分 | **低**（结构是静态可分析的）|
| 8（实测）| **干跑** 5 个 test-prompts，模拟"读完 skill 后执行 prompt 的思路" | **中**（干跑是 LLM 推演，可能跟实际跑有偏差）|

**降级 dry_run 标注**：

```tsv
2026-06-05T20:50	0549ad0	picturebook-video	87.5	91.0	keep	边界条件+实测表现	...	dry_run
```

`eval_mode=dry_run` 字段明确标识，跟 `full_test` 区分。

## 6. 棘轮机制记录

| 轮 | status | 加权分 | 关键改动 |
|---|---|---|---|

**如果某轮 revert**（改后 ≤ 改前）：

```bash
# 1. 标记 status=revert
git revert HEAD --no-edit  # 创建新 commit 回滚，不 reset --hard

# 2. 记录到 results.tsv
echo -e "2026-06-05\tNEW\tskill\told\tnew\trevert\tdim\t原因\tdry_run" >> results.tsv
```
