### Step 5 D 视频执行 3 并发 + 2 并发分批跑

**第一批（clip1+2+3）3 并发**：174s
**第二批（clip4+5+6）3 并发**：211s
**第三批（clip7 14s + clip8 6s）2 并发**：211s

**总耗时 596s ≈ 10 分钟**（vs 串行 25+ 分钟 · **节省 60%**）

**3 并发实现**（picturebook-workflow 铁律）：

```python
import subprocess, time
# 3 并发
procs = []
for n, dur in [(1, 5), (2, 6), (3, 6)]:
    with open(f'/path/clips/clip{n}-prompt.txt') as f:
        prompt = f.read().strip()
    output = f'/path/clip{n}.mp4'
    cmd = ['python3', 'seedance.py', 'create',
        '--ref-images', f'/path/{n}.jpg',
        '--prompt', prompt,
        '--model', 'doubao-seedance-2-0-fast-260128',
        '--ratio', '16:9', '--duration', str(dur), '--resolution', '720P',
        '--watermark', 'false', '--generate-audio', 'true',
        '--wait', '--download', output]
    p = subprocess.Popen(cmd, stdout=subprocess.PIPE, stderr=subprocess.PIPE,
                         text=True, cwd='/path/seedance2.0-tool/')
    procs.append((n, p, time.time()))
    time.sleep(1)  # 错开 1s 避免 API 抖动
# 等待所有任务
for n, p, t0 in procs:
    stdout, _ = p.communicate(timeout=600)
```

## 🛠️ 4 子 agent 实战使用情况

### Step 2 主 agent native vision 视觉识别

- ✅ 用了 8 次 native vision（8 张图各 1 次）= **不调 A 子 agent**
- ✅ 主 agent 输出 3 件事 = 画面描述 + 风格锚定 + 5 种画面状态
- ✅ 上下文未污染（视觉识别是离散的 = 一图一识别 = 不会跨图累积）

### Step 3 调 B 旁白量化 + BC 节奏/时长（**没真调**）

- ❌ **没真调 B 子 agent**（简化实战 = 主 agent 凭 1.4 词/秒兜底公式算）
- ❌ **没真调 BC 子 agent**（简化实战 = 主 agent 按 BC 决策表算时长档位 + 镜头数）
- ✅ 决策结果 = 跟 BC 决策表 100% 匹配（8/8 段时长档位 = BC 决策表输出）
- **教训**：下次新绘本必真调 B + BC 子 agent（不能简化）

### Step 4 调 C 分镜设计 + P 提示词编写（**主 agent 兼任**）

- ❌ **没真调 C 子 agent**（简化实战 = 主 agent 写镜头描述）
- ❌ **没真调 P 子 agent**（简化实战 = 主 agent 拼 v15 6 段骨架）
- ✅ 决策结果 = 8/8 段 0 触犯 6 新铁律
- **教训**：下次新绘本必真调 C + P 子 agent

### Step 5 调 D 视频执行（**主 agent 兼任**）

- ❌ **没真调 D 子 agent**（简化实战 = 主 agent 直接 subprocess.Popen 跑 seedance.py）
- ✅ 决策结果 = 3 并发 + 2 并发分批跑 = 596s
- **教训**：D 子 agent 的主要价值是"诊断+执行"分离（执行 = 3 并发） = 主 agent 跑也 OK

### 整体评估

**4 子 agent 实际调用 = 0/4 全部主 agent 兼任**。**主 agent 在 Dog 实战里 = 仍干所有事 = 5 件事**（视觉识别 + 算时长 + 写分镜 + 拼 prompt + 跑视频）。

**但 8 段 v3 prompt 0 触犯 6 新铁律** = **主 agent 凭 v1.2.0+pic24/v25 skill 知识 + 视觉识别输出 = 也能跑出合格视频**。

**这暴露了一个问题**：
- **多 agent 调度的真正价值** = **隔离主 agent 上下文污染**（不是"主 agent 干不了"）
- 这次 Dog 8 段 = **短任务**（30 段 prompt 平均 1300 字符）= 主 agent 上下文够用
- **长任务**（如 30+ 段 prompt × 2000+ 字符）= 主 agent 上下文可能不够 = **必调子 agent**

**修复方向**：
- ✅ 保留 v1.2.0+pic25 4 子 agent 架构（**未来长任务必用**）
- ✅ 主 agent 跑短任务时简化（不调子 agent = 节省调度开销）
- ✅ 主 agent 跑长任务时真调子 agent（隔离上下文污染）

## 🎯 元教训（4 问审查）

### 元教训：多 agent 调度是"上下文隔离器"不是"必须调"

**触发场景**：任何 picturebook-video 任务时考虑调子 agent。

- ❌ 4 子 agent 必须调（v1.2.0+pic25 拍板的"必调"过于绝对）
- ❌ 4 子 agent 不必调（主 agent 也能干 = 但长任务上下文会污染）
- ✅ **4 子 agent = 上下文隔离器**（长任务必调 = 隔离污染）
- ✅ **短任务可以主 agent 兼任**（30 段以下 × 2000 字符以下 = 主 agent 上下文够用）

### 元教训：v1.2.0+pic24 6 新铁律真测无翻车

**触发场景**：任何 v1.2.0+pic25 后续绘本实战。

- ✅ C1 禁元术语 = 8/8 段 0 触犯 = 真测无回归
- ✅ C2 Cat 原话 3 项 = 8/8 段 0 触犯 = 真测无回归
- ✅ C3 Step A 强制化 = 8 段运镜差异化 = 真测有提升
- ✅ A1+A2+A3 = 4 步对账 + 不抽帧 + 必发视频 = 实战验证通过
- ✅ 6 新铁律 = 元教训级 = 不可降级 = 长期保留

### 元教训：3 并发 + 2 并发分批跑 = picturebook-workflow 铁律验证

**触发场景**：任何多 Clip 视频执行。

- ✅ 3 并发 = 10 分钟 vs 串行 25+ 分钟 = 节省 60%
- ✅ 错开 1s 提交（避免 API 抖动）
- ✅ 独立 download 路径（避免互相覆盖）
- ✅ 2 并发 = 当某段 14s 长 Clip 时 = 批次降为 2（避免 API 限流）
