## 背景

v0.7.1+pic7 时代 picturebook-video skill 工作流 = **单 agent 串行干全部**：

```
主 agent 接收需求
  ├─ 1. 调 lark-cli 下载素材
  ├─ 2. vision_analyze 8 张图
  ├─ 3. 文本匹配简介旁白
  ├─ 4. 自己算旁白朗读时长（凭印象 + 算错）  ← 根因 7
  ├─ 5. 自己选节奏档位（凭印象 + 选错）      ← 根因 8
  ├─ 6. 自己拼 v15 范式 prompt（残留冗余）  ← 根因 9
  ├─ 7. seedance 提交
  ├─ 8. ffmpeg 抽帧
  ├─ 9. vision_analyze 抽帧诊断
  └─ 10. 自己判断翻车 + 自己改
```

**问题本质**：单一 agent 串行 = **每步都被上一步污染**：
- 第 4 步算朗读时长 ← 第 3 步看到旁白文本时凭"语感"估算，没量化
- 第 5 步选节奏 ← 第 4 步算错就直接错（**错进错出**）
- 第 6 步拼 prompt ← 第 5 步节奏错了就多塞"建立+跃入"两个空镜
- 主 agent 上下文里同时有"前一个绘本跑通"+"节奏元原则"+"标版要改"等多条记忆**互相干扰**（**根因 10**）

## 4 个新根因（v0.7.1+pic7 沉淀）

| 根因 | 问题 | 根类型 | MVP 处理 |
|---|---|---|---|
| **7** | 主 agent 凭印象算旁白朗读时长 | 方法不当 | ✅ **拆 B 旁白量化子 agent**（强制 TTS 实测或字数 × 0.15s 公式）|
| **8** | 节奏档位表有但套错（短句套长档）| 方法不当 | ✅ **铁律 #43** + **拆 B 子 agent**（算完时长直接查强制档位表）|
| **9** | v15 范式 prompt 残留"建立+跃入"两个空镜 | 方法不当 | ✅ **拆 C 分镜设计子 agent**（按节奏档位动态选镜头，**短句自动删空镜**）|
| **10** | 单 agent 上下文污染 | 流程缺失 | ✅ **拆 4 个子 agent + 主 agent 退化为调度员** |

## 重构路线 · 4 个子 agent 协作架构

### 拆分原则（4-skill 工具包融合用法）

- **skill-creator** 主管"写"：4 份子 agent SKILL.md 编写（brief 模板 + 输出 schema + 示例）
- **skill-organizer** 主管"整理"：拆分 references/ 到对的位置（别堆主 SKILL.md）
- **gardener-skill** 主管"根因归档"：本次重构的"为什么"（4 个根因 7/8/9/10）
- **darwin-skill** 主管"evals 量化"：v0.7.1+pic7 vs v1.0.0 的 evals 对比（**用户说"不重跑视频" → 跳过 darwin**）

### 4 个子 agent 分层

| 层级 | 子 agent | 输入 | 输出 | 不做 |
|---|---|---|---|---|
| **L1 · 并行** | **A · 风格识别** | 简介 + 8 张图分析 | `{调性, 节奏倾向, 风格锚定词}` | 不算时长 |
| **L1 · 并行** | **B · 旁白量化** | 旁白文本 | `[{句, 时长, 复杂度, 词组类型}, ...]` | 不选节奏 |
| **L2 · 依赖 L1** | **C · 分镜设计** | A 输出 + B 输出 + 图数 | `[{clip_n, 节奏, 镜头表, prompt 草稿, 自检}, ...]` | 不跑视频 |
| **L3 · 依赖 L2** | **D · 视频执行** | C 输出的 prompt | `{task_id, mp4, 抽帧报告, 是否翻车}` | 不算时长不选节奏 |

**关键约束**：
- A/B **并行**（L1 同时启动）
- C 等 A+B 都完成（L2）
- D 等 C 完成（L3）
- **绝不一次性下发 A+B+C+D**（避免子 agent 内部又自己串行 = 等于把单 agent 问题搬到子 agent 内部）

### 主 agent 退化为调度员

```
主 agent = 接收需求 + 调度子 agent + 汇总 + 报告
不直接做 A/B/C/D 任一步
```

**调度代码（伪）**：

```python
# L1 并行
task_a = delegate_task(skills=['A-style'], brief=...)
task_b = delegate_task(skills=['B-narration'], brief=...)
a_report = wait(task_a)
b_report = wait(task_b)

# L2 依赖 L1
task_c = delegate_task(skills=['C-storyboard'], brief={...a_report, ...b_report})
c_report = wait(task_c)

# L3 依赖 L2（可能 N 次重试）
for retry in range(3):
    task_d = delegate_task(skills=['D-video'], brief=c_report)
    d_report = wait(task_d)
    if d_report.翻车 == False:
        break
    # 翻车只重发 D（不重发 A/B/C，因为输入没变）
```

**调度员唯一需要做的决策**：
- 翻车判定 = 看 D 报告（结构化字段 `是否翻车`）
- 重试 = 重发 D（**不重发 A/B/C**，因为它们的输入没变）
- 风格/节奏/分镜错了 = 重发对应子 agent（**A 错只重跑 A，B 错只重跑 B**）

### 4 份子 agent SKILL.md 编写规范

按 `skill-creator` 规范：
- **name + description**（触发词，**pushy 一些**——主 agent 别忘了加载）
- **brief 模板**：明确输入字段（如 B 旁白量化要 `旁白文本` + `语言` + `TTS 时长`（可选））
- **输出 schema**：结构化 JSON，不是自然语言
- **示例**：1 个真实跑通的样例（如 B 跑通 "Please, Mama, read!" → 输出 `{朗读: 2.5, 复杂度: 标准词组, 静默系数: 1-1.5}`）
