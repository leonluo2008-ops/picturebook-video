## 纠错 1 · **数字方案必查物理约束**

**用户原话**："**你竟然写出了 18 秒的 clip，这肯定是有问题的，这是一个严重问题**"

- 任何 Clip 时长 / API 参数 / 文件大小 = 写进方案前**必查 schema 硬上限**
- seedance duration schema = `minimum=4, maximum=15`（API 拒收 > 15s）
- **凭印象编数字 = 严重违规**
- 写"X 时长 Clip"前 = 必跑 `python3 ~/.hermes/profiles/huiben/skills/creative/seedance2.0-tool/seedance.py create --help` 看 schema
- 例：算出 18s 必拆成 2 个 ≤ 15s · **不**报"18s"给用户

**反模式**：凭印象写"18s Clip" / "8s 是默认上限" / "20s 应该也能跑" · 全错。

**修复 SOP**：
1. 写 Clip 时长前 = 必查 seedance schema（duration minimum=4, maximum=15）
2. 算出 > 15s = 必拆 2 个 ≤ 15s · 不报"18s"方案
3. 算 < 4s = 必拼合（拼到相邻 Clip 或扩到 ≥ 4s）

**判断口诀**：**"seedance duration 4-15s 整数 · > 15 必拆 · 凭印象 = 违规"**

## 纠错 2 · **vision 必全 N 张图（不抽样）**

**用户原话**："**必须识别所有的图片，而不是只识别部分图片，这样你才能够根据图片来编写提示词**"

- 收到 N 张图 = vision **N 次** · 1 张都别跳
- 抽样 1/N/中/末 = 凭印象拼 prompt = 翻车征兆
- 写 `image-inventory.md`（图 | 风格 | 主体 | 背景 | 文字 | 动作 | 场景跳变 7 列）= 防止漏看

**反模式**：8 张图只 vision 1/中/末 3 张 = 凭印象猜图 3/4/5/6/7 内容 = 拼 prompt 错位。

**修复 SOP**：
1. 收到 N 张图 = 立即 vision **N 次**（不并行 · 不抽样）
2. 写 `image-inventory.md`（7 列必填）
3. 失败重试 1 张（临时过载 vs 真挂）· 4+ 全失败 = 降级 v7 模板兜底

**判断口诀**：**"N 张图 = N 次 vision · 1 张都别跳 · 写 image-inventory"**

## 纠错 3 · **画面时长匹配旁白时长**

**用户原话**："**这个重要的一点是根据对应的旁白时长，就是你的画面对应哪一句旁白，你要保证那句话和画面能够基本上匹配上**"

- Clip 时长 = 对应故事所有段 TTS 自然朗读总时长（不是凭印象算）
- TTS 兜底公式：**1.0 词/秒（英文）+ 3 字/秒（中文）+ 0.5s 词间停顿**
- ~~1.4 词/秒兜底~~ = 不用（过快估 = 漏算中文 + 漏算停顿）
- 用户给 TTS 数值 = **优先级最高**（视频总时长 = TTS + 5s 冗余 · 不缩短）

**反模式**：凭印象算 Clip 时长 = 写 14s 装 1 短句 3.5s TTS = 节奏拖慢 / 写 7s 装 4 词家族 13.67s TTS = 物理装不下。

**修复 SOP**：
1. 写 11 维 JSON 前**必先**按 1.0 词/秒 + 3 字/秒 + 0.5s 词间停顿公式算每段 TTS
2. 写每 Clip 时长 = 对应故事所有段 TTS 总和 + 视觉延展
3. 视频总时长 = 用户给 TTS 优先（兜底公式总和次之）+ 5s 冗余
4. **不缩短**视频（铁律）

**判断口诀**：**"Clip 时长 = TTS 总和（1.0 词/秒+3 字/秒+0.5s 词间停顿）· 视频 ≥ TTS"**

## 纠错 4 · **静默 = 编排工具不是铁律**

**用户原话**："**你要忘记那个静默，因为现在我们基本上是按照叙事故事情节，拉通整个来 ... 应该根据编排的故事来合理的设置有静默或者没有静默，不是每一个都要去静默的**"

- 静默 = 编排工具（按故事连贯性决定）· **不**是每段必塞 ≥ 2s 末帧静默
- 故事连贯 = 无静默 · 视觉断点 = 短静默 · **不**硬塞 v5 公式
- 旧铁律 #69（v5 公式"朗读完最低 3s + 末帧静默 ≥ 2s"）= **降级**为参考值，不是铁律

**反模式**：每段硬塞末帧静默 ≥ 2s = 短句旁白也加 2-3s 静默 = 节奏拖慢 = 用户体感"节奏卡顿"。

**修复 SOP**：
1. 写 11 维 JSON `end_frame_microaction.duration_seconds` = 按故事连贯性填（0 或 1-2s）
2. 故事连贯（同一场景连续镜头）= 无静默（duration_seconds=0）
3. 视觉断点（场景切换 / 收势）= 短静默（1-2s）
4. 末帧微动 = 必填（不允许定格海报 · 静默时也要微动）

**判断口诀**：**"静默 = 编排工具 · 故事连贯 = 0 静默 · 旧 v5 公式降级为参考"**

## 纠错 5 · **1 个故事 = 1 个 Clip（不按段拆）**

**用户原话**："**应该把参考图和旁白该合并成一个 clip 的，就要合并成一个 clip，合并之后，那就是一个故事，就不要有那么多的硬性的静默**"

- 段数 ≠ Clip 数 · **1 故事 = 1 Clip**（视觉连续 + 朗读连贯 = 合并）
- 按"段"机械拆 = 翻车（押韵词对 "quack! + truck" 应该合并成 1 个故事）
- 视觉跳变（水→陆）= 拆 Clip（不算"按段拆"= 是按"场景跳变拆"）

**反模式**：
- 8 段旁白 = 8 Clip（按段机械拆 = 节奏碎）
- 8 段旁白 = 2 Clip（强合 = 信息量爆炸）
- 押韵词对"quack + truck"拆 2 Clip = 押韵连贯断

**修复 SOP**：
1. 通读旁白 1 遍 → 标叙事弧（通常 3-5 弧）
2. 每个弧 = 1 个 Clip（视觉连续 + 朗读连贯）
3. 视觉跳变（水→陆）= 按场景跳变拆
4. 总 Clip 数甜区 = 3-5（> 5 合并叙事弧 · < 3 拆分叙事弧）

**判断口诀**：**"1 故事 = 1 Clip · 段数 ≠ Clip 数 · 视觉跳变 = 拆 · 押韵连贯 = 合"**

## 纠错 6 · **家族词组中文 TTS 必算**

**用户原话**："**你在计算家庭词组的时候是不是没有计算中文的阅读时间？你是不是把中文给全部去掉了？**"

- 家族词组（≥3 同字母家族词）/ 多词集合段 = **中文 TTS 必算**
- 段 7 例：`duck, truck, luck, buck` + `鸭子 duck 的 UCK 家族，duck、truck、luck、buck 大集合！` = 中文 8 字 + 英文 8 次发音 + 6 个 0.5s 词间停顿 = **13.67s**（不是我之前估的 4.29s）
- 把"中文说明文字"当 0s = **严重错误**（TTS 真要读的 = 必算）

**反模式**：把"中文家族说明"当 0s = 段 7 算 4.29s · 实际 13.67s = 差 3 倍 = 视频装不下 TTS = 黑场翻车。

**修复 SOP**：
1. 算每段 TTS = 英文字数/1.0 + 中文字数/3.0 + 0.5s × (词间停顿数)
2. 家族词组 = 词间停顿必算（4 词 = 6 个 0.5s = 3s 停顿）
3. 中文字符数 = 全文算（含 `duck/UCK` 等夹杂的英文部分单独算）
4. 算完 TTS 再写 Clip 时长

**判断口诀**：**"家族词组 = 词间停顿 0.5s/间隔 + 中文字数 必算 · 不漏中文"**
