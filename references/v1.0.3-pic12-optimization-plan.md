# picturebook-video skill 优化方案（v1.0.2 → v1.0.3+pic12）

> **核心原则**（用户 2026-06-07 纠错）：**不破坏 4 维底层核心逻辑 + v15 4 段骨架模板**。所有优化 = **在 4 维参数上做"领读绘本"场景的具体微调**。

---

## 0. 优化原则

| 原则 | 说明 |
|---|---|
| **不破坏 4 维底层** | 时间/风格/角色/声音 4 维控制原理是底层核心（用户根本性纠错）|
| **不破坏 v15 4 段模板** | 段 1 主体定义 + 段 2 分镜+文字保留 + 段 3 风格锁定 + 段 4 BGM 段 = 固定结构 |
| **只微调 4 维参数** | 整数时长（v6）/ 文字全程（v6）/ **情绪基调 AI 判定（v7·见下）/ 末帧静默（v5+）**= 在 4 维上做具体化 |
| **加铁律不破结构** | 新增铁律 #59-#64 = 在已有铁律基础上扩展，不冲突 |
| **沉淀实战** | pic4 v6 = 5 次迭代踩坑沉淀，每条铁律都有翻车教训支撑 |

> **v7 修正（2026-06-07 用户纠错）**：原"铁律 #62 表情温柔化"作废。绘本情绪基调**不硬编码温柔化**——温柔/严肃/欢快/惊险等**由题材类型决定**，A 风格识别 agent 必须**初步判断 + 推荐给用户确认**，不能替用户拍板。
> **修正根因**：原铁律 #62 是用 No 绘本单题材反推的硬规则，但绘本有多种类型（警示向/温情向/冒险向/知识向），不同类型需要不同情绪基调，AI 替用户决定 = 剥夺用户选择权。

## 1. v1.0.2 → v1.0.3+pic12 升级清单

### 1.1 新增 5 条铁律（铁律 #59-#63 → 含 #64 AI 判断标准）

**铁律 #59 · 整数时长铁律**：
- **内容**：seedance 不生成小数时长（5.1s → 6s, 6.5s → 7s）；设计时长必须是整数 5/6/7/8/9/10
- **根因**：pic4 v5 设计 5.5s 末帧消化 → 实际跑 6s → 设计 5s 装不下 6s 内容
- **修复**：v6 整数时长公式（5/6/7/8）
- **沉淀位置**：`picturebook-video/SKILL.md` §铁律清单

**铁律 #60 · 彩色文字全程可见铁律**：
- **内容**：领读绘本彩色文字必须从 t=0 到 t=末帧 全程可见 + 微动画（呼吸式明暗交替 0.5s/次 + 字符顺序浮现按朗读节奏 0.3s/字）
- **根因**：pic4 v5 文字 1-2s 后消失 = 失去领读锚点
- **修复**：v6 prompt 新增"文字持续可见段"
- **沉淀位置**：`picturebook-video/SKILL.md` §铁律清单 + `references/v15-4段骨架-模板.md` 加 v6 字段说明

**铁律 #61 · 末帧静默 ≥ 2s 铁律**：
- **内容**：所有绘本 clip 末帧静默必须 ≥ 2s（用户底线"朗读完留 2s 静默消化"）
- **根因**：pic4 v3 末帧 1.5s 静默不够（用户反馈"末帧还是太短"）
- **修复**：v6 公式 = 朗读完 + 2s 静默（极短档 5s / 短句档 6s / 中句档 7s / 长句档 8s）
- **沉淀位置**：`picturebook-video/SKILL.md` §铁律清单 + `references/2026-06-06-rhythm-action-cost.md` 加 v6 节奏公式

**铁律 #62 · ~~表情温柔化铁律~~（v7 作废）**：
- **v7 作废原因**：原铁律用 No 绘本单题材反推"温柔坚定"，但绘本有多种类型（警示向/温情向/冒险向/知识向），温柔/严肃/欢快/惊险等**由题材决定**，不能硬编码"温柔化"——否则 AI 替用户决定 = 剥夺选择权
- **替代方案**：见下方"铁律 #64 情绪基调 AI 判定铁律"——A 风格识别 agent 必填 `emotion_tone_ai_recommendation` 字段 + 标准判断维度 + 推荐给用户确认

**铁律 #63 · uguu.se 兜底铁律**：

**铁律 #64 · 情绪基调 AI 判定铁律（v7 替换 #62）**：
- **内容**：A 风格识别 agent 必须按"题材类型 → 情绪基调"标准判断 + 推荐给用户确认；用户拍板后才落到 C 子 agent 的 `emotion_tone` 字段
- **判断标准（4 维，AI 必须逐项打分）**：
  1. **题材类型**（核心锚点）：警示向/温情向/冒险向/知识向/治愈向 → 决定基调候选
  2. **画面色彩**（冷暖判断）：冷色调（蓝/灰/紫）→ 偏严肃/惊险；暖色调（黄/粉/橙）→ 偏温柔/欢快
  3. **角色表情静态特征**（基线情绪）：眉头紧锁/嘴下垂 → 偏严肃/警示；微笑/眯眼 → 偏温柔/欢快
  4. **故事走向**（叙事弧）：冲突→警示→保护 → 偏严肃/警示；日常→成长→领悟 → 偏温柔/治愈
- **AI 判定流程**（必跑）：
  1. A agent 看完所有原图 + 故事简介
  2. 4 维逐项打分（每项 1-5 分） → 加权算总分 → 推荐 1 个主导情绪基调
  3. 输出 `emotion_tai_ai_recommendation` 字段到 `style-recognition.json`：
     ```json
     {
       "emotion_tone_ai_recommendation": {
         "primary": "温柔坚定",
         "secondary": "慈爱守护",
         "confidence": 0.85,
         "dimension_scores": {
           "topic_type": 5,
           "color_temperature": 4,
           "character_expression": 5,
           "narrative_arc": 4
         },
         "ai_reasoning": "题材=警示向(5/5);色彩=暖橙+红(4/5);表情=眉头微蹙带笑意(5/5);叙事=冲突→保护(4/5)→ 主调温柔坚定",
         "alternatives": ["温柔坚定", "慈爱守护", "欢快探索"]
       }
     }
     ```
  4. **主 agent 必报用户**（不是直接落 C JSON）："AI 判定主调 = 温柔坚定（4 维加权 4.5/5），备选 = 慈爱守护/欢快探索，是否采用？"
  5. 用户拍板后 → 主 agent 落 C JSON 的 `emotion_tone` 字段 → C 子 agent 写到 prompt
- **沉淀位置**：`picturebook-video/SKILL.md` §铁律清单 + `agents/storyboard-style/SKILL.md`（A 风格识别子 agent）必填字段 + 新增 `references/emotion-tone-ai-judgment-standard.md`

**铁律 #63 · uguu.se 兜底铁律**：
- **内容**：chevereto 图床挂了时改用 uguu.se 备用图床 + 直接 curl 调 ark API
- **根因**：pic4 v6 跑视频时 chevereto 6 次全 timeout
- **修复**：v6 wrapper = `uguu_ark_wrapper.py`（uguu 上传 + 直接 ark API）
- **沉淀位置**：`picturebook-video/SKILL.md` §铁律清单 + `references/public-file-hosting-fallback.md` 加 uguu 优先级

### 1.2 新增"情绪基调判断标准"参考文档（必填）

> **核心**：用户 2026-06-07 二次纠错——"光说 AI 判断不说标准，等于没说"。所以 v1.0.3+pic12 必须把"判断标准"具体化、可操作化，**AI 不是凭感觉**。

**新建 `references/emotion-tone-ai-judgment-standard.md`**（标准库）：

| # | 判断维度 | 操作化定义 | 1 分 | 3 分 | 5 分 |
|---|---|---|---|---|---|
| 1 | **题材类型** | 看故事简介+画面叙事意图 | 纯知识无情绪 | 日常+小情绪 | 强冲突（警示/冒险/惊险） |
| 2 | **画面色彩** | 主色温 = 主导色块 + 笔触 | 冷灰/暗紫 | 中性（米/淡黄）| 暖橙/亮黄/粉 |
| 3 | **角色表情** | 看主角眼睛+嘴+眉头静态基线 | 眉头紧锁/嘴下垂 | 中性 | 微笑/眯眼/柔和 |
| 4 | **叙事弧** | 起承转合情绪走向 | 平铺直叙 | 起→落 | 冲突→警示→保护（强对比） |

**5 类题材 → 情绪基调候选表**（AI 必须先锚题材再选基调）：

| 题材类型 | 推荐主调 | 推荐备选 | 典型代表 |
|---|---|---|---|
| **警示向** | 温柔坚定 | 慈爱守护 | No 绘本（陌生人安全）|
| **温情向** | 慈爱温柔 | 欢喜陪伴 | 猜猜我有多爱你 |
| **冒险向** | 兴奋勇敢 | 紧张期待 | 野兽国 |
| **知识向** | 好奇探索 | 欢快轻松 | 第一次发现 |
| **治愈向** | 温暖陪伴 | 静谧安心 | 房间里的成年人 |

**AI 必跑 5 步流程**（不能跳）：
1. 看完所有原图 + 故事简介
2. 题材类型 → 锁定候选基调（5 选 1-2）
3. 4 维逐项 1-5 打分 → 加权算分（题材 40% + 色彩 20% + 表情 20% + 叙事 20%）
4. 输出 `emotion_tone_ai_recommendation` JSON（4 维分数 + 加权分 + 推理链 + 备选 3 个）
5. **主 agent 必报用户**（铁律 #64 第 4 步）——"AI 判定主调 = X（加权 Y/5），备选 = A/B/C，是否采用？"

**主 agent 报用户的固定话术**（避免信息不足）：
> "AI 看完原图+故事简介，按 4 维标准判定：
> - 题材类型 = 警示向（4-5 分锁定）
> - 画面色彩 = 暖橙主导（4 分偏温柔）
> - 角色表情 = 眉头微蹙带笑意（5 分温柔）
> - 叙事弧 = 冲突→保护（4 分警示）
> **AI 推荐主调 = 温柔坚定（加权 4.45/5）**，备选 = 慈爱守护 / 欢快探索
> **请拍板**：（A）采用温柔坚定 / （B）改用 X / （C）其他"

### 1.6 B 旁白量化 v6 节奏公式

**v6 节奏公式（整数时长版）**：

| 档位 | 旁白朗读 | 公式 | 总时长 | 末帧静默 | 镜头数 |
|---|---|---|---|---|---|
| 极短档 | < 1.5s | `0.5s 铺垫 + 朗读 + 静默` | **5s** | 3.8s | 2 |
| 短句档 | 1.5-2.5s | `1s 铺垫 + 朗读 + 静默` | **6s** | 2.9s | 3 |
| 中句档 | 2.5-3.5s | `1s 铺垫 + 朗读 + 静默` | **7s** | 3.1s | 3 |
| 长句档 | 3.5-5s | `1s 铺垫 + 朗读 + 静默` | **8s** | 3.4s | 3 |

**核心铁律**：
- 整数时长（5/6/7/8 = seedance 实际跑）
- 末帧静默 ≥ 2s（用户底线）
- 铺垫镜头数 2-3（铁律 #54 不主动加）
- 不为建立画面浪费 1-2s
