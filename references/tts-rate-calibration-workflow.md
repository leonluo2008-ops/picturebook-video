# TTS 速率方案校对工作流（绘本视频第 8 个根本性纠错 · 2026-06-13 绘本实战沉淀）

> 用户原话（**关键**）："**你计算的旁白时长总和与 TTS 音频时长进行比对，如果差别不大，比如多三五秒或刚好，说明计算方式正确。如果差别过大，就需要衡量计算方式。在设计 clip 时长时，要进行校对，思考如何操作，先找出中间的计算逻辑。**"

---

## 一句话

**算出 8 段旁白 TTS 总和（1.0 词/秒 + 3 字/秒 + 0.5s 词间停顿）→ 跟用户给 TTS 比对 → 差 3-5s 正常 · 差 ≥ 5s 重算速率公式 · 视频总时长 = TTS + 5s 冗余**（不缩短）

---

## 3 步校对流程（**Step 3 必跑** · 算 TTS 后**立刻校对**）

```
Step 1 · 按标准速率算 8 段旁白 TTS 总和
   ↓
Step 2 · 跟用户给 TTS 比对（差值 + 容忍判定）
   ↓
Step 3 · 差 ≥ 5s = 重算（多种速率对比 + 选最接近用户给 TTS 的）
   ↓
视频总时长 = 用户给 TTS + 5s 冗余（不缩短）
```

### Step 1 · 按标准速率算 8 段旁白 TTS 总和

**标准速率公式**（用户原话"自然阅读"）：
- **英文 1.0 词/秒**
- **中文 3.0 字/秒**
- **家族词组 / 多词集合 = 0.5s 词间停顿**（必加）

**每段 TTS = 词数/1.0 + 字数/3.0 + (0.5 × 词间停顿数)**

### Step 2 · 跟用户给 TTS 比对

| 差值 | 判定 | 动作 |
|---|---|---|
| **\|差\| ≤ 3s** | ✅ 计算方式**高度正确** | 直接用 · 视频总时长 = 用户给 TTS + 5s 冗余 |
| **3s < \|差\| ≤ 5s** | ✅ 在 3-5s 容忍范围 · 计算方式**可接受** | 直接用 · 不必重算 |
| **5s < \|差\| < 10s** | ⚠️ **警告** · 速率可能偏快/偏慢 | 跑多种速率对比（Step 3）· 选最接近的 |
| **\|差\| ≥ 10s** | ❌ **严重错误** · 速率公式错或漏算 | 必重算 · 常见根因 = 漏算家族词组中文 / 漏算词间停顿 / 漏算中文字数 |

### Step 3 · 多种速率对比（**Step 2 判定 ⚠️ 或 ❌ 时必跑**）

```python
rate_schemes = [
    ("1.4/3.5（兜底公式，过快估）", 1.4, 3.5, 0.3),  # ❌ 已废弃 · 漏算中文
    ("1.2/3.0（中等估）",           1.2, 3.0, 0.4),
    ("1.0/3.0（自然估·标准）",     1.0, 3.0, 0.5),  # ✅ 推荐默认
    ("0.9/2.5（慢速自然估）",       0.9, 2.5, 0.5),
    ("0.8/2.5（最慢自然估）",       0.8, 2.5, 0.5),
]
```

**判定**：选**差 ≤ 3s** 的速率 = 重新算 8 段 TTS · 视频总时长 = 用户给 TTS + 5s 冗余。

## 实战案例（2026-06-13 绘本 8 段 · 用户给 TTS = 50s）

| 速率方案 | 8 段总 TTS | 差 vs 50s | 判定 |
|---|---|---|---|
| 1.4/3.5（兜底公式）| 35.51s | -14.49s | ❌ 差过大（漏算中文 + 漏算停顿）|
| 1.2/3.0 | 41.73s | -8.27s | ❌ 差过大 |
| **1.0/3.0（自然）** | **47.33s** | **-2.67s** | **✅ 在 3s 容忍内** |
| 0.9/2.5（慢速自然）| 53.53s | +3.53s | ✅ 在 3-5s 容忍内 |
| 0.8/2.5（最慢自然）| 57.70s | +7.70s | ❌ 差过大 |

**判定** = 用 **1.0 词/秒 + 3 字/秒 + 0.5s 词间停顿**（标准速率 · 差 2.67s 在 3s 内）= 视频总时长 = 50s + 5s 冗余 = **55s**。

## 5 类常见根因（差 ≥ 5s 时的 5 个排查方向）

| 根因 | 症状 | 修复 |
|---|---|---|
| ❌ 漏算家族词组中文 | 段 7 算 4.29s · 实际 13.67s（差 3 倍）| 把"中文说明"当 TTS 真要读的 · 必算字数 |
| ❌ 漏算词间停顿 | 家族词组 4 词 = 6 个 0.5s 停顿 = 3s 漏算 | 4 词/多词集合 = 必加 0.5s × (词数-1) 停顿 |
| ❌ 漏算中文字数 | 把"中文"当 0s · 实际 4-12 字 = 1.3-4s | 中文字数 = 全文所有中文字符（duck/UCK 等夹杂的英文部分单独算）|
| ❌ 用了 1.4 词/秒兜底 | 算出来偏快（差 10-15s）| 1.0 词/秒（自然估）= 用户给 TTS 优先 |
| ❌ 把 readme 的"非朗读中文"当 0s | readme 写"X 家族"但实际 TTS 包含 · 误判 | 必查用户是否真要 TTS 读这段 = 不读=0s · 读=必算 |

## 跟用户给 TTS 数值的关系

**用户给 TTS = 视频总时长基准**（必收 · 不可漏）：
- 视频总时长 = TTS + 5s 冗余（**不缩短** · 必加 · 算上视觉延展）
- 校验：算 8 段 TTS 总和 vs TTS 差 ≤ 5s = 校验通过
- 校验失败 = 必查 5 类根因 · **不**擅自调速率到差异变 OK（**不**是用户 TTS 错）
