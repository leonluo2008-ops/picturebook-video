# picturebook-video · 绘本动画制作工作流（v0.7.0 精简版）

> 将静态绘本图片转换为儿童动画视频的完整工作流。
> 输入：静态图（≥1张）+ 旁白；输出：可播放动画视频。
> **当前范式**：v15 系列（v15 + v15.1 · 2026-06-05 起绘本默认）。

> 📚 **遇到问题先看范式索引**：[VERSION_INDEX.md](VERSION_INDEX.md) —— 速查 v15/v15.1 核心/状态/适用场景。

## 🎯 v15 系列范式（绘本默认 · 2026-06-05 起）

### 4 大件（v15 默认）
1. **启动前 6 必问**（比例/时长/切分/运镜调性/运镜规则/范式风格）
2. **v14 4 段 prompt 骨架**（段 1 主体 / 段 2 分镜 / 段 3 风格 / 段 4 BGM）
3. **运镜调性**（活泼有动感 / 治愈舒缓 / 动态开合 / 静态展示）
4. **v7-Say 密集音效型**（5 拟声/8s + 拟声+动作一对一 + 双重禁 BGM）

### v15.1 增补（长旁白单图多 Clip 拆分）
1. **触发公式**：`朗读 + 单词数 × 2.5s 留白 > 15s` → 必须拆
2. **语义块拆分**（3+3+1 不 7.5+7.5）
3. **节奏公式**：动作-动作-词-词-词（"2-1-3-3-3" 常见节奏示例 = 短-短-长-长-长，**参考不是必跑**，按铁律 #39 元原则灵活调整）
4. **末帧铁律**：用户原话"把最后一个单词读完过后，仍然要像前面的每一个单词读完那样，留足够的呼吸、消化的时间"

### v15.2 候选（单图多 Clip 衔接 · 未实测）
- `--last-frame` 锁末帧 + 拟声渐弱 + 情绪承接
- ⏳ 待 6 段拼接验证

### 3 必问 + 3 默认

| # | 必问项 | 默认值 |
|---|---|---|
| 1 | 视频比例？| **16:9**（不查原图比例，模型自动适配）|
| 2 | 单 Clip 时长？| **4s 硬下限** · **4-15s** 范围 |
| 3 | 切分方案？| **旁白优先于图片**（铁律 #33）|

**v15 启用的 3 必问**（运镜调性/运镜规则/范式风格 = 启动 v15 时**默认已知**，**不重复问**）：
- 运镜调性 = **活泼有动感**（绘本默认）
- 运镜规则 = **按画面类型定制**（封面/动作/多人/情感/动态 6 种）
- 范式风格 = **v15 + v7-Say 音效密集型**

### 5 必避

1. ❌ 擅自合并 Clip（领读型默认 = 4-Clip 合并 `8+8+9+10=35s`，**不擅自改切分**）
2. ❌ 套硬编码默认值（4:3 / 8s）—— 必问用户
3. ❌ 私自延伸单 Clip 改动到全局（用户说 "Clip 1 = 4s" ≠ "全部 4s"）
4. ❌ 写"中景→中近景缓推 3 秒"套运镜公式（**秒级控制 / 时间感描述允许**——v10/v15 标版核心）|
5. ❌ 默认重提交 task（先查 ark list 端点 / 问用户）—— 20 元/任务

### 4 必跑 · 启动前自检（铁律 #32-#35）

1. **算旁白朗读时长**（铁律 #32）：中文 0.25s/字 · 英文 0.7s/词 · **拖长韵 1.0s/词**
2. **跑切分决策树**（铁律 #33）：先算旁白时长 → 算 Clip 容量 → 决定切分
3. **不擅自设时长上限**（铁律 #34）：seedance 4-15s 都支持
4. **跑前必 `create --help`**（铁律 #35）：子命令结构 create/status/wait

### 决策权判断口诀

- **不擅自延伸用户指令**（铁律 #29）：用户说 A 只做 A；不凭直觉猜，不直接说"我猜"
- **可逆操作不打扰**（铁律 #30）：默认重提交 task 这种事**先查 ark list 端点**，不浪费 20 元
- **任务叫停 = 废弃结果**（铁律 #31）：Seedance API 不支持中途 cancel

### ⚠️ 反"反问废话"原则（用户最痛点 · 2026-06-05 Cat 实战沉淀）

**根因**：用户多次情绪化纠错（"你TM哪来这么多问题？"、"我被你搞得有点迷糊了"）—— **agent 习惯性把"明显可定的事"包装成 4 选 1 / 2 选 1 反问用户**。

**铁律（强化版）**：

| 维度 | 错 | 对 |
|---|---|---|
| **明显默认的事** | "你想 A 还是 B？" | **直接做 A + 报你**（"我打算 X，因为 Y"）|
| **可逆操作** | "你确认要不要 X？" | **直接做 + 报你**（你不同意 → 你说停/换）|
| **真开放的事** | 4 选 1 / 2 选 1 凑数 | **不列选项**（直接说"你想怎么搞？"）|
| **必须问的事** | — | 不可逆操作 / 真没方向 / 关键判断 |

**判断口诀**：

```
问之前先自问 3 遍：
1. 我能直接做吗？（能 → 做）
2. 问错了的代价是什么？（不可逆才问）
3. 是否有 skill 能查？（有 → 查 skill，不问用户）
```

**反面案例**（本对话发生）：

| ❌ 我做的 | ✅ 应该做的 |
|---|---|
| "12s 是 seedance 最长单 Clip 吗？要不要查一下？" | 查 skill：`4-15s 都支持` → 直接跑 |
| "跑前要不要先拼 prompt 给你看？" | 拼好 + 报你，等你说"跑" |
| "A 还是 B 还是 C？"（凑数）| 报你"我选 A 因为 B/C 的理由" → 你不同意再说 |

**正面案例**（本对话已修复）：

- 用户问"合并 clip 数量" → 我**直接给 B 方案 + 报你**（不是反问"你选 B 还是 C"）
- 精简 SKILL.md → 我**直接动手 + 报你精简结果**（不是反问"你想精简哪部分"）

### 切分表自检清单（铁律 #38 · 跑前必勾）

- [ ] **1. 切分表列全了吗**？1图=1Clip + 拆分子 Clip（4a/4b）全部列出？
- [ ] **2. 收势图含了吗**？最后 1 张图（封底/收势）必须有 Clip？
- [ ] **3. 拆分子 Clip 列了吗**？单图分多 Clip 必须明列（4a/4b 不可只列 4）？

#### 4.0 漏图连环错根因（2026-06-05 Cat 实战 · 必须看）

**连环错现场**（Cat 完整跑通后用户指出）：

| 错 | 现象 | 根因 |
|---|---|---|
| 漏图 2 | clip1 写"图 1"没写"图 1+2" | 切分表只列主图，没标"双图合并" |
| 漏图 8 | B 方案切分表只到 4b，没列 clip5 | **写切分表时偷工，没按"1图=1Clip + 拆分"完整列全** |
| 缺 4b | 实跑只跑 4a（cat+hat+mat）| 切分表写"clip4=图7" 跟 跑"4a"不一致 |
| 老鼠一闪 | clip3 10s 装不下 7.65s 朗读+鼠戏 | 决策树算时长**漏算留白**（每词 2-3s 静默）|

**根因 1：写切分表时"偷工"**（图 1=单图 vs 图 1+2=双图，没严格对齐）

**根因 2：跑前 vs 跑后切分表没对齐**（写 5 个 Clip 但只跑 4 个，**没人发现**）

**根因 3：决策树算时长漏算留白**（7.65s 朗读 + 留白 > 10s 应该拆）

**修复（铁律 #38 自检清单）**：

| 修复 | 操作 |
|---|---|
| 根因 1 | 切分表**逐图核对**（1, 2, 3, 4, 5, 6, 7, 8 = 全列）|
| 根因 2 | 跑前 vs 跑后**严格对齐**（写 5 = 跑 5）|
| 根因 3 | 决策树**加上留白**（每词 2-3s 静默 = 朗读 × 2-3 倍容量）|

| **v13 范式：主体外观@ImageN + 场景参考@ImageM 分离** | 即梦支持，可用但有限制（铁律 #40）| **只用于"主体外观跟场景图冲突"场景** |

### 铁律 #40 · 主体外观 vs 场景分离的可控范围

**触发场景**：某镜头需要"@ImageM 的场景 + @ImageN 的主体外观"（如 Pic Clip 2 镜头四：要 @Image4 花丛场景 + @Image3 立耳兔外观）。

**写法**：
```
主体小兔子@Image3（= 主角外观，耳朵竖立），
场景参考@Image4（= 花丛+闻花构图）
```

**实测失败根因（Pic Clip 2 v2）**：seedance 2.0 `--ref-images 3.jpg 4.jpg` **同时传**时，模型**默认按参考图视觉权重**轮询——**@Image4 末段的视觉特征（垂耳+闻花构图）会压过 prompt "立耳" 文本约束**。**7-11s 镜头四全部跑出垂耳兔**。

**3 个候选修法**：

| 方案 | 做法 | 风险 | 推荐 |
|---|---|---|---|
| **A · `--image` 首帧锁** | 不用 `--ref-images`，`--image 3.jpg`（立耳首帧）+ prompt 写"6-11s 切到花丛闻花" | ✅ 角色外观锁死；❌ 失去多图参考 | ⭐ 首选 |
| **B · prompt 加权** | 保留 `--ref-images 3.jpg 4.jpg`，prompt 加"耳朵必须竖立" 加粗 | ✅ 改动小；❌ 不一定生效（v15 测过）| 备选 |
| **C · 撤回 V13 范式** | 镜头四用 @Image4，**接受原图外观** | ✅ 简单；❌ 违反"用图三的兔子"明确指令 | ❌ 不推荐 |

**修复判定**：默认走 A；A 跑出来不满意再退 B；用户没明确说"必须分离"就回 C。

### 铁律 #41 · 主体一致性不在 prompt 里二次声明

**反模式**：v15 段 1 主体定义写"奶白色小兔子@Image1（...），**同一个**奶白色小兔子@Image2（...）"——**自作主张跨图一致性声明**。

**根因**：`--ref-images` **本身**已处理多图角色一致性，prompt 二次声明是**冗余 + 干扰**——模型在多参考图场景下，**跨图一致性由视觉权重决定**，**不是文本约束**。

**正确写法**：段 1 主体定义**对每张图各说一次**，**不写"同一个"**：
```
奶白色小兔子@Image1（耳朵竖立 + 张开双臂草坪），
奶白色小兔子@Image2（背对观众 + 窗前看太阳）
```
**不写**："同一个奶白色小兔子@Image2" / "两只都是同一只兔子"。

**用户原话**（2026-06-06）："镜头一、镜头二，里面所使用的参考图 image 一、image 二已经处理好角色一致性了，不要随意去错开子代。"

### 4 段骨架

| 段 | 内容 |
|---|---|
| 段 1 | 主体定义（角色 + 场景 + 卡片信息）|
| 段 2 | **分镜绑定 + N 个镜头 + 拟声 + 收势**（**最关键**）|
| 段 3 | 风格（2D paper collage / 插画 / 3D 等）|
| 段 4 | BGM 段（v15 双重禁 BGM：段尾明写 + 段内不写"持续铺底"）|

### 段 2 核心约束

| 约束 | 说明 |
|---|---|
| `@ImageN` 必含 | 唯一参考帧（**官方语法 = @Image1/@Image2/...，`@图N` 是中文别名模型不识别，2026-06-06 Pic Clip 1 v1 翻车实测**）|
| **角色指代对齐子代** | 每个镜头里 `主角@ImageN` 的 N **必须等于该镜头对应的图**。同图连续多镜头 → 全部写 `主角@Image1`；切到 @Image2 → 写 `主角@Image2`。**主语不乱用**（Pic 绘本 v1 翻车根因）。 |
| 镜头编号 | 镜头一/二/三...（≤ 4 镜头，**禁起承转合模板**）|
| 拟声+动作一对一 | 每个镜头配 1 个具体拟声（沙/咚/叮/噗/吱/沙沙，不是"轻响"）|
| **元素增/减由剧情决定** | 段 1 主体定义 / 段 4 文字保留段**不写"其他元素不出现"这种隔离句**。绘本原图元素是否出现在生成画面里，**由故事剧情需要决定**（Pic 绘本 2026-06-06 用户原话撤回 v1 翻车的伪铁律）。 |
| **v3 文字保留措辞**（绘本默认）| `参考图原有的所有文字（<列举具体文字>）必须完整保留作为画面元素，模型不得删除或替换这些文字，让文字自然融入场景`。v2 措辞"自然融入场景"在 `--ref-images` 双图/单图首参模式不够强 → Pic Clip 2 v2 翻车（末帧彩色字被吃）。v3 加 **"必须完整保留"** + **"不得删除或替换"** + **列举具体文字** → Pic Clip 3/4/5 全部命中（详见 [references/绘本文字保留铁律-v1v2.md](references/绘本文字保留铁律-v1v2.md)）。 |
| v3 文字保留措辞（绘本默认）| 完整措辞见 [references/绘本文字保留铁律-v1v2.md](references/绘本文字保留铁律-v1v2.md) 升级表。Pic 绘本 2026-06-06 Clip 3/4/5 全部命中。 |
| ⚠️ **Pitfall · 双图模式"参考图视觉权重 > 文本约束"** | `--ref-images 2.jpg` 双图模式**默认按参考图视觉权重轮询**——prompt 写"立耳"被参考图"垂耳"压过。**Pic Clip 2 v2 翻车实测**：7-11s 全部跑出 @Image4 垂耳特征，prompt 写"耳朵竖立"全失效。**Pitfall 反模式**：写"耳朵必须竖立" / "彩色字必须保留" / "风格必须 2D" 加粗（**文本加粗不会压过视觉权重**）。**正确做法**：① **关键元素进段 1 主体定义**（不只写在镜头描述）；② **强约束措辞** v3 措辞（"必须完整保留...不得删除或替换...列举具体文字"——列举 + 否定 + 强制三件套）；③ **实在不行用 `--image` 单图首参**（视觉权重 100% 锁死，参考铁律 #40 方案 A）。 | 双图 / 多图 prompt 写法 |
| 收势词在最后一句画面描述 | 不追加任何动作 |
| 段尾明写 | "**无任何背景音乐、无旁白人声、无哼唱**" |
| **末帧（v15.1）** | 朗读 1s + 静默 2-3s（**不是 0.5s 收势**）|

### v15 范式 prompt 模板

**v15 基础版**（3 词以下 / 单段用）：

```
主体定义：<角色 + 场景>
分镜绑定：@ImageN 作为唯一参考帧；
镜头一（0-2s · 场景建立）：<拉远>，<建立性拟声>；
镜头二（2-3s · 角色跃入）：<跃入>，<落地拟声>；
镜头三（3-6s · 朗读+静默）：镜头定格在 <单词 1>，<拟声>一响，<动作>（朗读 1s + 静默 2s 停留）；
镜头四（6-9s · 朗读+静默）：镜头水平切到 <单词 2>，<拟声>一响，<动作>（朗读 1s + 静默 2s 停留）；
镜头五（9-12s · 末帧=朗读+静默）：镜头水平切到 <单词 3>，<拟声>一响，<动作>（朗读 1s + 静默 2s 停留，**画面继续微动**（微动 = 角色微微转头/眨眼/风吹树叶/水波轻动，**不是定格海报让画面死去**），短消化时间让观众感受呼吸/消化）；
参考图原有的所有文字（<列举具体文字>）必须完整保留作为画面元素，模型不得删除或替换这些文字，让文字自然融入场景；
无任何背景音乐、无旁白人声、无哼唱。
风格：2D paper collage style, 儿童绘本纸艺拼贴风，柔和哑光色，浅米色纸张背景，柔光无强烈阴影，温馨可爱调性。
```

**v15 范式节奏**（基础版）

### 📊 节奏决策指南（铁律 #39 元原则落地 · 优先级：元原则 > 数字节奏）

> **"标版"= 常见情况默认参考**，不是必跑。元原则 #39 主导——按旁白长度 + 故事调性，**agent 自己在脑内选**。Pic 5 Clip 跑通沉淀：节奏 2-1-3-5（11s 装 2 词）跨 4 个 Clip 稳定可复用。

**旁白长度 → 节奏选择**：

| 旁白总朗读 | Clip 数量 | 推荐节奏 | 总时长 | 实测 |
|---|---|---|---|---|
| < 4s | 单 Clip | 2-1-3 (6s) | 6s | 单句场景 |
| **5-8s** | **单 Clip** | **2-1-3-5 (11s)** | **11s** | **Pic Clip 2/3/4/5 全部命中** |
| 8-12s | 单 Clip | 2-1-3-3-3-3-5 (15s) 接近 15s 上限 | 15s | ⚠️ 接近上限，谨慎 |
| 8-12s | 双 Clip | 2-1-3-3 + 2-1-3-3 (12s+12s) | 24s | Cat 4a + 4b |
| > 12s | 拆 N Clip | 跑 v15.1 语义块拆分 | 按块 | Cat 6 Clip |

**末帧静默 → 故事调性**（铁律 #39 落地）：

| 故事调性 | 末帧静默 | 示例 |
|---|---|---|
| **退场 / 收尾 / 角色远去** | **4-5s 画面微动+短消化**（**不是定格海报**）| **Pic Clip 1-5 全部命中**（绘本收势）——但末帧 4-5s **是画面继续动**（角色微微转头/眨眼/风吹树叶/水波轻动），**不是定格 4-5s 让画面死去**（用户原话纠错）|
| 标准 / 平稳 | 2-3s（默认）| Cat 单词朗读 |
| 紧张 / 情绪骤变 | 1-2s（急收）| ⚠️ 未实测，可标"待补" |

**反模式（标版当必跑）**：机械套 2-1-3-3-3 = 节奏过密 + 末帧过短 + 故事感被切断（Pic v1 翻车前倾向）。**正模式（元原则主导）**：算旁白 → 算 Clip 容量 → 选节奏 → **按故事调性定末帧**。

---

**v15.1 标版**（长旁白 / 6 词韵家族用）= 节奏公式 **2-1-3-3-3**：

- 镜头一 = 2s（场景建立）
- 镜头二 = 1s（角色跃入）
- 镜头三-五 = 各 3s（每个单词 = 朗读 1s + 静默 2s）
- **末帧 = 镜头五** = 同其他单词镜头（**不是 0.5s 收势**）
- **总 12s 装 3 词**，节奏"短-短-长-长-长"

**两者区别**：

| 维度 | v15 基础版 | v15.1 标版 |
|---|---|---|
| 适用 | 3 词以下 / 单段 | 6 词韵家族 / 长旁白 |
| 节奏 | 2-1-3-3-3 | **强制** 2-1-3-3-3 |
| 末帧 | 末帧镜头内静默 2s | **同**末帧镜头 = 其他单词镜头等长 |
| 信息密度 | 30-50% | **强制 25%** |

### Phase 5 · 分镜设计（v15 重点）

**Step 1 · 叙事单元设计**：
1. **算每段旁白朗读时长**（铁律 #32 公式）
2. **算单 Clip 容量**（视频时长 - 0.5s 收势留白）
3. **跑切分决策树**：
   - 每段 ≤ 4s？→ 1图=1Clip
   - 合并后 ≤ 7.5s？→ 2图=1Clip
   - 合并后 ≤ 11.5s？→ 3图=1Clip
   - 仍超？→ 拆 2 个 Clip + 跑 v15.1 语义块拆分

**Step 2 · 分镜计时**：
- 单 Clip 时长 = `max(旁白时长 + 留白, 4s 硬下限)`
- 末帧时长 = **朗读 + 静默 2-3s**（铁律 #36 / #39）

**Step 3 · 分镜组合**：
- v15 标版节奏：动作-动作-词-词-词（"2-1-3-3-3"）

**Step 4 · 连贯性校验**：
- 角色一致性：眼睛圆大/体型一致/神态活泼
- 场景一致性：背景色/纸艺风格统一
- 文字一致性：参考图所有文字 = 画面元素（铁律 14）

### Phase 8 · 分镜生视频（v15 + seedance.py）

**Pic 绘本 Clip 1 v1 翻车根因**：prompt 写 `@图1` / `@图2`（中文"图"）→ 模型不识别 → 分镜绑定失效 → 画面重复 2 轮。

**用户原话沉淀**（2026-06-06）："必须使用图一图二的时候，必须使用@。具体如何使用请你查看官方的文档。"

**官方语法 = `@Image1` / `@Image2` / `@Video1` / `@Audio1`**（director-mode.md §@-Tags 精确语法）。
