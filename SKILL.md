# picturebook-video · 绘本动画制作工作流

> 将静态绘本图片转换为儿童动画视频的完整工作流。
> 输入：静态图（≥1张）+ 旁白；输出：可播放动画视频。
> 严格遵循即梦官方 SOP 架构（Phase 0-9），针对绘本动画场景进行核心适配。

---

## 定位说明

本 skill 针对**已有静态绘本图片 + 旁白**的半成品素材，跳过 Phase 0-4（创意发散 / 内容大纲 / 剧本创作 / 素材挖掘），直接进入：

```
Phase 5: 分镜设计（四步走）
Phase 6: 需求确认
Phase 7: 参考素材生成（注：绘本场景直接复用静态图，跳过本 phase 的主体生成步骤）
Phase 8: 分镜生视频
Phase 9: 视频剪辑/成片
```

---

## ⚠️ 阅读前必须理解的核心差异

| | 官方短剧 SOP | 绘本动画（本 skill） |
|--|--|--|
| **输入** | 剧本（连续动作） | 静态图 + 旁白（独立叙事单元） |
| **图间关系** | 物理连续动作，可设计「接力」 | 每张图独立，无物理连续动作 |
| **clip 衔接** | 靠动作接力 + 剪辑转场 | 靠每个 clip 内部的「叙事收势」设计 |
| **分镜切分** | 从剧本拆分镜头 | 从「静态图 + 旁白」设计叙事单元 |
| **结尾处理** | 动作进行中，下一个 clip 接上 | 必须设计完整的「起承转合」，clip 有头有尾 |

**官方 SOP 的「衔接」机制（scene-reflection 检查动作衔接 + Phase 9 剪辑转场）在绘本动画场景下不适用。**
**解决方案在 Phase 5 Step 1（叙事单元设计）阶段，不在剪辑阶段。**

## 完整工作流程

```
Step 1  图片分析（Vision）
  → 分析每张图：场景/人物/构图/色彩/氛围/关键元素
  → 输出分析汇总表

Step 2  旁白与图片匹配
  → 将旁白匹配到对应图片，形成 1→1 对应关系
  → 输出匹配表

Step 3  分镜设计（四步走）
  ↳ Step 3.1  叙事单元设计（绘本版分镜切分）
  ↳ Step 3.2  分镜计时
  ↳ Step 3.3  分镜组合
  ↳ Step 3.4  连贯性校验

Step 4  需求确认
  → 输出完整 Clip 表（含时长），用户确认后进入下一阶段

Step 5  分镜生视频（并行）
  → 为每个 Clip 编写完整 Prompt（含叙事收势设计）
  → 所有 Clip 并行提交生成

Step 6  成片剪辑
  → ffmpeg 拼接 + BGM 合并
  → 输出最终视频

Step 7  交付与反馈
  → 发送实际文件给用户
```

### Step 3.1 叙事单元设计（script-chunk · 绘本版）

> 官方 SOP 中，分镜切分（script-chunk）是将剧本拆分为单个镜头。
> **绘本动画场景的适配**：输入是「静态图 + 旁白」，不是剧本。
> 本步骤将「静态图 + 旁白」转化为「带叙事弧的分镜」，核心是设计每个 Clip 的**起承转合**和**收势**。

#### 官方 script-chunk 的核心要素

| 字段 | 内容 |
|------|------|
| 镜号 | 从1开始连续编号 |
| 景别 | 远景/全景/中景/中近景/近景/特写 |
| 画面描述 | 当前镜头中看到的内容，**必须完全基于图片** |
| 台词/音效 | 对应本镜头的旁白或环境音效 |

#### 绘本版的核心增强：叙事单元设计

每个叙事单元（对应一张图片 + 一段旁白）必须设计完整的**四段式叙事弧**：

| 阶段 | 说明 | 设计要求 |
|------|------|----------|
| **起（开场）** | 这个 Clip 从什么状态开始 | 基于静态图画面，描述起始状态 |
| **承（发展）** | 中间发生了什么动作/变化 | 设计1-2个动作变化，不要只是「静止展示」 |
| **转（转折）** | 叙事的高潮点或关键变化 | 旁白核心信息在这里呈现 |
| **合（收势）** | Clip 结尾的状态 | **必须设计「收势动作」——动作完成/达到目的/稳定状态**，避免「动作进行中被截断」的观感 |

#### 「收势」设计的三种模式

根据旁白的叙事功能，选择合适的收势模式：

| 模式 | 适用场景 | 收势设计 |
|------|----------|----------|
| **完成式收势** | 旁白描述的是一个完整动作 | 以「动作完成」收尾：举起火把→火焰燃起稳定；制作火把→完成品举起展示 |
| **进行式收势** | 旁白描述的是持续状态 | Clip 结尾保持运动状态，但给出一个「小目标达成」：队列行进→抵达目的地，举起火把 |
| **悬念式收势** | 需要连接下一个叙事单元 | 结尾留一个未完成的动作/问题，但不做大的情绪起伏，保持平稳期待感 |

#### 输出格式

```markdown
## 火把节分镜清单

### Clip 4 · 火把制作

| 镜号 | 景别 | 画面描述 | 旁白 | 叙事阶段 |
|------|------|----------|------|----------|
| 1 | 中景 | 阳光明媚的乡村庭院，一男一女坐在木凳上整理松木和松脂，男子用柴刀削木棍，女子整理火把材料。 | 人们用松木和松脂制作火把，保证火焰燃烧得又稳又亮。 | 起：开始制作 |

**叙事单元设计说明：**
- 起（00-02s）：男子用柴刀削木棍，女子整理火把材料
- 承（02-04s）：两人配合制作，男子将削好的木棍递给女子，女子用松脂绑缚
- 转（04-06s）：火把制作完成，男子举起火把
- 合（06-07s）：男子用火绒点燃火把，火焰稳定燃烧，火光照亮两人脸庞 → **收势在「火焰稳定燃烧」，不是「制作中」被截断**
```

#### 执行检查清单（Step 3.1）

- ✅ 每个叙事单元都有「起承转合」四段设计
- ✅ 每个叙事单元都有明确的「收势」设计，避免「动作进行中被截断」
- ✅ 收势模式选择合理（完成式/进行式/悬念式）
- ✅ 画面描述完全基于静态图，没有用旁白臆造画面
- ✅ 旁白的核心信息在「转」阶段呈现

### Step 3.2 分镜计时（shots-timing）

严格遵循官方规则：

| 类型 | 计算方式 |
|------|----------|
| 台词镜头 | 旁白字数 × 0.3 秒/字 |
| 最低保底 | 每镜 ≥1 秒 |
| 情绪节奏 | 快节奏1-3秒，情感戏3-8秒，空镜2-4秒 |
| 景别影响 | 远景/全景偏长（3-6秒），近景/特写偏短（1-4秒） |
| 单Clip硬约束 | 4s ≤ 时长 ≤ 15s |

> ⚠️ `--duration` 参数必须是**整数**，向上取整（`ceil`）。

### 输出格式

```markdown
## 火把节分镜计时表

| 镜号 | 景别 | 画面描述 | 旁白 | 时长(秒) |
|------|------|----------|------|----------|
| 1 | 中景 | 阳光明媚的乡村庭院... | 人们用松木... | 2.0 |
| ... | ... | ... | ... | ... |

**总时长：7.0 秒**
```

---

### Step 3.3 分镜组合（shots-assembly）

严格遵循官方规则：

**硬约束**：每个 Clip **4s ≤ 时长 ≤ 15s**，绝对不能超。

| 情况 | 处理方式 |
|------|----------|
| < 4秒 | 合并到相邻 Clip |
| > 15秒 | 拆分为多个 Clip |
| 场景完整性 | 同一场景尽量不拆分 |
| 情节连贯性 | 同一段连续对话/动作不拆分 |

> ⚠️ 绘本动画场景：每张静态图默认对应一个 Clip（1图=1旁白=1叙事单元=1Clip），只有在该 Clip 时长超出 15s 时才需要组合。

### 输出格式

```markdown
## 火把节Clip表

| Clip编号 | 对应图号 | 总时长(秒) | 画面描述 |
|----------|----------|------------|----------|
| 1 | 1.jpg | 7.5 | 白天梯田村庄，多人民族服饰 |
| 2 | 2.jpg | 7.0 | 夜晚火炬光带，村庄星空 |
| ... | ... | ... | ... |

**总Clip数：10个**
**总时长：67.0 秒**
```

### Step 3.4 连贯性校验（scene-reflection）

> **强制 gate，必须完成才能进入 Phase 6 需求确认，不能跳过。**

从以下五个维度逐一检查：

| 维度 | 检查内容 |
|------|----------|
| **人物一致性** | 服装/外观/位置/动作衔接；上一Clip结束动作和下一Clip开始动作要连贯 |
| **道具一致性** | 存在性/位置/外观 |
| **场景一致性** | 光线/天气/陈设 |
| **逻辑连贯性** | 情节/时间/物理规则 |
| **色调统一性** | 整体色调风格统一 |

#### 绘本场景的特殊检查项

| 检查项 | 说明 |
|--------|------|
| **叙事收势是否完整** | 每个 Clip 是否都有「收势动作」，而不是「动作进行中被截断」 |
| **收势模式是否一致** | 收势模式（完成式/进行式/悬念式）选择是否与旁白叙事功能匹配 |
| **相邻Clip动势衔接** | 上一Clip结尾的动作状态，是否与下一Clip开头的画面匹配（即使没有物理连续动作，也要保持叙事逻辑连贯） |

#### 修复规则

当发现不一致/不连贯问题时，按以下优先级修复：

1. **最小修改原则**：只修改有问题的部分，不改变整体分镜和时长
2. **添加说明原则**：在问题分镜的描述中添加「需要保持与前一ClipXX一致」的明确说明
3. **拆分调整原则**：如果时长拆分导致不连贯，适当调整Clip拆分边界
4. **标注提醒原则**：在修复后的Clip表末尾，汇总所有需要保持一致性的提醒点

#### 输出格式

```markdown
## 火把节Clip表（连贯性校验修复版）

## 校验结果
- 发现问题1：[问题描述]
  - 修复方式：[具体修复方式]
- 未发现其他一致性问题

---

[修复后的完整Clip表]

---

## 一致性注意事项
1. [人物XX]在所有Clip中保持[服装/外观]一致
2. [场景XX]在所有Clip中保持[光影/色调]一致
3. [道具XX]位置保持不变
```

#### 执行检查清单（Step 3.4）

- ✅ 五个维度都做了检查（人物/道具/场景/逻辑/色调）
- ✅ 叙事收势完整性已检查（每个Clip都有收势动作）
- ✅ 所有发现的不一致问题都已标注并修复
- ✅ 修复没有改变原分镜的总时长和情节顺序
- ✅ 最后汇总了所有一致性注意事项

## Step 5 · 分镜生视频（video-prompt）

### Prompt 编写规范（5要素）

严格遵循官方 video-prompt 的5要素格式，**且第3项「分镜动作描述」必须包含叙事收势设计**：

| # | 要素 | 内容 |
|---|------|------|
| 1 | **参考主体引用** | 首帧图片用 `@ImageN`，参考主体用 `<subject>主体名称</subject>` |
| 2 | **场景描述** | 当前 Clip 所在场景，明确要求保持与参考图一致 |
| 3 | **分镜动作描述** | 按时间线描述每个镜头的画面内容和人物动作，**必须包含「收势动作」描述** |
| 4 | **台词/音效说明** | 本片段对应的旁白或音效提示 |
| 5 | **风格一致性说明** | 整体风格与参考素材保持一致，画面连贯自然 |

### 第3项「分镜动作描述」的编写要求

> ⚠️ **这是绘本动画场景的核心差异化要求。**

必须包含：
1. **起**：Clip 开头的起始状态（基于静态图画面）
2. **承**：中间的动作发展（设计1-2个动作变化）
3. **转**：旁白核心信息的视觉化呈现
4. **合**：Clip 结尾的「收势动作」（动作完成/达到目的/稳定状态）

**错误写法（无收势）**：
```
@[Image]，场景保持图片一致。
人物在庭院中制作火把，男子用柴刀削木棍，女子整理材料。
旁白：人们用松木和松脂制作火把...
整体风格保持温暖插画风格。
```

**正确写法（有收势）**：
```
@[Image]，场景保持图片一致。
[00-02s] 中景，阳光明媚的乡村庭院，男子坐在木凳上用柴刀削松木棍，女子整理火把材料。
[02-04s] 男子将削好的木棍递给女子，女子用松脂绑缚加固，火把雏形完成。
[04-06s] 男子举起完成的火把，用火绒引燃，火焰开始燃烧。
[06-07s] 火焰稳定燃烧，火光照亮两人脸庞，两人对视露出笑容，镜头缓慢拉远。
旁白：人们用松木和松脂制作火把，保证火焰燃烧得又稳又亮。
整体风格保持温暖插画风格，色彩饱和，叙事完整有收尾感。
```

## Step 6 · 成片剪辑

### 6.1 视频拼接

```bash
ls -v ./output/*.mp4 > clips.txt
cat clips.txt | while read f; do echo "file '$f'"; done > concat.txt
ffmpeg -f concat -safe 0 -i concat.txt -c copy concat_raw.mp4
```

### 6.2 BGM 合并

```bash
ffmpeg -i concat_raw.mp4 -i "bgm.mp3" -shortest -c:v copy -c:a aac output_final.mp4
```

## 关键教训（火把节实战）

1. **绘本图片是「独立叙事单元」**：每张图是独立画面，图间没有物理连续动作，视频生成时每个 clip 从头渲染——这是正常行为，**不是缺陷**
2. **「突然中断」问题的根因**：原 Prompt 只描述「画面是什么」，没有设计 Clip 结尾的「收势动作」，导致动作进行中就被截断
3. **解决方案在 Step 3.1（叙事单元设计）**：为每个 Clip 设计完整的「起承转合」，尤其要设计明确的「收势动作」，让 Clip 在「完成状态」结束
4. **scene-reflection 是强制 gate**：必须完成才能进入 Phase 6，跳过会导致收势设计缺失
5. **video-prompt 的第3项是核心**：必须包含「收势动作」描述，不是只写「画面里有什么」
6. **时长必须是整数**：向上取整，`--duration 7.5` 会报错
7. **并行生成不串行**：同一阶段多个 Clip 一次性提交
8. **飞书云盘文件名错乱**：官方导出文件名与实际内容对不上，下载后必须比对首行 `# 标题` 确认内容
