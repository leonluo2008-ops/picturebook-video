## 📚 长期积累机制（蒸馏法 · 2026-06-14 沉淀 · 必走）

### 三层架构（业务/work/通用方法论 分层）

| 层 | 位置 | 内容 | 命名 | 寿命 |
|---|---|---|---|---|

```

```

## 📋 沉淀前自检清单（**2026-06-14 绘本中段 clip 反复翻车后沉淀 · 必走 · 不靠记忆**）

> **核心问题**：agent 写完 SKILL.md / references / prompt 后，**必跑自检**才认为"沉淀完成"。写完 ≠ 沉淀完。

### 5 项必跑自检（**沉淀前不跑 = 翻车**）

| # | 自检项 | 命令 / 判定 | 期望 |
|---|---|---|---|
| 1 | **0 绘本名污染** | `grep -nE "(duck clip\|duck 绘本\|duck v6\|clip2 v6\|绘本名+版本号)" SKILL.md references/*.md` | **0 处**（duck 泛指/TTS 旁白文本除外）|
| 2 | **0 vX.X+picN 标签** | `grep -nE "v[0-9]+\.[0-9]+\+pic[0-9]+" SKILL.md references/*.md` | **0 处**（反例占位符也用通用占位符）|
| 3 | **所有 references 引用 = 真实存在** | `grep -oE "references/[a-zA-Z0-9_-]+\.md" SKILL.md \| sort -u \| while read f; do test -f "$f" \|\| echo "❌ $f"; done` | **0 missing**（见下方 §3.1 断链判定 3 分类） |
| 4 | **元方法论命名 = 通用方法论名**（不绑具体绘本）| 反问自检："这个方法论是因为某本绘本踩坑，还是因为通用方法论？" | **通用方法论** → 写 skill 仓 |
| 5 | **patch 工具前 = 必先全量 read**（防 sibling 并发覆盖）| `read_file` 全文件**不用 offset/limit** → 再 `patch` | 避免"partial view"警告 + sibling 改动丢失 |

**判定口诀**：

- 5/5 全过 = 沉淀完成
- 任意 1 项失败 = **先修自检项** → 再交付
- 漏跑 = 翻车（绘本名污染 / 引用空文件 / 方法论绑专名 / 跨 agent 覆盖）= 下个 session 复用 skill 时必翻车

**§3.1 断链引用判定 3 分类 + 修复决策树**（2026-06-14 duck 维护实战沉淀 · 改 1 句不补空文件 = 治本）：

自检 §3 报 missing 时,先判定是真断链还是假断链,再选修复策略（**不**默认补空文件 = 治标）：

| 分类 | 触发 | 判定方法 | 修复策略 |
|---|---|---|---|
| **A · 假断链**（反例占位符）| 在"反模式"区 + 文件名是泛指占位符（如 `A-validation.md` / `XX-绘本.md`）| 看上下文 = 反例 / 教学 = 占位符 = 不是真引用 | **不改** = 保留作为反例教学 |
| **B · 真断链**（正文引用）| 在正文 / Step 章节 + 说"详细见 X" / "修复路径见 X"= 真指引用 | grep 该引用的内容是否在 SKILL.md 表格/章节内已含 | **删引用 + 改成内联指向上文**（信息 0 损失）|
| **C · 真断链**（目录索引）| 在 references 列表 = 目录索引 | 看文件名是否含具体绘本名（如 `mango-validation.md`）= 违反铁律 #116 业务数据不入 skill 仓 | **删索引条目** = 不建空文件 |

**修复优先级**：

```
报 missing
   ↓
1. 看上下文判定 A/B/C
   ↓
A → 保留（反例占位符）
B → 删引用 + 改成内联指向上文表格/章节
C → 删索引条目
   ↓
不补空文件（"该删就删"原则 · 违反 = skill 仓多个永远不补的废文件）
   ↓
重跑自检 §3 = 期望 0 missing
```

**反模式**：

- ❌ 报 missing → 立即建空文件标"待补"= 治标 = 违反"该删就删"
- ❌ 报 missing → 不看上下文直接删 = 误删反例占位符 = 教学价值丢失
- ❌ 报 missing → 在 references/ 加 `XX-validation.md` 通用占位文件 = 污染

**根因**（为什么之前不区分）：

- 自检脚本 `test -f` 只判文件存在,不看上下文语义
- 人工判定 3 分类 = 看引用位置的"反模式区 / 正文 / 目录索引"3 个固定区域
- 实战验证（2026-06-14 修 3 处 missing）：A = line 45（保留）+ B = line 841（删引用改内联）+ C = line 1291/1294（删索引）= 3 处不同分类 = 3 种不同修复

**配合铁律**：

- 铁律 #116 业务数据不入 skill 仓 = C 类判定依据（含具体绘本名 = 业务数据 = 删）
- 维护陷阱已整合到 `references/skill-distillation-governance.md`（元方法论 #M4）

- ❌ 写完 = 沉淀完 = **没验证就交付** = 翻车

**判定优先级**（**用户原话优先**·2026-06-14 绘本实战沉淀）：
- 用户原话："**视频时长规划 50 秒，本次 TTS 时长有冗余**" → 视频总时长 = 50s = 不加 5s 冗余
- 含义：用户给的 TTS 数值已经预留过冗余 = 不要再 +5s
- **默认**（用户没说）：视频总时长 = TTS + 5s 冗余（按之前绘本翻车的硬底线）
- **反模式**：用户说"TTS 有冗余"还强行 +5s = 实际比 TTS 长 = 多余空镜拖沓
- **Seal 海豹实战验证**（2026-06-15）：用户原话"**36s 已经有冗余,所以你可以完全按照 tts 时长来计算,不额外加静默时长**" → 视频总时长 = 36s。Agent 实战中曾套 5s 冗余算出 41s · 用户纠正后才到 38s = 实战教训：**用户说"TTS 已有冗余"时,任何额外冗余都是错的**。

**实战 5 步走**(用户说"TTS 已有冗余"时)：
1. 1.0/3.0 算段 TTS(可能差 10-20s)= 慢估,看上限
2. 1.4/4.0 算段 TTS(可能差 4-5s)= 较快估,看下限(**绘本短句领读型默认档 · 铁律 #25**)

视频模型能非常好地理解参考图。**prompt 只需要做好关键引导**（时间点、运镜、声音、末帧约束、风格词），**不需要描述每一个细节**。

- **必须约束**（关键引导）：① 镜头运动（推/拉/摇/切，每镜 1 种）② 末帧微动（不变成定格海报）③ 声音策略（无 BGM / 拟声）④ 文字保留 / 风格锁 ⑤ 大动作（鸭子游动 / 起飞 / 张嘴）
- **让模型自由组织**（不约束）：① 角色每个肢体的具体动作（翅膀朝后还是朝前、几度、几秒）② 表情的微变化 ③ 头发/羽毛/水花的物理细节 ④ 镜头内具体的秒数切分（官方原话："模型对精确时间支持不稳定"）

**反模式**：prompt 写 5+ 句动作描述塞 1 个镜头 = 5 个动作里必有 2 个跟参考图姿态冲突 = 模型二选一时抽 = 动作僵硬 / 抽搐 / 跟参考图姿态对不上

**修复**：每镜头 1-2 个核心动作（"鸭子继续向前游"）+ 让模型照参考图自己组织细节 = 自由发挥 = 自然连贯

**官方背书**（seedance doc2 §1.3 · line232-237）："**模型对精确时间（如 0–3 秒）的支持不稳定，强行限制时长可能导致生成结果异常**" + 实战案例 line320-389 官方案本**根本不写秒数** = 印证"信任参考图 + 让模型自由组织"是官方推荐写法

**规则推导方法论**（2026-06-14 沉淀 · 元方法论 #M2 · L0 顶层 · 与 #M1 并列）：

**总结一句话**：**每条规则 = 一个跨场景通用的"判断方法"，不是一个特定情境的"结论"**。

## 9. 官方原话纪律（2026-06-14 绘本 clip2 沉淀 · 3 次翻车反复修正）

> **本节是 #M2 规则推导方法论的第 2 个支柱：不光要从"决策逻辑"推导，引用官方原话时也必须**逐字**用，不许改写、不许"补全意思"、不许加"看似合理"的修饰词。**

### 9.1 反复翻车的同一个根因

| 翻车次序 | 我写的（错）| 官方原话（对）| 漏了什么 |
|---|---|---|---|
| 第一次 | "1 镜 1 动作" | "**1 种运镜方式**" + "**1 镜多动作 OK**" | 漏"运镜"4 字 + 漏"多动作 OK"原话 |
| 第二次 | "1 镜 1 运镜 + **1-2 核心动作（1 主 1 辅）**" | "**1 镜多动作 OK**" | 我"补全"了 1 主 1 辅 = 官方原文没说 |
| 第三次 | "1 镜 1 运镜 + **1 主 1 辅动作**" | "**1 镜多动作 OK**" | 还是没直接用原话，又绕到 1 主 1 辅 |

**根因** = 每次都"自己加工"官方原话 = 漏 1 字 / 加 1 句 = 错全意。

### 9.2 正确范式（3 步）

```
1. 找官方原话（grep 文档 / 用户原话记录 / 历史 commit）
2. **逐字**写到 skill（不改写、不"补全"、不修辞美化）
3. 在原话后面附"判断方法"扩展（决策逻辑）

例（#11 铁律 + #15 铁律原文）：
- 官方原话："1 镜多动作 OK"（逐字写）
- 扩展判断方法（决策逻辑）：
  "1 镜 = 1 种运镜（可固定/可推/可切/可拉，每镜 1 种）+ 多动作 OK（没说数量上限）"
```

### 9.3 反模式（必避）

- ❌ "1 镜 1 动作"（自创简化）= 漏字 = 错全意
- ❌ "1 镜 1 运镜 + 1 主 1 辅动作"（自创扩展）= 官方没说过 1 主 1 辅
- ❌ "时长 = 凑整到 8s"（自创经验）= 用户原话是"为什么 4 秒的事用 8 秒做" = 反着
- ❌ "拟声符号 = 显式说消失"（自创表层规则）= 漏决策逻辑

### 9.4 凡引用官方原话必跑 3 问

```
1. 我引用的这句话 = 原文还是改写？
   ├─ 原文 → 继续
   └─ 改写 → 必找回原文（grep 文档 / 查历史 commit / 问用户）

2. 我有没有"加字" / "减字" / 改顺序？
   ├─ 没有 → 继续
   └─ 有 → 必删我加的字 / 找回漏的字

3. 漏 1 字 = 会不会"错全意"？
   ├─ 会 → 必逐字核对（用 5 维属性：原文+位置+语气+上下文+级别）
   └─ 不会 → 但仍要回原文档 grep 验证
```

### 9.5 跟 #15 铁律的配合

- **#15 铁律** = 引用官方原话必带 4 维属性（原文+位置+语气+上下文）+ 漏 1 字 = 错全意
- **本节 #M2 §9** = 把 #15 铁律的"字面纪律"扩展到"规则表述纪律"= 写规则时也必须逐字用原话，不许改写

**两节合并 = 写规则 + 引用原话 = 都要"逐字" + "决策逻辑"双层**。

### 9.6 实战案例（2026-06-14 duck clip2 v6→v7 翻车）

| 时点 | 我写的 | 官方原话 | 错点 |
|---|---|---|---|
| v6 prompt | "缓缓推近 + 切中景 + 点头 + 张嘴嘎嘎叫"（4 动作堆 6s）| "1 镜多动作 OK" | **没引用原话** + **没限制运镜 1 种** |
| v7 prompt | "固定中景（不推不切不拉）· 鸭子张开嘴巴嘎嘎叫" | "1 种运镜方式" | 引用了"运镜"= ✅ 没引用"多动作 OK"= ⚠️ |
| skill 沉淀 | "1 镜 1 运镜 + 1 主 1 辅" | "1 镜多动作 OK" | **"1 主 1 辅"= 自创** = 官方没说 |
| 最终修 | "1 镜 1 运镜 + 多动作 OK（#15 铁律原文）" | "1 镜多动作 OK" | ✅ **逐字**引用 = 稳定 |

**3 步修 = 关键**：
- 第 1 步：找原文（grep "#15 铁律"）
- 第 2 步：逐字写（"1 镜多动作 OK"）
- 第 3 步：附判断方法（"1 镜 = 1 种运镜 + 多动作 OK"）

**写规则后 4 问**（必跑）：

3. 我能不能说出这条规则背后的"判断方法"？

**反模式**（必避）：
- ❌ 看到 A 场景的"嘎嘎叫彩条消失了" → 直接总结"所有拟声符号都消失"（表层规则 = 下次翻车）
- ❌ 看到一个翻车 = 直接写"X 必做 Y"反模式（表层规则 = 没找出根因）

**真实绘本踩出的反常识案例**（跨本验证前先在 work/ 笔记，**3 次跨本验证再升铁律**）：

- **速率方案可以反过来**（短句多 + 领读式场景）：1.0 词/秒 + 3 字/秒 算出来比用户给 TTS 长 10s，1.4 词/秒 + 3.5 字/秒 才匹配。**根因**：1.0/3.0 是"慢估兜底"，但绘本短单词 + 领读式真实读速偏快；脚本默认把 1.4 标"已废弃"是因为它"漏算停顿"，**而本绘本 gap_count=0 停顿不是问题**。**判定口诀**：gap_count=0 的短句绘本 = 1.4/3.5 反而对 · 跑 Step 3 必同时给 5 档速率对比 = 选最匹配用户给 TTS 的那档，**不**无脑选 1.0/3.0。
- **`end_frame_microaction` 字段名陷阱**（主 agent 直干场景）：JSON 里写 `action` → 填脚本第 96 行 `end_frame_microaction.get('specific_motion', '')` 取不到值 → 末帧微动段落变空 → verify 4 项查不到这个细节 = 静默翻车。**根因**：C 子 agent 输出字段名是 `specific_motion`，**不是** `action`。**修复**：JSON 里必用 `specific_motion`（C 子 agent 范式）；主 agent 直干时如果字段名拼错 = 改 JSON + 重 fill。
- **`time_breakdown` 默认 silence_seconds=2.0 过长**（短句多场景）：不显式设 `silence_seconds` → 脚本默认 2.0s → 镜头里被自动拼上"（朗读 1.28s + 静默 2.0s 停留）" → 7s 装不下 1.28+2+4.57+2+末帧静默 = 节奏拖垮。**根因**：脚本默认 2.0s 是 Cat 长单词领读的兜底。**修复**：绘本短句场景 = 每个 `time_breakdown` 项必显式设 `silence_seconds: 0.5`（≤ 1.0s 即可），**不**用默认 2.0s。
- **末尾 4 行约束 fill 脚本不自动加**：fill_v15_template.py 输出里**没有** "保持无字幕 / 不要生成水印 / 不要生成 Logo / 无人声、无歌唱、无配音、无朗读" 4 行 → verify_filled_prompts.py 4 项检查不查这 4 行（只查 @Image 语法 / 段 4 B 档 / char_floats 动态）= 静默漏约束。**修复**：fill 后**必**手工补这 4 行末尾约束（或者用一次性脚本 sed/append），**不**靠 fill 脚本。**根因**：fill 模板作者默认用户后期手动加约束 = 跟 4 词末尾约束铁律 #17 不自洽 = 该修脚本。

**适用场景**：所有 skill 沉淀过程 · 跨场景（绘本/漫剧/视频/账号/任何用 skill 沉淀方法论的领域）

**整本故事节奏优先原则**（2026-06-14 绘本 clip 翻车沉淀 · 元方法论 #M1 · 通用创作规范 · 跨本验证 1 次 · **2026-06-16 Run 跑强化合并 Clip 判定**）：

> **设计视角 = 站在整本故事的维度设计每个 clip，不是站在单个 clip 的维度。**

这是**比"信任参考图"更高一层**的设计视角。适用于所有"由多个片段组成完整作品"的创作场景（绘本视频 / 漫剧分镜 / 短视频合集 / 长视频章节 / 广告系列片），**不只是绘本**。

**5 条核心规范**（每条都是通用创作规范，不只是绘本）：

1. **整本节奏分配给 N 个 clip**（不是每个 clip 各做各的起承转合）
   - 整本作品的"起承转合"分配在 N 个 clip 上
   - 开篇 clip = 唯一可以做"起"的铺垫
   - 中段 clip = 直接切入故事情节，节奏紧凑
   - 高潮 clip = 直接切入，不铺垫
   - 收势 clip = 情绪留白，可适当放慢

2. **单 clip 镜数 = 1 镜 1 种运镜 + 多动作 OK**（#15 铁律原文 · 不是 1 clip = 1 个"小短片"）
   - 1 个 clip = 1 种运镜（可固定/可推/可切/可拉，每镜 1 种）+ 多动作 OK（没说数量上限）
   - 反对"1 clip = 1 个完整微短剧"思维（推拉摇移 + 多镜铺垫 + 收势）
   - 反模式：clip 内塞"缓缓推近→切中景→点头→再张开"4 步铺垫

3. **整本节奏 = 儿童内容本来就快**
   - 旁白短、动作密 = 不要设计复杂慢节奏的镜头
   - "紧凑对齐 TTS"优先于"显得饱满"
   - 领读绘本/儿童动画/快节奏广告 = 通用场景

4. **时长 = 真实叙事秒数**（不凑整 · 铁律 #27 强化）
   - 旁白 TTS 真实需要 4s → 提交 4s，**不**凑整到 8s/10s
   - 时长是给叙事用的，不是给"显得时长饱满"用的
   - 整数（4-15 任意）都合法

5. **运镜 = 让参考图自身构图说话**（不是必须每个 clip 都设计运镜）
   - 固定镜头 = 合法运镜
   - 推拉摇移 = 工具不是义务
   - 让模型照参考图自由组织 = 自然连贯

**镜头设计快查表**（整本故事维度）：

| Clip 位置 | 节奏 | 运镜 | 动作密度 | 镜数 |
|---|---|---|---|---|
| **开篇**（clip1/2）| 慢 / 有铺垫 | 可推拉、铺氛围 | 低-中 | 1-2 |
| **中段**（中段 clip）| **快**、紧凑、**快切** | **直接切参考图角度**、少运镜 | 高 | 1 |
| **高潮**（高潮 clip）| **更快**、**直接切入** | 固定 / 简单运镜 | 极高 | 1 |
| **收势**（末 1-2 个 clip）| 稍慢（情绪留白）| 简单 | 中-低 | 1 |

**⚠️ 判定 clip 节奏位 = 看段内容，不是看 clip 序号**（**2026-06-14 绘本整本验证沉淀**）：

快查表是按 clip 序号假设的"开篇/中段/高潮/收势"位置，但**实际节奏位 = 看这个 clip 包含的旁白段内容决定**，不是按 clip 序号套：

- 旁白开头是 1 个单词（"DONKEY!"）+ 1 句短描述（"A little donkey"）+ 1 句特征描述（"long ears"）= **3 段短句 = 中段节奏 = 直接切入不铺垫** · **不**因为是 clip1 就套"开篇慢铺"
- 用户原话（2026-06-14）："**你看开始的时候就只是一个简单的单词，驴子 donkey，这种节奏是很快的。不需要有铺陈**" → 直接证据：clip1 = 短单词+短描述 = 快切不铺垫
- 修复：套快查表前**先读这个 clip 包含的旁白段** = 决定是开篇/中段/高潮/收势

**判定决策树**（写 prompt 前必走 2 步）：
1. 读这个 clip 包含的所有旁白段 = 看每段长度（单词数 + 字符数）+ 是否有铺垫（"在很久以前..."）= 决定节奏
2. 单段 ≤ 3 个词 = 快切 · 多段都有铺垫 = 可慢铺 · 多段都很短 = 中段直接切

**反模式**：
- ❌ clip1 = 默认"开篇慢铺"（用户原话纠错：短单词开篇 = 中段节奏）
- ❌ clipN = 默认"中段直接切"（如果整本只有 3 个 clip + 收势在第 3 段 = 第 3 段 = 收势不是中段）

**反问自检**（写 prompt 前必跑 4 问）：

- [ ] **Q1：这个 clip 在整本故事里是开篇/中段/高潮/收势哪个位置？**（决定节奏）
- [ ] **Q2：起承转合分配上，这个 clip 该做"起"还是该"直接切入"？**（决定是否有铺垫）
- [ ] **Q3：这个 clip 的旁白 TTS 真实多少秒？**（决定时长 = 不凑整）
- [ ] **Q4：这个 clip 必须做运镜吗？还是固定镜头 + 让参考图说话？**（决定运镜复杂度）

**反模式（必避）**：

- ❌ **每个 clip 都设计"起承转合"** = 把整本故事的维度套到单 clip
- ❌ **每个 clip 都做"推拉摇移"** = 镜头堆砌 = 呆板
- ❌ **时长凑整**（4s 凑 8s）= 多余时长 = 节奏拖沓
- ❌ **每 clip 都做"末帧定格静默 ≥ 2s"** = 整本故事有 N 个 2s 静默 = 节奏断裂

**适用场景**（通用方法论·不只是绘本）：

- 绘本视频 = 1 故事 N 个 clip（本场景）
- 漫剧 = 1 故事 N 个分镜（适用·AI drama skill）
- 短视频系列 = 1 主题 N 个视频（适用·内容账号）
- 直播切片 = 1 场直播 N 个高光（适用·快节奏拼接）
- 广告系列片 = 1 品牌 N 个分镜（适用·统一调性）

**合并 Clip 决策的"参考图 5 项一致性"判定**（**2026-06-16 Run 跑 强化 · 铁律 #26 衍生**）：

> 合并 N 张参考图到 1 个 Clip 时，**必查 5 项一致性** = 5 项一致 → 可合并 / 不一致 → 必拆

| 维度 | 一致 | 不一致 |
|---|---|---|
| **① 主体位置**（左/中/右）| 都中央 / 都左侧 = 可合并 | 左 vs 右 = 必拆 |
| **② 主体朝向**（侧身/正面 + 朝向）| 都朝右 / 都朝镜头 = 可合并 | 朝左 vs 朝右 = 必拆 |
| **③ 景别**（全身/中景/半身）| 都中景 / 都全身 = 可合并 | 全身 vs 特写 = 必拆 |
| **④ 主体姿态**（手/脚/头位置）| 接近 = 可合并（同一类动作）| 差别大 = 必拆 |
| **⑤ 招牌/文字位置**（顶部/中央/角落）| 都在同位置 = 可合并 | 位置不一致 = 必拆 |

**判定决策树**：

```
N 张参考图 → 能否合并到 1 个 Clip？
   ↓
逐对查 5 项
   ↓
全部一致 → 合并到 1 个 Clip（多图参考 v15 4 段）
部分一致 → 按"哪些一致"分 N 组 → N 个 Clip
全部不一致 → N 个 Clip
   ↓
**反模式**：
❌ 凭印象"4 张图能拼一个 Clip"（没查 5 项 = 必拼出冲突）
❌ 1 Clip 内 4 个不同主体（每图换主角 = 视频"瞬移"）
```

**旁白-镜头映射法**（2026-06-14 绘本 clip1 翻车沉淀 · **元方法论 #M3** · L0 顶层 · 与 #M1 / #M2 并列 · **2026-06-16 Run 跑 强化"参考图驱动 5 项"= 已被 sibling 修订 = 核心命题改为"参考图是起点"· 详见铁律 #26 + `references/reference-image-as-starting-point.md`**）：

> **核心命题**：写镜头序列 = 在画"旁白进度的视觉化映射表" · **不是**在给"几张参考图排景别"。

这是**比"信任参考图"和"整本节奏"更深一层**的设计视角。适用于所有"旁白驱动视频"场景（绘本/漫剧/短视频/广告/带解说词的纪录片/教学视频/产品演示）。

**为什么这是必需的思考方式**（不是"加几个动作"就能修的表层问题）：

镜头序列的本质 = **把"声音维度（旁白讲什么）"翻译成"视觉维度（画面展示什么）"**。如果这两条轨道脱钩：
- 旁白在推进叙事 + 画面在静态 = 视频没故事性（用户原话："**生成的是几乎静态的图片，光看视频的话没有故事性**"）
- 旁白在介绍特征 + 画面在景别排列 = 视频是"PPT 翻页"不是"故事"
- 旁白在高潮 + 画面在铺垫 = 视频节奏拖沓

**5 条核心规范**（每条都是通用创作规范，不只是绘本）：

1. **每个镜头 = 旁白进度里的 1 个"同步点"**（不是"1 张参考图"）
   - 同步点 = 旁白在这个时间段里"正在说什么/做什么"的瞬间
   - 画面 = 这个瞬间的"主角状态/动作/位置"的视觉表达
   - **判定口诀**：写 1 个镜头前先问"旁白在 0-X 秒在说什么？画面在展示这个时刻的主角哪个状态？"

2. **镜头数量 = 旁白叙事转折点**（不是"参考图数量"）
   - 旁白在"亮相" → 镜头 1
   - 旁白在"介绍特征" → 镜头 2
   - 旁白在"高潮/转折" → 镜头 N
   - **N 个旁白转折点 = N 个镜头**（不是 N 张参考图 = N 个镜头）
   - 1 张参考图可被多镜头复用（铁律 #4 跳过冲突图也是这个思路）

3. **末帧微动 = 这个旁白末态的姿态**（不是统一一个微动塞 N 个镜头）
   - 镜头 1 末帧 = 旁白在"亮相"末态的姿态（"抬头亮相定格"）
   - 镜头 2 末帧 = 旁白在"特征"末态的姿态（"特征定格"）
   - 镜头 3 末帧 = 旁白在"高潮"末态的姿态（"高潮定格"）
   - **反模式**：3 个镜头共用"长耳朵轻轻摆动"= 没有"末帧差异"= 没有"故事推进"

4. **镜头之间 = 旁白进度的"接续"**（不是"景别递进"）
   - 镜头 A 末态 → 镜头 B 起态 = 旁白从"上一句"过渡到"下一句" = 视觉化这个过渡
   - 不是"全身→脸特写→超大特写"这种"摄影机位排列"
   - **反模式**：旁白在"DONKEY! A little donkey. The donkey has long ears."（亮相→介绍→特征）= 画面在"全身→脸→超大脸"（3 个景别的同一姿势）= 脱钩

5. **参考图选择 = 服务于"同步点"**（不是"每张图用一次"）
   - 同步点需要的视觉状态 = 在参考图里找最接近的
   - 找不到 = 跳过/复用/描述延伸（铁律 #4）
   - **反模式**：拿到 8 张图 = 安排 8 个镜头（按"图"分）= 跟旁白脱钩

**必跑 4 步自检**（写每个镜头前必走·不靠记忆）：

```
镜头 N 写作前：
① 旁白进度：在 0-X 秒旁白在说什么/做什么？
   ├─ 单段 ≤ 3 词 = 快切镜头（#M1 整本节奏）
   ├─ 多段连读 = 中段直接切（铁律 #11）
   └─ 多段都有铺垫 = 可慢铺

② 画面状态：参考图里有这个时刻主角的状态/动作吗？
   ├─ 有 → 复用 + 描述延伸
   ├─ 没有 → 跳过冲突图（铁律 #4）+ 复用最接近的 + 描述延伸
   └─ 凭空加 = 必删（铁律 #3）

③ 末帧微动：这个旁白末态的姿态是什么？
   ├─ "亮相"末态 = 抬头定格
   ├─ "特征"末态 = 特征定格
   └─ "高潮"末态 = 高潮定格
   → 不与其他镜头共用

④ 跟上下镜头接续：上镜头末态 → 旁白下一句 → 镜头 N 起态 = 旁白进度的视觉化
   ├─ 旁白从 A 过渡到 B = 镜头从 A 状态过渡到 B 状态
   └─ 不是"景别递进"（全身→脸→超大脸 = 不是接续）
```

**+ 5 项必查（写 prompt 前必走 · 2026-06-16 Run 跑 强化）**：

```
镜头 N 写 prompt 前必查 5 项（**不**靠记忆）：
① 主体在画面哪个位置？（左/中/右/上下）→ 写进段 1 主体定义（**起点式**·不锁定）
② 主体朝向？（侧身/正面/背面 + 朝左/朝右）→ 写进段 1 + 段 2（**启发**·模型可自然过渡）
③ 景别？（全身/中景/半身/特写）→ 段 2 镜头描述（**参考图原景别 = 起点·但运镜自由推拉切**）
④ 主体姿态？（手/脚/头的具体位置）→ 写进段 1 主体定义（**核心动作起点**·非死锁）
⑤ 招牌/文字位置？（顶部/中央/右侧/角落）→ 段 2 文字保留 + 段 2 镜头描述（**位置准确**·不挪）
   ↓
5/5 全过 = 动笔
任 1 项漏 = 凭印象 = 跟参考图冲突 = 翻车（铁律 #26 · 配套见 reference-image-as-starting-point.md）
   ↓
⚠️ **5 项 = 启发性参照 ≠ 枷锁**（v2 翻车教训）
   ↓
✅ 允许的（用户原话级）：
   - 跑出画面（往画面外跑 = 镜头有方向感）
   - 转弯往回跑（动态 = 有变化）
   - 跑 1-2 秒再换动作（动态 = 节奏点）
   - 姿态一直在变（动画感来源）
   - 多镜头 / 单镜头 / 多机位（时长能容纳）
   - 镜头推进拉远切换角度（自由运镜）
```

**⚠️ 5 项一致性判定 = 合并 Clip 决策用**（不是每镜必查全部对齐 = 翻车根因）：

```
合并 N 张参考图到 1 个 Clip 时必查 5 项一致性：
- 一致 → 可合并
- 不一致 → 必拆 Clip

但每镜内部 = 不需要每条对齐 = 让模型自由发挥 = "动态呈现"
```

**反模式**（必避）：

- ❌ 3 张参考图 = 3 个景别（全身→脸→超大脸）= 静态景别排列
- ❌ 旁白在推进 + 画面在静态 = 两条轨道脱钩 = 视频没故事性
- ❌ 末帧微动 3 个镜头共用"长耳朵轻轻摆动"= 没有"末帧差异"
- ❌ 旁白在"特征揭示" + 画面在"全身像静态"= 旁白说的特征没在画面里被看见
- ❌ "8 张图 = 8 个镜头"（按"图"分）= 跟旁白脱钩（#M1 整本节奏的反面）

**正确范式**（口诀）：

> **"先看旁白说什么 → 找这个时刻主角的状态 → 在参考图里找最接近的 → 复用 + 描述延伸"**

**判定决策树**（写镜头序列前必走）：

```
写镜头序列前：
1. 通读本 clip 包含的所有旁白段
2. 标出每个旁白段的"叙事转折点"（亮相/介绍/特征/行动/高潮/收势）
3. 转折点数量 = 镜头数量（不 = 参考图数量）
4. 每个镜头 = 1 个转折点 + 1 张最接近的参考图（可复用）
5. 末帧微动 = 这个转折点的"末态姿态"（不与其他镜头共用）
```

## ⚡ 7 步工作流的节点应变能力（**2026-06-14 绘本实战沉淀 · 不破坏流程 = 流程不变 + 每个节点更灵活应对**）

> **本节是 7 步流程的"应变指南"，不替代流程本身。**流程是入门用的（7 步概述） + 详细说明（Step 1-7 详细段落）+ 应变能力（本节）= 完整。
>
> **什么时候用本节**：
> - 流程跑得通 = 跳过本节
> - 流程卡住 / 异常 / 输入不全 / 参数错 / 反馈不对 = 查本节对应 Step 的应变清单
> - 流程没卡但用户问"还有别的方案吗"= 用本节"灵活替代"区

### ⚡ Step 1 节点应变（接收需求）

| 异常 / 灵活场景 | 应变方案 |
|---|---|
| 4 件事缺 1-2 件（缺简介/缺 TTS/缺平台）| 列出**问题清单**问用户 · **不能编**（编 = 凭空捏造 = 翻车）|
| 文件格式乱（中文 GBK / 无扩展名 / .txt 当 readme）| `force utf-8 decode` · 不当数据源读 |
| `0.jpg` / `cover.jpg` 命名存疑 | 问用户"封面 / logo / 不用？"· 不假设 |
| 0 张图（用户说"我只有简介"）| **必停** + 问用户要图 · 不能凭简介编 |
| 收到 N 张图但简介说 M 张 | 以实际 N 为准 + 报告用户差异 |
| 多绘本混在一个目录 | 报告让用户分 · 不假设哪本是目标 |
| readme.txt vs 数据源 | 读内容不当数据源（**不强信**）|
| TTS 文件是 *.mp3 | 问"TTS 干声 / 完整音频 / 不用？" |
| 简介超长（>500 字）| 提炼**核心叙事弧** · 不全用 · 过长 = 拆 2 本 |

### ⚡ Step 2 节点应变（vision 自检 + 通读旁白）

| 异常 / 灵活场景 | 应变方案 |
|---|---|
| vision 部分失败（1-2 张图）| **重试失败的 1 张**（区分"临时过载" vs "真挂"）|
| vision 全部失败 | **立即降级** = 凭简介 + 兜底描述 + 主 agent 直干 · 必在 image-inventory 标"vision 缺失" |
| 图片模糊 / 太小 / 有水印 | 提示用户换图 / 或**先用低质图跑兜底版**（标"低质"等用户反馈）|
| 图片跟简介对不上（json 描述说 1 大 2 小，实际图 1 大 3 小）| **报告用户**（不是自己编）· vision 是真相 · 改 image-inventory |

| 图片包含文字（顶部大字）| 列 image-inventory "文字列" · 决定 v15 vs v6 范式 |

| 旁白 10+ 段（超长绘本）| 5 clip 上限 · 合并叙事弧 |
| 旁白 0 段（纯视觉绘本）| 靠简介 + 1.0 词/秒兜底估算 · 标"无 TTS" |

## Step 2 · vision 自检 + 通读旁白（**必全 N 张图**）

**2.0 image-inventory 必含"有/没有"两列**（**2026-06-14 绘本 元教训 #11 强化版**）：

```markdown

1 | 3D 折纸 | 1 只黄鸭·草地·纸艺 | 小鸭·荷叶·水·歪头·水花 | 顶部大字 | 站草地 | 草（独立）

3 | 3D 折纸 | 1 只黄鸭·踏浪姿态·蓝色水花 | 小鸭·荷叶·游泳·歪头 | 顶部大字 | 踏浪/跑步 | 水（连续·但姿态跟"游泳"主题冲突·**铁律 #4 跳过此图**）
```

**判定**：
- prompt 里**每个**名词必须在"有"列里 = 不凭空捏造
- "没有"列是**警告**——写 prompt 时**必**避免出现
- 任何"有"列元素都被某镜头引用 = 0 浪费

**反模式**（绘本 v1 v2 翻车）："4 只鸭立荷叶"——4 只鸭 / 荷叶 / 歪头 都不在"有"列 = 凭空捏造 = 必删

**2.1 vision 必跑·必全**（native vision·**全 N 张图必看光** · **流程内不确认**：分批 ≤ 3 张串行，但**不在批间跟用户确认**——直接连贯跑完写到 image-inventory.md 再进下一步 · 详见 `references/workflow-internal-vs-external-confirmation.md`）：

| vision 状态 | 动作 |
|---|---|
| 全成功 | 继续 |
| 部分失败 | 立即重试失败的 1 张验证（临时过载 vs 真挂） |
| 全部失败 | **立即降级**——不依赖 vision，凭简介+兜底描述走主 agent 直干 |

**⚠️ 必看全 = 全部 N 张图 = 标准**（不是抽样 1/N/中/末）。**只识别部分图 = 凭印象拼 prompt = 翻车征兆**。收到 8 张图 = vision 8 次 · 1 张都别跳。

**每图必看的 7 维**（缺 = prompt 拼错）：
1. 风格类型（彩纸拼贴 / 水彩 / 平面矢量 / 3D 毛毡 / 2D 卡通 / 3D 折纸拼贴）—— 决定 prompt 风格词
2. 主体数量（1 颗 / 2 颗 / 1+鸟 / 1+3 小 / 1+truck）—— 决定主体描述
3. 背景色调（米黄 / 青绿 / 草绿 / 蓝水 / 混合）—— 决定 prompt 背景描述
4. 文字是否在画面（顶部 1/2 居中 / 1/6 / 散落 / 角字卡 / 无文字）—— 决定是否让字母动画
5. **主体动作**（duck 游 / 跳 / 站 / 排队 / 看 truck / 张嘴 quack）—— 决定是否需要拟声
6. **场景跳变**（水 → 陆 = 跳变 · 同色系 = 连续）—— 决定是否拆 Clip
7. **主体对位**（多图合并时多主体关系：duck vs truck / 大鸭 vs 3 小鸭）—— 决定段 2 镜头拆分

**每图必看的 5 维加强版**（**铁律 #26 · 2026-06-16 Run 跑跨本验证补强 · 必查**）：
8. **主体位置**（画面左/中/右/上/下）—— 写 prompt 时**必写**进主体定义段（**不写 = 凭印象 = 翻车**）
9. **主体朝向**（面对镜头/侧身/背面/朝左/朝右）—— 写 prompt 时**必写**进主体定义段
10. **景别**（全身/中景/半身/特写）—— 写 prompt 时**必写**"固定 XXX 景别不做推拉"
11. **主体姿态细节**（手/脚/头/躯干的具体位置）—— 写 prompt 时**必写**"左脚在前右脚在后 + 双臂前后摆动"等

**说明**：维度 1-7 是"是什么"（决定描述内容），维度 8-12 是"在哪里/怎么朝"（决定描述位置/朝向）。**两者合并 12 维 = 必查**。漏 8-12 中任一项 = 写出来的 prompt 跟参考图冲突（v1 Run 跑 Clip 3 实战错例 = 漏 8/9/10/12 共 4 项 = 5 处冲突全部浮现）。详见铁律 #26 + `references/reference-image-as-starting-point.md`（铁律 #26 支撑文档）。

## Step 3 · 标叙事弧 → 合并到 3-5 Clip（**导演分镜**·核心步骤）

**3.1 合并决策树**：

```
通读旁白 → 标叙事弧
   ↓
弧的数量 = N（N 通常 2-4）
   ↓
每个弧 = 1 个 Clip（每 Clip 装 1-3 张图）
   ↓
N=3 → 3 Clip（最常见·甜区）
N=4 → 4 Clip
N=2 → 2 Clip（极简绘本）
N>5 → 合并叙事弧到 5 Clip 以内
   ↓
   ↓
Step 3.2 算 TTS 时长（1.0 词/秒 + 3 字/秒 + 0.5s 词间停顿·用户给 TTS 优先）
   ↓
Step 3.3 整数时长分配（每 Clip 时长 = 装得下对应故事段 TTS + 视觉延展）
   ↓
**每 Clip 时长硬上限 ≤ 15s**（seedance duration 物理上限·API 拒收 > 15s）
**> 15s 必拆**（拆成 2 个 ≤ 15s Clip）· 拆依据看 image-inventory 的"场景跳变"列
   ↓
**视频总时长 = 用户给 TTS + 5s 冗余**（不缩短·按用户原则）
   ↓
静默按故事连贯性编排（不是每段必塞 ≥ 2s 末帧静默）
```

**3.2 算 TTS 时长（**必算**·用户原话 6 个根本性纠错 + 铁律 #25 跨本验证升级）**：

| 速率 | 公式 | 用途 |
|---|---|---|
| **用户给 TTS** | 必收· = 视频总时长基准 | **优先级最高**·视频总时长 = TTS + 5s 冗余 |

## 与子 agent 的关系

主 agent **必做**：
- ✅ 验证子 agent 输出 schema
- ✅ 持久化子 agent 输出到磁盘
- ✅ 翻车时决定重发哪个子 agent
- ✅ 接受/降级决策
- ✅ 端到端验证 + 用户目检
- ✅ **vision 全 N 张图必跑**（不抽样）
- ✅ **必算 TTS 时长**（1.0 词/秒 + 3 字/秒 + 0.5s 词间停顿）

主 agent **不做**：
- ❌ 拼 prompt（用 `scripts/fill_v15_template.py` 填模板）
- ❌ 跑视频（用 `seedance.py` 或 `mcp_seedance_generate_video`）
- ❌ 抽帧验证（vision 是辅助不是真理·等用户目检）
- ❌ **凭印象算 Clip 时长**（不查 TTS = 严重违规）
- ❌ **用首尾帧范式**（`--image`+`--last-frame` = 用户元偏好"绝对不用"）

**4 个子 agent**（不直接调，记下来即可）：

| Agent | 职责 | Skill |
|---|---|---|
| **A · 风格识别** | 调性 + 节奏倾向 + 风格锚定词 | `storyboard-style` |
| **B · 旁白量化** | 朗读时长 + 复杂度 + 静默推荐 | `storyboard-narration` |
| **C · 分镜设计** | 节奏公式 + 镜头表 + 11 维 JSON | `storyboard-design` |
| **D · 视频执行** | seedance 跑 + 抽帧 + vision 自检 | `video-executor` |

**默认调用规则**：
- Clip ≤ 4：主 agent 直干（不调 C）
- Clip > 4：调 C 子 agent
- A + B 始终主 agent 干（纯计算/少量 vision）
